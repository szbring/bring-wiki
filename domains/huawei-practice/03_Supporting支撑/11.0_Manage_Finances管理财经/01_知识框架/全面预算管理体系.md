# 全面预算管理体系

<!-- chunk: 全面预算管理体系.一句话定义 -->
## 一句话定义
以客户为源头、以项目为基础，通过"计划-预算-核算-考核"闭环运作，实现战略牵引与资源配置的动态平衡。

---

<!-- chunk: 全面预算管理体系.一-华为全面预算的核心逻辑 -->
## 一、华为全面预算的核心逻辑

华为全面预算的根本出发点是任正非提出的：**"我们的利润来源于客户，因此我们的预算源头也应该是客户。"** 年度预算制定遵循"战略计划—项目—预算"逻辑，基于项目/机会点进行预算分配。

### 六条预算原则

| 序号 | 原则 | 含义 |
|---|---|---|
| 1 | 预算反映经营责任 | 以绩效评价闭环，谁承担预算谁对结果负责 |
| 2 | 预算以客户为源头 | 从客户销售预算向后分解，而非从费用项向前加总 |
| 3 | 预算以项目为基础 | 项目是最小经营单元，平台预算从项目反推 |
| 4 | 预算反映业务实质 | 业财一体，预算与业务计划强关联 |
| 5 | 预算实行弹性管理 | 权力下放，滚动预测，动态调整 |
| 6 | 预算牵引资源配置 | 预算不是简单控制工具，而是资源优化配置手段 |

## 二、预算管理框架

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ 中长期财务规划 │──→│  年度计划预算  │──→│  管理预测    │
│ (3-5年战略)  │    │ (自上而下+    │    │ (月度滚动)  │
│              │    │  自下而上)    │    │             │
└─────────────┘    └─────────────┘    └─────────────┘
                          │                   │
                          ▼                   ▼
                   ┌─────────────┐    ┌─────────────┐
                   │  预算授予管控  │    │ 财务绩效评价  │
                   │ (刚性+弹性)  │    │ (考核闭环)   │
                   └─────────────┘    └─────────────┘
```

### 关键组织

| 组织 | 预算角色 |
|---|---|
| 董事会（BOD） | 年度预算最终审批 |
| 财经委员会（FC） | 全面预算日常管理 |
| 集团财经管理部 | 日常工作协调、预算编制组织 |
| 各级责任中心 | 预算编制与执行主体，纵横互锁 |

<!-- chunk: 全面预算管理体系.三-预算编制流程 -->
## 三、预算编制流程

华为年度预算编制遵循"三步走"：

**第一步：自上而下** — 经营目标下达
- 董事会审议年度经营目标
- FC分解到BG/区域/职能部门
- 明确收入、利润、现金流、人均效益等核心指标

**第二步：自下而上** — 业务及项目预算上报
- 各责任中心基于业务计划编制预算
- 项目预算是基本单元，平台预算从项目反推
- 业务做"语文题"（业务计划），财务做"数学题"（预算测算）

**第三步：上下结合** — 全面预算最终形成
- FC汇总审核，多轮质询与调整
- 董事会批准发布

### 年度预算编制时间线

| 时间节点 | 工作内容 |
|---|---|
| Q3（7-9月） | 中长期战略审视，年度业务计划启动 |
| Q4初（10月） | 经营目标自上而下下达 |
| Q4中（11月） | 各责任中心编制并上报预算 |
| Q4末（12月） | 预算质询、调整、审批 |
| 年初（1月） | 预算正式发布，授权执行 |

<!-- chunk: 全面预算管理体系.四-平台预算与项目预算分开管理 -->
## 四、平台预算与项目预算分开管理

华为强调将**平台预算**和**项目预算**分开管理，这是预算体系的核心创新：

| 维度 | 项目预算 | 平台预算 |
|---|---|---|
| 性质 | 直接产生收入 | 不产生收入，为项目服务 |
| 编制方式 | 基于机会点自下而上 | 从项目预算反推 |
| 管控方式 | 项目经理控制 | 基线管控，最低成本运行 |
| 机制 | 项目向平台购买资源 | 平台向项目"要钱" |

核心机制：**项目向平台购买资源**，项目经理严格控制费用，平台要想方设法把费用合理分摊到项目中，形成双向博弈与平衡。

<!-- chunk: 全面预算管理体系.五-弹性预算与-节约不归己 -->
## 五、弹性预算与"节约不归己"

### 弹性预算
- 预算不是刚性的天花板，而是资源分配的参考基线
- 业务计划发生变化时，可以重新申请预算
- 通过月度/季度滚动预测，动态调整资源配置

### "节约不归己"原则
华为对研发战略投入设有"节约不归己"原则：
- 研发战略预算若未花完，年底核算时仍按原预算额度计算成本
- 例如：研发战略预算20亿元，实际只花15亿元，核算时仍按20亿元计入
- 目的：**防止经营管理者为短期绩效牺牲公司长远能力**

<!-- chunk: 全面预算管理体系.六-预算执行与考核闭环 -->
## 六、预算执行与考核闭环

```
计划预算(牵引) → 执行管控(过程) → 核算监控(评估) → 绩效考核(闭环) → 下一轮计划
```

### 考核核心指标
- 有效增长（收入增长率）
- 利润（营业利润率）
- 现金流（经营性现金流）
- 人均效益（收入/人、利润/人）

**刚性约束**：凡不能达到公司人均效益提升平均线以上的，对体系团队负责人进行问责。超越平均线以上的部门，按正利润、正现金流、战略目标实现进行排序。

<!-- chunk: 全面预算管理体系.七-常见问题与诊断要点 -->
## 七、常见问题与诊断要点

| 典型问题 | 根因 | 改进方向 |
|---|---|---|
| 预算编制变成"数字游戏" | 业务计划与预算脱节，增量法简单加总 | 建立"战略-计划-预算"联动，零基预算 |
| 预算执行偏差超30% | 缺乏滚动预测，业务变化未及时反映 | 月度滚动预测，弹性预算机制 |
| 平台费用膨胀 | 平台预算与项目预算未分离 | 项目制核算，平台从项目反推 |
| 研发投入被压缩保利润 | 短期考核导向 | "节约不归己"，战略投入单列考核 |
| 预算管控全靠审批 | 事前事中管控缺失 | 建立预算基线，系统化管控 |
