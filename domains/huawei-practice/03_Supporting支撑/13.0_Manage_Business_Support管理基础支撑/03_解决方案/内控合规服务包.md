# 内控合规服务包

<!-- chunk: 内控合规服务包.一句话定义 -->
## 一句话定义
帮助企业建立内控三道防线和[[华为质量运营管理]]体系，满足上市监管要求，实现从"被动应对"到"体系管控"的转变。

---

<!-- chunk: 内控合规服务包.一-服务包概述 -->
## 一、服务包概述

| 项目 | 内容 |
|---|---|
| 服务名称 | 内控合规管理体系建设 |
| 服务周期 | 3-6个月 |
| 投入人天 | 20-40人天 |
| 适用客户 | 上市前企业、内控体系缺失企业、合规风险较高企业 |
| 报价范围 | 30-80万（视规模和范围） |

---

<!-- chunk: 内控合规服务包.二-服务内容-四个模块 -->
## 二、服务内容（四个模块）

### 模块1：内控体系建设（8-15人天）

| 工作项 | 交付物 |
|---|---|
| 内控现状诊断 | 《内控成熟度评估报告》 |
| 内控组织与三道防线设计 | 《内控组织与权责手册》 |
| 授权体系设计 | 《授权审批矩阵》 |
| 关键控制点识别 | 《关键控制点清单》 |

### 模块2：流程控制设计（6-12人天）

| 工作项 | 交付物 |
|---|---|
| 核心流程梳理（采购/销售/资金/资产） | 《流程控制手册》 |
| 职责分离矩阵设计 | 《职责分离矩阵》 |
| 控制活动设计（审批/核对/校验） | 《控制活动规范》 |
| 流程控制文档化 | 《流程图+控制说明》 |

### 模块3：合规管理体系（4-8人天）

| 工作项 | 交付物 |
|---|---|
| 合规义务清单梳理 | 《合规义务清单》 |
| 合规风险评估 | 《合规风险矩阵》 |
| 合规制度体系设计 | 《合规管理办法》 |
| 合规审查流程设计 | 《合规审查SOP》 |

### 模块4：内控自评与持续改进（4-8人天）

| 工作项 | 交付物 |
|---|---|
| 内控自评工具设计 | 《内控自评问卷和模板》 |
| 缺陷分级与整改流程 | 《缺陷整改管理办法》 |
| 内控培训 | 《内控培训材料》 |
| 持续监控机制 | 《内控监控指标体系》 |

<!-- chunk: 内控合规服务包.三-实施路径 -->
## 三、实施路径

```
月1        月2        月3        月4        月5        月6
├──────────┤──────────┤──────────┤──────────┤──────────┤
│ 诊断+    │ 内控体系  │ 流程控制  │ 合规体系  │ 自评+    │ 验收固化  │
│ 方案设计 │ 设计      │ 设计      │ 建设      │ 培训     │ +知识转移 │
```

### 关键里程碑

| 里程碑 | 时间 | 验收标准 |
|---|---|---|
| M1 诊断报告通过 | 月1末 | 客户确认诊断发现 |
| M2 内控组织与授权发布 | 月2末 | 三道防线明确、授权矩阵审批 |
| M3 流程控制手册发布 | 月3末 | 关键流程控制文档化 |
| M4 合规制度发布 | 月4末 | 合规制度审批发布 |
| M5 首轮内控自评 | 月5末 | 自评完成、缺陷清单确认 |
| M6 项目验收 | 月6末 | 体系可独立运行 |

<!-- chunk: 内控合规服务包.四-常见阻力与应对 -->
## 四、常见阻力与应对

| 阻力 | 表现 | 应对策略 |
|---|---|---|
| "内控就是限制业务" | 业务部门消极 | 内控=保障业务安全运行，不是限制 |
| "我们小公司不需要这么复杂" | 以规模为借口 | 适度简化，但核心控制点不可省 |
| "三道防线增加成本" | 管理层质疑投入 | 合规成本 < 违规成本，量化风险 |
| "审计发现问题没人改" | 整改不闭环 | 缺陷纳入考核，闭环率100%目标 |

## 五、轻量版服务包

适合快速达标场景：

| 项目 | 内容 |
|---|---|
| 服务周期 | 1-2个月 |
| 投入人天 | 8-15人天 |
| 报价 | 15-30万 |
| 范围 | 三道防线+授权体系+核心流程控制（不含合规专项） |

---

<!-- chunk: 内控合规服务包.六-成功标准 -->
## 六、成功标准

1. 三道防线组织到位，职责清晰
2. 关键流程控制文档化，控制点有明确SOP
3. 授权审批矩阵发布并执行
4. 首轮内控自评完成，缺陷有整改计划
5. 合规制度发布，合规审查流程可运行
