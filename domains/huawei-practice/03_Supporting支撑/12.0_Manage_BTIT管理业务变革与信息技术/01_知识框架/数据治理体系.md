# 数据治理体系

<!-- chunk: 数据治理体系.一句话定义 -->
## 一句话定义
通过"4层保障+2个抓手"，让数据"进得来、理得清、出得去、用得安"，实现数据从成本到资产的转化。

---

<!-- chunk: 数据治理体系.一-华为数据治理两阶段 -->
## 一、华为数据治理两阶段

| 阶段 | 时间 | 核心任务 | 关键成果 |
|---|---|---|---|
| 阶段一 | 2007-2017 | 建立数据管理框架 | 统一信息架构与标准、数据质量改进机制、数据全流程贯通 |
| 阶段二 | 2017-至今 | 建设数据底座 | 数据湖/仓、全域数据汇聚、数据服务化、业务可视与自助 |

---

<!-- chunk: 数据治理体系.二-4层保障-2个抓手 -->
## 二、4层保障+2个抓手

### 2.1 四层保障

| 保障层 | 核心内容 | 关键机制 |
|---|---|---|
| 政策保障 | 目的、适用范围、管理原则、问责 | 公司统一政策，业务与IT共同参与 |
| 组织保障 | 按领域任命数据Owner和团队 | 实体化数据管理组织承接改进目标 |
| 流程保障 | 数据管理流程嵌入变革与运营体系 | 变革指导委员会决策，流程运营落地 |
| IT落地保障 | 数据底座、数据服务、统一IT平台 | 数据全流程流转与价值变现 |

### 2.2 两个抓手

| 抓手 | 核心任务 | 解决问题 |
|---|---|---|
| 信息架构 | 构建面向"业务交易"的信息架构，统一"数据语言" | 数据孤岛、语义不一致、跨系统集成困难 |
| 数据质量 | 建立数据质量管理框架和运作机制 | 数据不准确、不完整、不及时 |

<!-- chunk: 数据治理体系.三-数据质量管理 -->
## 三、数据质量管理

### 3.1 数据质量六维度

| 维度 | 定义 | 示例 |
|---|---|---|
| 完整性 | 数据不缺失 | 客户信息字段填充率>95% |
| 准确性 | 数据与事实一致 | 订单金额与合同一致 |
| 一致性 | 跨系统数据一致 | ERP与CRM中客户名称一致 |
| 及时性 | 数据按时可用 | 日结数据T+1可用 |
| 唯一性 | 数据不重复 | 客户主数据无重复记录 |
| 合规性 | 数据符合规则 | 手机号格式、身份证校验 |

### 3.2 数据质量管理流程

```
质量规则定义 → 数据探查 → 质量评估 → 问题定位 → 根因分析 → 改进措施 → 持续监控
```

### 3.3 数据质量运营

- 例行开展公司级数据质量评估
- 数据管理组织定期发布质量报告
- 牵引各业务领域持续改进
- 质量指标纳入数据Owner绩效

<!-- chunk: 数据治理体系.四-数据标准 -->
## 四、数据标准

### 4.1 数据标准体系

| 标准类型 | 内容 | 管理方式 |
|---|---|---|
| 业务数据标准（BDS） | 数据定义、业务规则、取值范围 | 业务Owner制定，IT实现 |
| 技术数据标准 | 数据类型、长度、格式、校验规则 | IT部门制定，开发遵守 |
| 主数据标准 | 核心业务对象的标准定义和编码 | 数据治理委员会审批 |
| 指标标准 | 指标定义、计算公式、口径 | 管理会计/BI团队制定 |

### 4.2 数据标准落地

```
标准制定 → 系统对标 → 差异分析 → 改造实施 → 持续遵从检查
```

## 五、主数据管理

### 5.1 主数据识别

| 主数据类型 | 说明 | 典型字段 |
|---|---|---|
| 客户主数据 | 客户唯一标识和核心属性 | 客户编码、名称、分类、行业、区域 |
| 产品主数据 | 产品/SKU唯一标识 | 产品编码、名称、品类、规格、BOM |
| 组织主数据 | 组织架构和人员 | 组织编码、层级、人员ID、岗位 |
| 供应商主数据 | 供应商唯一标识 | 供应商编码、名称、分类、资质 |
| 科目主数据 | 财务科目体系 | 科目编码、名称、层级、核算规则 |

### 5.2 主数据管理原则

1. **单一信源** — 每个主数据只有一个权威数据源
2. **集中管理** — 主数据集中创建、分发、维护
3. **全局共享** — 跨系统一致，消除数据版本差异
4. **变更管控** — 主数据变更经过审批流程

<!-- chunk: 数据治理体系.六-数据底座建设 -->
## 六、数据底座建设

华为数据底座实现"进得来、理得清、出得去、用得安"：

| 能力 | 解决问题 | 技术支撑 |
|---|---|---|
| 进得来 | 数据全面入湖，打破孤岛 | 数据集成工具、批量+实时入湖 |
| 理得清 | 端到端数据质量保障 | 信息架构、质量规则、质量稽核 |
| 出得去 | 数据按需自助消费 | 数据服务、数据地图、API管理 |
| 用得安 | 数据安全与隐私保护 | 数据分级分类、脱敏、水印、权限管控 |

<!-- chunk: 数据治理体系.七-薄云咨询适用场景 -->
## 七、薄云咨询适用场景

1. **数据治理体系搭建** — 从0到1建立数据治理组织、制度、流程
2. **主数据管理** — 客户/产品/组织等主数据管理专项
3. **数据质量提升** — 质量规则定义、质量评估、改进闭环
4. **数据标准建设** — 统一数据语言，消除系统间差异
5. **数据底座规划** — 数据湖/仓架构设计、数据服务化
