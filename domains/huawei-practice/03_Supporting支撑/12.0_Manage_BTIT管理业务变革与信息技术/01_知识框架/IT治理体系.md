# IT治理体系

<!-- chunk: it治理体系.一句话定义 -->
## 一句话定义
确保IT投资与企业战略一致，有效管理IT风险，保障IT合规，构建IT可持续发展的长效机制。

---

<!-- chunk: it治理体系.一-it治理的定义与目标 -->
## 一、IT治理的定义与目标

### IT治理 vs 企业治理 vs IT管理

| 概念 | 关注点 | 类比 |
|---|---|---|
| 企业治理 | 利益相关者权益、战略方向、投资回报 | 宪法 |
| IT治理 | IT投资与战略对齐、IT风险管控、IT价值交付 | 法律 |
| IT管理 | 在既定治理模式下实现目标的具体行动 | 行政 |

### IT治理四大目标

| 目标 | 说明 |
|---|---|
| 1. 战略对齐 | IT投资与企业战略方向一致，构筑核心竞争力 |
| 2. 价值交付 | IT按承诺在成本/时间/质量约束内交付业务价值 |
| 3. 风险管理 | 识别和管理IT相关风险（安全、合规、业务连续性） |
| 4. 资源管理 | 优化IT资源使用效率，有效利用信息资源 |
| 5. 绩效度量 | 建立IT绩效度量体系，跟踪和改进IT交付效果 |

<!-- chunk: it治理体系.二-it治理组织模式 -->
## 二、IT治理组织模式

| 模式 | 特点 | 优势 | 劣势 | 适用场景 |
|---|---|---|---|---|
| 专制型 | IT部门集中决策 | 成本可控，资产统一管理 | 响应慢，缺乏创新 | 追求短期利润的公司 |
| 分散型 | 各业务部门独立决策 | 响应及时，创新强 | 标准不一，成本高 | 新成立/研发型公司 |
| 联邦型 | IT与业务联合决策 | 平衡控制与灵活 | 协调难度大 | 持续增长+创新的公司 |

**华为采用联邦型**：IT部门与业务部门联合进行IT管理，各自有明确职责。IT准则强调流程、系统、技术和数据模型的共享与重用。

<!-- chunk: it治理体系.三-it投资决策 -->
## 三、IT投资决策

### 投资决策框架

```
业务需求 ──→ 架构评审 ──→ 投资优先级排序 ──→ 预算分配 ──→ 项目立项
    │            │              │              │           │
    ▼            ▼              ▼              ▼           ▼
 业务价值    架构合规性     ROI分析      资源约束    Charter审批
```

### IT投资分类

| 投资类别 | 说明 | 决策层级 | 评估重点 |
|---|---|---|---|
| 战略型 | 支撑公司战略的关键项目 | 董事会/EMT | 战略对齐度、长期价值 |
| 增长型 | 业务增长驱动的项目 | BG/区域 | ROI、收入贡献 |
| 运营型 | 维持现有运营的项目 | CIO | 成本优化、效率提升 |
| 合规型 | 满足法规/安全要求 | CIO/CRO | 风险降低、合规达成 |

### 华为IT投资原则

1. **战略驱动** — IT投资必须支撑业务战略
2. **架构牵引** — 新建系统必须通过架构合规审查
3. **价值导向** — 投资决策基于业务价值而非技术偏好
4. **平台优先** — 优先建设可复用的平台能力

<!-- chunk: it治理体系.四-it风险管理 -->
## 四、IT风险管理

### IT风险分类

| 风险类别 | 具体风险 | 管控措施 |
|---|---|---|
| 业务连续性 | 系统宕机、灾备失效 | 双活/多活架构、灾备演练、RTO/RPO达标 |
| 信息安全 | 数据泄露、网络攻击 | 等保合规、零信任架构、安全运营中心 |
| 隐私保护 | 个人信息违规使用 | 数据分级分类、脱敏、合规审计 |
| 项目风险 | 超预算、超期、需求变更 | 项目治理、变更管控、里程碑评审 |
| 技术风险 | 技术选型错误、供应商锁定 | 技术评估框架、开源策略、多供应商 |
| 合规风险 | 法规不合规（如GDPR） | 合规扫描、制度更新、定期审计 |

### IT风险管理流程

```
风险识别 → 风险评估 → 风险处置 → 持续监控 → 报告与改进
```

## 五、IT合规管理

### 合规框架

| 合规领域 | 主要法规/标准 | 关键要求 |
|---|---|---|
| 网络安全 | 等保2.0、ISO 27001 | 安全架构、访问控制、审计日志 |
| 数据隐私 | GDPR、个保法、数据安全法 | 数据分类分级、隐私影响评估、跨境合规 |
| 软件可信 | 软件工程能力、SBOM | 安全开发生命周期、供应链安全 |
| 行业监管 | SOX、金融监管 | IT内控、审计轨迹、变更管理 |
| 开源合规 | 开源许可证 | 开源使用审批、许可证扫描、合规报告 |

### 华为合规实践

- 制定员工商业行为准则（BCG），全员培训与签署
- 全球统一的内控管理制度，基于COSO模型
- 设立内控与风险管理部门，定期风险评估
- 内部审计独立评估IT内控有效性

## 七、薄云咨询适用场景

1. **IT治理体系建立** — 建立IT投资决策、风险管理、合规管理机制
