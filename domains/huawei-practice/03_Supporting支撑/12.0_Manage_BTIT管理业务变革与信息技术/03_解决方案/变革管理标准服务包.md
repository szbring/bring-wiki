## 二、各服务包详细描述

### 服务包1：变革就绪度诊断（Change Diagnostics）

**适用场景**：
- 变革项目启动前，需要了解"现在是否是合适的时机"
- 变革项目执行受阻，需要找到根本原因
- 董事会/高管层想了解组织变革能力的真实水平

**服务内容**：

| 模块 | 具体工作 | 交付物 |
|------|---------|--------|
| 六维就绪度评估 | 高管访谈（5-8人）+ 管理层问卷（20-50人） | 变革就绪度评估报告 |
| 利益关系人分析 | 识别关键干系人，绘制影响力/支持度地图 | 利益关系人分析矩阵 |
| 阻力识别 | 深度访谈阻力群体（5-10人），识别根因 | 阻力清单与应对建议 |
| 诊断报告呈现 | 向IPMT/高管汇报诊断发现，提出行动建议 | 汇报PPT + Q&A会议 |

**周期**：3-4周

**人天**：15-20人天（顾问2人）

**报价参考**：8-15万元（根据企业规模调整）

---

### 服务包2：变革蓝图设计（Change Blueprint）

**适用场景**：
- 需要专业辅助编写高质量的变革项目Charter
- 变革策略不清晰，需要从零设计变革路径
- 想学习华为[[IPD]]管理方法论并落地应用

**服务内容**：

| 模块 | 具体工作 | 交付物 |
|------|---------|--------|
| Charter辅导 | 逐章辅导填写Charter，2-3轮评审反馈 | 高质量Charter材料包 |
| 变革路径设计 | 设计诊断→设计→试点→推广→固化路径 | 变革路标甘特图 |
| 变革管理策略 | 制定沟通策略、培训策略、Champion体系 | 变革管理策略文档 |
| 沟通计划制定 | 设计关键信息和沟通渠道矩阵 | 变革沟通计划 |
| DCP0评审准备 | 模拟IPMT评审，提前应对关键质疑 | 评审准备清单 |

**周期**：4-6周

**人天**：20-30人天（顾问2人）

**报价参考**：15-25万元（根据Charter复杂度调整）

---

### 服务包3：变革实施陪跑（Change Accelerator）

**适用场景**：
- 变革项目已立项，需要全程专业的变革管理支持
- 内部缺乏专职变革经理，需要外部资源补充
- 变革推进遭遇严重阻力，急需外部力量介入

**服务内容**：

| 模块 | 具体工作 | 交付物 |
|------|---------|--------|
| 变革经理嵌入 | 驻场或远程担任变革经理（每周2-3天） | 周报、阻力跟踪表 |
| 赞助人辅导 | 每月1次赞助人辅导会议，提升赞助人效能 | 赞助人辅导笔记 |
| Champion体系 | 建立并运营变革大使网络 | Champion培训材料 |
| 沟通执行 | 辅助准备关键沟通材料（全员信/路演PPT等） | 沟通材料库 |
| 培训设计 | 设计变革相关培训方案并推动执行 | 培训计划与材料 |
| 进度监测 | 月度变革成效评估（KPI追踪+问卷） | 月度变革成效报告 |
| DCP评审支持 | 准备并参与各阶段DCP评审 | DCP评审材料 |

**周期**：6-12个月（按项目周期灵活调整）

**人天**：60-120人天/年（顾问1-2人，以季度合同为主）

**报价参考**：40-80万元/年（月度合同可按30%溢价）

**适用场景**：

- 希望培养内部变革管理能力

**服务内容**：

| 变革管理入门工作坊 | 1天 | 10-20人 | 变革管理框架、工具入门、案例讨论 | 2-3万元 |
| Charter编写实战营 | 2天 | 5-10人 | 逐章填写Charter，现场顾问点评 | 4-6万元 |
| 赞助人有效性工作坊 | 半天 | 3-5名高管 | 赞助人角色、行为示范、阻力应对 | 2-3万元 |

## 三、变革管理与IPD咨询的结合方案

### 为什么变革管理和IPD要结合？

IPD（集成产品开发）是华为产品开发体系的核心，对于中国科技企业来说，引入IPD是一项重大的组织变革。但许多企业发现：**IPD流程设计得很好，但推不动**——根本原因在于缺乏专业的变革管理。

```
纯IPD咨询的局限：
 流程设计 ✅
 系统实施 ✅
 人员培训 ⚠️（通常不足）
 文化适应 ❌（通常缺失）
 阻力管理 ❌（通常忽视）
 
IPD + 变革管理的完整方案：
 流程设计 ✅  + 变革路径规划 ✅
 系统实施 ✅  + Champion体系 ✅
 人员培训 ✅  + 文化氛围建设 ✅
 变革就绪度评估 ✅ + 阻力应对方案 ✅
```

### 结合方案的三种模式

#### 模式1：并行模式（推荐）

IPD咨询顾问 + 变革管理顾问同时进场，各司其职，定期同步。

**适用**：大型IPD变革项目（>1年周期，>100人受影响）

**优势**：变革管理从一开始就介入，防患于未然

**资源**：IPD团队3-5人 + 变革管理顾问1-2人

#### 模式2：嵌入模式

在IPD咨询团队中嵌入1名变革管理顾问，统一指挥。

**适用**：中型IPD变革项目（6-12个月，50-100人受影响）

**优势**：沟通成本低，协同效率高

**资源**：IPD团队3-4人（其中1人变革管理专长）

#### 模式3：救援模式

IPD咨询已经推进一段时间，遭遇阻力后再引入变革管理顾问。

**适用**：IPD推行受阻，需要快速诊断并解决

**优势**：聚焦最紧迫问题，效率高

**劣势**：比从一开始引入需要更多时间恢复信任

**注意**：救援模式成本通常比并行模式高30-50%（因为要先解决已有问题）

### 薄云组合服务报价参考

| 组合方案 | 适用规模 | 周期 | 报价区间 |
|---------|---------|------|---------|
| IPD全程陪跑 + 变革管理陪跑 | 500人以上企业 | 18-24个月 | 200-400万元 |
| IPD重点阶段 + 变革诊断+蓝图 | 100-500人企业 | 6-12个月 | 60-120万元 |
| Charter编写 + 变革诊断 | 50-100人企业 | 2-3个月 | 20-40万元 |

> 💡 **内部报价说明**：以上为对外参考报价，实际报价需考虑客户行业、项目复杂度、竞争格局等因素。大客户可有15-20%浮动空间。

## 四、完整变革生命周期服务设计

> 补充来源：《常变与长青》郭平著，2024年——变革管理8阶段框架

薄云咨询的变革管理服务覆盖华为变革管理8阶段全生命周期：

### 4.1 变革生命周期与薄云服务对应

| 华为变革阶段 | 阶段目标 | 薄云服务内容 |
|------------|---------|------------|
| **变革规划** | 战略对齐，识别变革机会 | 业务战略解读，变革机会诊断，BT/IT S&P规划辅导 |
| **概念（DCP0）** | 立项，确定范围与目标 | Charter编写辅导，变革愿景设计，Burning Platform构建 |
| **计划（DCP1）** | 详细路标，资源预算 | 4A架构蓝图设计，项目规划辅导，变革管理计划制定 |
| **开发（DCP2）** | 流程/系统/培训材料开发 | 变革管理策略执行，沟通计划实施，利益关系人管理 |
| **验证（DCP3）** | 小范围验证方案可行 | 验证环境搭建辅导，问题识别与解决 |
| **试点（DCP4）** | 受控试点，收集反馈 | 试点辅导，阻力应对，试点评估报告 |
| **推行（DCP5）** | 全面推广至所有单元 | 推行变革经理陪跑，Champion网络激活，培训体系运营 |
| **运营管理** | 固化，持续优化 | 固化评估，绩效监控，持续改进机制建立 |

### 4.2 三个关键过渡节点的服务重点

**节点1：从规划到试点**
- 确保方案"足够好"（防完美主义陷阱）
- 试点单位选择与准备
- 试点评估标准预先定义

**节点2：从试点到推行**
- 试点经验总结与方案优化
- 推行路线图设计（谁先谁后）
- 大规模培训体系准备

**节点3：从推行到固化**
- 新行为是否真正内化
- 绩效考核挂钩检查（防反弹）
- 持续优化机制建立

## 五、顾问服务模式

### 5.1 咨询顾问+业务专家的组合模式

薄云咨询的服务模式参照华为与IBM合作经验，采用"咨询顾问+客户业务专家"组合：

```
薄云咨询顾问团队          +          客户业务专家团队
─────────────────                    ─────────────────
• 变革管理方法论                      • 业务知识与流程理解
• 华为最佳实践经验                    • 组织文化理解
• 外部视角与专业判断                  • 内部资源调动能力
• 变革工具与模板                      • 执行与落地能力
─────────────────                    ─────────────────
        ↓                                    ↓
              联合项目团队
         （共同设计、共同交付）
```

### 5.2 "授人以渔"的服务原则

薄云咨询的服务设计遵循华为的"授人以渔"原则：

- **不制造依赖**：每个项目都以"客户能独立运营"为成功标准
- **能力转移计划**：每个项目内置客户内部团队的能力建设模块
- **知识资产留存**：所有方法论、工具、模板交付客户，并归属客户所有
- **逐步放手**：早期顾问主导 → 中期顾问辅助 → 后期客户主导

### 5.3 三种顾问介入模式

| 模式 | 适用时机 | 顾问角色 | 时间投入 |
|------|---------|---------|---------|
| **深度陪跑** | 重大变革启动，客户缺乏变革管理经验 | 实质性推动者 | 每周3-5天，持续6-18个月 |
| **顾问教练** | 客户有一定能力，需要方向指导 | 导师+质量把关 | 每周1-2天，持续3-6个月 |
| **专项支持** | 特定阶段遇到困难，需要外部介入 | 问题解决者 | 按需，2-8周 |

## 六、薄云咨询变革管理服务定位

### 6.1 核心价值主张

> **"让变革真正发生，而不只是设计变革"**

薄云咨询的差异化不在于设计更漂亮的方案，而在于帮助客户解决"方案有了但推不动"的问题——这正是中国企业管理变革失败的最主要原因。

### 6.2 服务定位矩阵

```
变革管理深度
    ↑
高  │  ④专业变革        ③华为体系
    │  管理服务          标准化落地
    │  （仅变革管理）     （变革+业务）
    │
低  │  ①入门工作坊      ②就绪度诊断
    │  （认知建设）      +蓝图设计
    │
    └────────────────────────────────→
         业务咨询范围
        窄（变革专项）  宽（端到端）
```

**薄云的甜蜜区**：右上角——将华为体系的业务咨询与专业变革管理深度结合，形成端到端交付能力。

### 6.3 目标客户画像

| 维度 | 理想客户特征 |
|------|------------|
| **规模** | 500-10000人企业，有足够复杂度但又不过于官僚 |
| **阶段** | 规模扩张期或转型期，管理体系明显滞后于业务发展 |
| **意愿** | 高层有明确变革意愿，不只是"听说华为模式好" |
| **行业** | 科技、制造、服务业，已建立基础流程管理意识 |
| **痛点** | IPD/LTC推行受阻；数字化转型落地难；组织能力不足以支撑战略 |

### 6.4 与竞争对手的差异化

| 维度 | 传统大型咨询公司 | 华为系咨询 | **薄云咨询** |
|------|--------------|-----------|------------|
| 方法论 | 国际通用框架 | 华为原版复制 | 华为实践+本地化适配 |
| 变革管理 | 通常作为附加项 | 较少专项投入 | 核心能力，专项深度 |
| 性价比 | 高价，长周期 | 高价，多人天 | 聚焦关键阶段，效率优先 |
| 授人以渔 | 较少主动转移 | 以项目结果为主 | 明确的内部能力建设目标 |

## 七、FE（Functional Excellence）专项服务模块

> 来源：《常变与长青》明确记载的华为FE实践，是薄云变革管理服务的重要专项方向

### 7.1 华为FE的历史背景

**什么是FE（Functional Excellence，职能卓越）**：

1999年华为启动IPD变革时，IBM顾问Lew Kimmel建议并行设立FE项目——专门提升各职能部门支撑IPD运作的能力。这一建议来自IBM自身的变革经验：**[[华为的管理变革]]失败的重要原因往往不是流程本身，而是职能部门的能力跟不上新流程的要求**。

**FE的具体设计**（华为原版）：
- 覆盖范围：研发、销售、技术服务、采购、制造、供应链等主要职能领域
- 项目规模：每个职能领域设8-10个子项目
- 周期设计：每个子项目以2-3个月为周期，快速迭代
- 与IPD的关系：FE是IPD变革的"能力基础设施"，两者并行推进

**为什么FE如此重要**：

> 华为的经验证明：职能部门能力是变革能否落地的关键因素之一。如果HR、财经、IT等职能的能力仍停留在"事务处理"层次，IPD需要的"跨部门协同"、"数据驱动决策"就无从实现。

### 7.2 薄云FE专项服务设计

基于华为FE原版实践，薄云咨询设计了适合中国成长型企业的FE专项服务：

#### FE-A：职能能力诊断（4-6周）

**服务目标**：建立各职能的成熟度基线，识别与业务战略的差距

**服务内容**：
- 各职能FE成熟度评估（参见02_诊断工具/FE成熟度诊断.md）
- 职能能力与业务战略需求的匹配度分析
- 优先改进领域识别（重要性×差距矩阵）
- FE提升路标建议

**主要交付物**：
- FE成熟度评估报告（含各职能雷达图）
- FE改进优先级矩阵
- 12-36个月FE提升路标建议

#### FE-B：职能能力蓝图设计（8-12周）

**服务目标**：为各职能设计清晰的"从当前状态到目标状态"的能力提升路径

**服务内容**（以HRBP体系为例）：
- 目标职能能力定义（参照华为L3-L4水平）
- 职能组织架构重设计（HRBP/CoE/SSC三支柱）
- 关键岗位能力模型设计
- 职能KPI体系重构
- [[华为IT管理]]工具选型建议

**各职能模块**：

| 职能模块 | 主要内容 | 典型周期 |
|---------|---------|---------|
| HR三支柱 | HRBP体系/CoE专家/SSC共享中心设计 | 12-18个月落地 |
| 财经BP | 从事务财务到业务财务的转型路径 | 8-12个月落地 |
| IT使能 | IT治理框架/数据平台/业务赋能机制 | 12-24个月落地 |
| 采购战略化 | 战略采购/供应商分层管理/采购数字化 | 6-12个月落地 |

#### FE-C：职能能力提升陪跑（3-12个月）

**服务目标**：陪伴客户完成FE改进措施的落地实施

**三种陪跑模式**：

| 模式 | 描述 | 适用场景 | 顾问投入 |
|------|------|---------|---------|
| **全程陪跑** | 顾问全程主导或深度参与 | 职能能力极弱（L1-L2），需要手把手 | 每周3-4天 |
| **关键节点陪跑** | 在里程碑节点介入，日常由客户推进 | 职能有基础（L2-L3），主要需要方向把控 | 每月3-5天 |
| **教练辅导** | 定期辅导FE内部推进者（HRBP/BP等） | 职能人员有意愿但缺方法（L2+） | 每月1-2次 |

### 7.3 FE与整体变革管理的关系

在薄云的综合方案设计中，FE专项服务通常与业务变革（IPD/[[LTC]]等）结合设计：

```
                 业务变革（如IPD）
                     ↓ 需要职能能力支撑
    ┌────────────┬──────────────┬────────────┐
    │  HR FE     │  财经 FE     │  IT FE     │
    │ （组织能力）│ （决策支持）  │ （系统使能）│
    └────────────┴──────────────┴────────────┘
                     ↑
              FE是变革的"地基"
     如果地基不稳，业务变革大厦就摇摇欲坠
```

**最佳实践建议**：
1. 业务变革启动前，先做FE现状诊断，了解各职能"地基"状况
2. 根据业务变革对各职能的依赖程度，确定FE提升的优先序
3. FE提升与业务变革并行推进，不要先后、以免"等"出问题
4. 每个FE项目周期控制在2-3个月，小步快跑，持续见效
