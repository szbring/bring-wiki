# BTIT 业务变革与信息技术

## 定义
华为把"业务变革（BT）+ IT 治理"整合为一条端到端流程，由 BTC（Business Transformation Council）牵头，使 IT 投资始终服务于业务变革需求，避免技术驱动失控。

## 输入 / 输出
- **输入**：DSTE 战略与必赢战役、各业务流程 Owner 的能力差距诉求、外部技术趋势
- **输出**：变革项目立项与交付、IT 系统能力提升、流程数字化方案、组织能力沉淀
- **核心子模块**：[[id:huawei.btit.concept.business-transformation]] BT 体系 / [[id:huawei.btit.concept.it-governance]] IT 治理 / [[id:huawei.btit.concept.digital-transformation]] 数字化转型 / [[id:huawei.btit.concept.enterprise-architecture]] EA 企业架构

## 角色
- **决策方**：BTC（变革指导委员会）+ EMT
- **执行方**：变革项目 PMO / IT 部门
- **业务方**：各业务流 Owner（提需求 + 共担变革目标）
- **方法论团队**：参考 [[id:huawei.btit.concept.change-methodology]] 提供变革方法支持

## 上下游
- **上游**：[[id:huawei.dste.concept.dste-framework]] 战略与年度计划提出变革诉求
- **下游**：变革项目交付 → 流程能力提升 → 反哺 IPD/LTC/HR 等业务流
- **平行**：HR（组织调整）、Finances（变革预算）、各业务流 Owner（共建）

## 典型误区
- **变革当 IT 项目做**：上系统不改流程不动组织——投资浪费，业务不接
- **业务变革与 IT 割裂**：BT 团队和 IT 团队各做各的——最后系统不支撑变革目标
- **没有 BTC 治理**：变革项目独立立项，没人统筹优先级与资源——多个项目互相打架
- **数字化 = 上 AI**：把数字化窄化为 AI 应用——忽视流程数字化、数据治理等基础设施
