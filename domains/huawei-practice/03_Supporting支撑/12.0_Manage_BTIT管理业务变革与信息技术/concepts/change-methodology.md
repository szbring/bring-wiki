# 华为变革管理方法论

## 定义
华为吸收 Kotter 八步法、ADKAR、IBM 咨询方法等外部经验后形成的本土化变革方法论，强调"创建紧迫感 → 建立强大联盟 → 设计愿景与战略 → 沟通愿景 → 授权行动 → 创造短期胜利 → 巩固成果 → 嵌入文化"八阶段。

## 输入 / 输出
- **输入**：变革诉求 + 现状差距诊断 + 利益相关方 stakeholder map
- **输出**：变革路线图 + 沟通方案 + 试点设计 + 短期胜利清单 + 嵌入机制
- **核心工具**：变革就绪度评估 / 利益相关方分析矩阵 / ADKAR 个体变化模型 / Kotter 八步

## 角色
- **方法论 Owner**：变革管理团队 / 流程 Owner
- **应用者**：BTC、变革项目 PMO、各业务流 Owner
- **培训对象**：变革代理 / 中层干部

## 上下游
- **上游**：[[id:huawei.btit.concept.business-transformation]] 项目立项后选择适用方法
- **下游**：方法论指导项目方案设计与执行；项目复盘后回炉迭代方法论

## 典型误区
- **照搬 Kotter 八步**：不结合华为权力结构与文化特点——"沟通愿景"在华为更需要"高层亲自宣讲 + 反复"
- **只用方法不看现状**：变革就绪度未评估就启动，遇到强阻力才发现晚了
- **重设计轻执行**：方案漂亮但没有变革代理网络，落地无人推
- **不庆祝短期胜利**：项目持续 2-3 年，缺中间里程碑庆祝——团队疲惫流失
