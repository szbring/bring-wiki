## 三、T-PM-02：利益关系人分析矩阵

### 使用步骤

1. 列出所有受变革影响的关键角色/群体
2. 评估每人的"影响力"（对变革成功的影响程度）和"支持度"（当前对变革的支持程度）
3. 确定管理策略

### 利益关系人矩阵

| 干系人姓名/群体 | 职位/角色 | 影响力（高/中/低） | 当前支持度（+/-/0） | 期望支持度 | 关键关切 | 管理策略 | 负责人 | 状态 |
|-------------|---------|---------------|----------------|---------|---------|---------|-------|------|
| | | | | | | | | |

### 影响力×支持度四象限与策略

```
     高支持度
         │
 高影响力  │  高影响力
 高支持度  │  高支持度（核心赞助人）
 ──────── │ ──────────────────────→ 高影响力
 低影响力  │  高影响力
 低支持度  │  低支持度（关键风险点）
         │
     低支持度
```

| 象限 | 特征 | 管理策略 |
|------|------|---------|
| 高影响力+高支持度 | **核心盟友** | 深度绑定，让其发声，保持热情 |
| 高影响力+低支持度 | **关键风险** | 一对一深度沟通，理解顾虑，逐步转化 |
| 低影响力+高支持度 | **种子用户** | 培养Champion，让其感染周围 |
| 低影响力+低支持度 | **边缘观望** | 广播式沟通覆盖即可，监控是否升级 |

## 四、T-PM-03：变革管理沟通计划

### 沟通矩阵

| 沟通内容 | 目标受众 | 核心信息 | 渠道/形式 | 频次 | 责任人 | 时间节点 |
|---------|---------|---------|---------|------|-------|---------|
| **变革启动公告** | 全员 | 为什么变、变什么、对我的影响 | 全员会议+邮件 | 一次性 | 赞助人 | 项目立项后1周内 |
| **高管对齐会** | 各部门VP | 变革范围、影响、支持要求 | 专题会议 | 月度 | 项目经理 | 项目启动第1周 |
| **中层管理者宣贯** | 中层 | 中层角色要求、支持方式 | 工作坊 | 季度 | 变革经理 | 按阶段 |
| **项目进展通报** | 所有干系人 | 进展、里程碑达成、下一步 | 简报/邮件 | 月度 | 项目经理 | 每月固定日期 |
| **试点经验分享** | 推广目标部门 | 试点成果、经验教训 | 分享会 | 试点结束后 | 试点Champion | 推行前1周 |
| **固化宣贯** | 全员 | 新流程已固化、操作要求 | 培训+公告 | 固化节点 | 培训经理 | 固化阶段启动时 |
| [补充] | | | | | | |

### 关键沟通内容：三件事原则

每次重要沟通必须回答清楚：
1. **WHY**：为什么要做这件事？（Burning Platform + Vision）
2. **WHAT**：具体会发生什么变化？（对受众最相关的内容）
3. **WIIFM**：对你有什么好处？（What's In It For Me）

## 六、T-PM-05：变革风险管理清单

### 常见变革风险清单（通用版）

| 风险类别 | 具体风险 | 早期信号 | 推荐应对措施 |
|---------|---------|---------|------------|
| **领导力风险** | 赞助人支持减弱 | 赞助人不出席关键会议；回复变慢；公开场合不提项目 | 立即与赞助人1对1沟通，重建对齐 |
| **领导力风险** | 高管层出现分歧 | 高管在公开场合态度矛盾；听说"某VP不支持" | 安排专场高管对齐会，不能拖 |
| **人员风险** | 中层选择性执行 | "嘴上说好，执行没动作"；下属反馈与上级表态不一致 | 激活中层Champion，绩效挂钩 |
| **人员风险** | 关键人员在变革期离职 | 异常低落情绪；频繁请假；接到外部公司电话 | 提前做人才保留识别，变革期一对一关怀 |
| **范围风险** | 范围蔓延 | 不断有新需求加入；"顺便也做这个" | 严格执行Charter变更控制 |
| **进度风险** | 试点无法按时完成 | 试点部门配合度低；资源被业务抽调 | 提前锁定试点资源，赞助人保障 |
| **质量风险** | 流程设计脱离业务实际 | 业务代表反馈"这个不实际"；流程走不通 | 增加业务深度参与，做流程穿行测试 |
| **文化风险** | 旧习惯反弹 | 固化阶段发现员工又回到旧做法 | 定期固化检查，绩效考核持续挂钩 |
| **IT风险** | 系统上线延迟 | 开发进度持续落后；bug率高 | 提前设置甘特图缓冲，制定降级方案 |
| **外部风险** | 市场变化导致优先级下调 | 高层突然关注其他紧急事务；预算被冻结 | 定期重新对齐业务战略，保持变革价值可见性 |

### 风险等级定义

| 等级 | 定义 | 响应时间 |
|------|------|---------|
| 🔴 高 | 可能导致项目失败或重大延期 | 24小时内上报赞助人 |
| 🟡 中 | 可能影响里程碑或质量 | 本周内提出应对方案 |
| 🟢 低 | 可管控，不影响整体 | 跟踪监控即可 |

## 七、T-PM-06：变革Champion激活手册

### 什么是Champion

**Champion**（变革推动者）是在基层推动变革的"种子"网络。他们不是项目团队成员，而是在各部门内部自愿或被选定的变革支持者。

### Champion网络设计

**Champion选择标准**：
- [ ] 在所在团队有非正式影响力（同事信任）
- [ ] 对变革内容有基本理解和认同
- [ ] 愿意花时间支持变革（每周2-4小时）
- [ ] 不一定是最高绩效者，但一定是文化影响者

**Champion激活三步骤**：

**Step 1：Champion遴选与招募**
```
遴选方式：□ 上级推荐  □ 自愿报名  □ 问卷识别
目标比例：受影响员工每15-20人配1名Champion
招募时间：试点阶段前4-6周
```

**Step 2：Champion培训与赋能**
```
培训内容：
• 变革背景与目标（"为什么"）
• 新流程/新工具的使用（"怎么做"）
• Champion角色与行为要求（"我的任务"）
• 常见疑问Q&A（"如何回应同事"）

培训形式：集中工作坊（0.5-1天）+ 后续定期沟通会
```

**Step 3：Champion持续激活**
```
定期活动：
• 双周Champion例会（15-30分钟，项目进展+问题收集）
• 月度Champion共享会（经验分享，互相支持）
• 里程碑节点Champion表彰（让英雄被看见）

信息优先：Champion先于其他员工获知变革进展（"信息特权"激励）
```

### Champion工作内容清单

| 工作类型 | 具体行为 | 时间投入/周 |
|---------|---------|-----------|
| 日常推广 | 在团队内介绍和推广新流程/工具 | 1-2小时 |
| 问题收集 | 收集同事疑虑和问题，反馈给项目团队 | 30分钟 |
| 示范引领 | 率先使用新系统/流程，做榜样 | 自然行为 |
| 参加例会 | 参加Champion双周例会 | 30分钟 |
| 激励同伴 | 在团队内传播"正能量"，维持变革士气 | 自然行为 |
