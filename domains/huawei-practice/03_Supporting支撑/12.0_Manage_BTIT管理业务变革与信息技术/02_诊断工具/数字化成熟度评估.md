# 数字化成熟度评估

<!-- chunk: 数字化成熟度评估.五级成熟度 -->
## 五级成熟度

| 级别 | 特征 |
|---|---|
| L1 | 主要靠Excel/邮件，数据靠手工汇总，信息孤岛 |
| L2 | 有ERP/CRM等单点系统，但相互不联通，数据重复录入 |
| L3 | 核心系统集成，数据可流动，有统一数据源 |
| L4 | BI看板驱动决策，实时数据，预测分析能力 |
| L5 | AI辅助决策，智能自动化，数字孪生 |

---

<!-- chunk: 数字化成熟度评估.快速诊断问题 -->
## 快速诊断问题

1. **「核心业务数据（订单/研发进度/财务）靠什么系统管理？集成了吗？」**
   - Excel+邮件为主 → L1
   - 有系统但不集成 → L2
   - 系统集成，数据流通 → L3+

2. **「管理层需要数据时，多长时间能拿到？」**
   - 让人手工汇总，1-3天 → L1-L2
   - 有报表但不实时 → L2
   - 实时看板随时查 → L3+

3. **「[[华为IT管理]]建设最大的痛点是什么？」**
   - 没有系统 → 从基础开始
   - 有系统没人用 → 变革管理缺失
   - 数据质量差 → 数据治理问题
   - 系统不集成 → 架构设计问题

4. **「近三年在IT/数字化上的投入是多少？ROI怎样？」**
   - 投了很多但效果差 → 缺乏数字化规划，盲目上系统
   - 投入少/没什么效果 → L1-L2，待升级
