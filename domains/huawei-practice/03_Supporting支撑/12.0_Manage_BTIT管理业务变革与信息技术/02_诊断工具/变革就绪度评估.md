<!-- chunk: 变革就绪度评估.一-变革就绪度六维评估模型 -->
## 一、变革就绪度六维评估模型

### 模型概述

变革就绪度（Change Readiness）是指组织在启动一项重大变革时，各方面条件的准备程度。薄云咨询的六维评估模型从战略、领导力、文化、能力、流程、员工六个维度综合评估。

```
              战略就绪度
             （方向清晰度）
                  │
  员工就绪度 ─────┼───── 领导力就绪度
 （意愿与能力）   │      （赞助与推动）
                  │
  流程就绪度 ─────┼───── 文化就绪度
 （机制与体系）   │      （氛围与价值观）
                  │
              能力就绪度
             （人员与技术）
```

### 六个维度详解

| 维度 | 定义 | 评估焦点 | 低就绪信号 |
|------|------|---------|-----------|
| **战略就绪度** | 变革方向是否与公司战略高度对齐，目标是否清晰 | 目标清晰度、战略一致性、优先级明确度 | 高管层对变革目标理解不一致 |
| **领导力就绪度** | 各级领导者是否有意愿和能力推动变革 | 赞助人可见度、中层管理者支持度、领导行为示范 | 赞助人不公开表态支持 |
| **文化就绪度** | 组织文化是否支持变革所需的新行为 | 对失败的容忍度、跨部门协作氛围、创新开放度 | "我们一直是这么做的"心态盛行 |
| **能力就绪度** | 员工是否具备或能够获取变革所需技能 | 技能缺口分析、培训体系完备性、学习文化 | 关键岗位技能严重欠缺 |
| **流程就绪度** | 现有流程和机制是否支持或阻碍变革 | 流程复杂度、系统适配性、制度障碍识别 | 旧制度与新要求存在根本冲突 |
| **员工就绪度** | 受影响员工是否了解并接受即将到来的变化 | 知晓度、认同度、参与意愿 | 员工对变革知之甚少，焦虑情绪弥漫 |

## 二、快速诊断问卷（20题，5分钟版）

### 使用说明

- **适用对象**：变革项目发起人、HRBP、变革经理
- **评分方式**：每题1-5分（1=完全不同意，5=完全同意）
- **建议填写人**：邀请3-5名高管和5-10名中层管理者分别填写，取平均值
- **时间预估**：约5分钟

---

### 维度一：战略就绪度（4题）

**Q1**：公司高管层对"为什么需要这次变革"有清晰且一致的认识。
> 评分：1 / 2 / 3 / 4 / 5

**Q2**：这次变革与公司的年度战略优先级高度对齐。
> 评分：1 / 2 / 3 / 4 / 5

**Q3**：变革完成后的目标状态（愿景）已经清晰定义并可以描述出来。
> 评分：1 / 2 / 3 / 4 / 5

**Q4**：公司已经将资源（人力、预算）优先分配给这次变革。
> 评分：1 / 2 / 3 / 4 / 5

---

### 维度二：领导力就绪度（4题）

**Q5**：有一位高层赞助人公开表态支持这次变革，且愿意投入时间。
> 评分：1 / 2 / 3 / 4 / 5

**Q6**：中层管理者（受变革影响最大的部门主管）总体上支持这次变革。
> 评分：1 / 2 / 3 / 4 / 5

**Q7**：高层领导已经通过自身行为示范了变革所倡导的新方式。
> 评分：1 / 2 / 3 / 4 / 5

**Q8**：当变革遇到跨部门阻力时，有明确的升级路径和决策机制。
> 评分：1 / 2 / 3 / 4 / 5

---

### 维度三：文化就绪度（3题）

**Q9**：公司文化鼓励尝试新方法，对失败有一定容忍度。
> 评分：1 / 2 / 3 / 4 / 5

**Q10**：员工愿意坦诚反馈问题和担忧，而不是选择沉默。
> 评分：1 / 2 / 3 / 4 / 5

**Q11**：不同部门之间愿意为共同目标放弃局部利益，协同行动。
> 评分：1 / 2 / 3 / 4 / 5

---

### 维度四：能力就绪度（3题）

**Q12**：受影响员工具备推行变革所需的技能（或有明确的培训计划）。
> 评分：1 / 2 / 3 / 4 / 5

**Q13**：有足够的内部或外部资源（如顾问、工具）支持技能发展。
> 评分：1 / 2 / 3 / 4 / 5

**Q14**：组织具有良好的学习文化，员工愿意接受新知识和技能培训。
> 评分：1 / 2 / 3 / 4 / 5

---

### 维度五：流程就绪度（3题）

**Q15**：现有的关键流程和系统支持（而非阻碍）变革后的新工作方式。
> 评分：1 / 2 / 3 / 4 / 5

**Q16**：已经识别出流程/制度障碍，并有计划在变革过程中修订。
> 评分：1 / 2 / 3 / 4 / 5

**Q17**：绩效考核和激励机制与变革目标一致（不存在反向激励问题）。
> 评分：1 / 2 / 3 / 4 / 5

---

### 维度六：员工就绪度（3题）

**Q18**：受影响员工已经被告知这次变革将如何影响他们的工作。
> 评分：1 / 2 / 3 / 4 / 5

**Q19**：大多数受影响员工认为这次变革对公司有益。
> 评分：1 / 2 / 3 / 4 / 5

**Q20**：员工有渠道提出问题、表达担忧，并得到及时响应。
> 评分：1 / 2 / 3 / 4 / 5

---

<!-- chunk: 变革就绪度评估.三-评分解读与行动建议 -->
## 三、评分解读与行动建议

### 单维度评分解读

| 维度得分 | 就绪等级 | 解读 | 建议行动 |
|---------|---------|------|---------|
| 17-20分 | 🟢 高就绪 | 该维度基础扎实，可作为变革推进的优势 | 维持现状，注重监控 |
| 12-16分 | 🟡 中等就绪 | 有一定基础但存在明显短板，需要针对性补强 | 制定专项行动计划，2-4周内启动 |
| 8-11分 | 🟠 低就绪 | 存在重大障碍，若不解决将影响变革成功率 | 必须在变革启动前重点改善 |
| 4-7分 | 🔴 严重不就绪 | 该维度是变革推进的重大风险，需要即刻干预 | 考虑推迟变革启动，优先解决根本问题 |

### 总体评分解读（满分100分）

| 总分 | 综合就绪度 | 建议决策 |
|------|-----------|---------|
| 85-100分 | 高度就绪 | 可以按计划推进变革，同时监控低分维度 |
| 70-84分 | 基本就绪 | 制定1-2个关键维度的补强计划，再推进变革 |
| 55-69分 | 部分就绪 | 建议推迟4-8周，重点提升低分维度后再启动 |
| <55分 | 尚未就绪 | 变革条件不成熟，建议先解决根本性障碍 |

### 典型场景与行动建议

**场景1：战略清晰但领导力不足（Q1-4高分，Q5-8低分）**
- 症状：大家都知道要变，但没有人真正带头
- 行动：①聘请外部顾问建立赞助人Coach机制 ②设计高管可见度活动（全员信、启动会主讲等） ③建立赞助人月度进展汇报机制

**场景2：领导力强但员工阻力大（Q5-8高分，Q18-20低分）**
- 症状：高层推得很猛，但一线员工不理解、不配合
- 行动：①开展员工沟通路演（下沉到一线） ②建立FAQ机制，及时响应疑问 ③发展Champion大使，覆盖基层

**场景3：文化和能力双低（Q9-14低分）**
- 症状：员工不敢变、不会变
- 行动：①启动文化氛围专项（心理安全感建设） ②详细技能缺口分析，定制培训方案 ③从小成功开始，建立"变革有收益"的信心

## 四、变革阻力识别工具

### 阻力雷达（快速识别高风险群体）

在访谈或问卷后，用下表快速定位最需要关注的群体：

| 群体/角色 | 受影响程度 | 当前支持度 | 影响力 | 优先关注级别 |
|----------|-----------|-----------|--------|------------|
| CEO/高管委 | 低 | 高 | 极高 | 维持关系 |
| 变革涉及部门的VP | 高 | ？ | 高 | **重点关注** |
| 中层管理者（主管级） | 极高 | ？ | 中高 | **重点关注** |
| IT/系统团队 | 高 | ？ | 中 | 需要确认 |
| 一线员工（直接受影响） | 极高 | ？ | 低 | 需要告知 |
| HR/培训部门 | 中 | 通常较高 | 中 | 合作伙伴 |
| 外部客户 | 低-中 | — | 中 | 监控风险 |

### 阻力深度访谈问题指引

用以下问题深度了解阻力根源（建议1对1访谈，30分钟，保密承诺）：

1. "你对这次变革的总体看法是什么？"（开放式，不引导）
2. "你认为这次变革对你的团队最大的影响是什么？"
3. "你认为这次变革推进的最大障碍在哪里？"
4. "有什么事情，如果做到了，你会更愿意支持这次变革？"
5. "你见过类似的变革项目，成功或失败的，你觉得原因是什么？"

> 💡 **访谈技巧**：将前两题的回答与变革的实际影响进行对比，可以快速识别信息不对称（不了解型阻力）vs 价值观冲突（利益/情感型阻力）。

## 五、利益关系人4象限分析（意愿×能力）

> 补充来源：《常变与长青》郭平著，2024年——变革管理"船模型"利益关系人管理要素

### 5.1 4象限模型

利益关系人管理是华为变革管理"船模型"8要素之一。用"意愿"（是否愿意支持变革）和"能力"（是否具备执行变革的能力）两个维度构建4象限：

```
高意愿
  │          │
  │ 跟随者   │ 盟友/大使
  │ （高意愿 │ （高意愿
  │  低能力）│  高能力）
  │──────────┼──────────
  │ 旁观者   │ 潜在阻力
  │ （低意愿 │ （低意愿
  │  低能力）│  高能力）
  │          │
低意愿─────────────────── 高能力
        低能力
```

### 5.2 四类利益关系人的特征与管理策略

| 象限 | 类型 | 特征描述 | 核心风险 | 管理策略 |
|------|------|---------|---------|---------|
| **高意愿×高能力** | 盟友/变革大使 | 既支持变革，又有能力推动变革落地 | 过度依赖，成为单点故障 | ①培养为Champion；②赋予正式角色；③公开认可其贡献；④防止流失 |
| **高意愿×低能力** | 跟随者 | 态度积极但技能不足，想帮但帮不上 | 行动力弱，影响执行质量 | ①重点培训和辅导；②配对能力强的盟友；③给小成功建立信心 |
| **低意愿×高能力** | 潜在阻力 | 能力强，但不认同或有私心，可能产生强烈阻力 | 利用影响力传播负面情绪 | ①一对一深度沟通；②理解阻力根因；③利益对齐（找到共同利益）；④纳入决策圈参与设计 |
| **低意愿×低能力** | 旁观者 | 既不支持也没能力，处于观望状态 | 沉默的多数，可能被负面声音带走 | ①基础沟通，确保信息知晓；②防止旁观者被阻力方拉拢；③用榜样效应带动 |

### 5.3 动态管理：象限会变化

利益关系人的象限不是固定的，随变革推进会发生变化：
- 沟通和培训可以将"跟随者"升级为"盟友"
- 利益对齐可以将"潜在阻力"转化为"盟友"
- 如果变革初期成果可见，"旁观者"会逐步转移到上方象限

**建议**：每季度更新一次利益关系人分析，跟踪象限变化。

## 六、变革准备度评估维度

> 补充来源：《常变与长青》郭平著，2024年——变革规划三阶段

在变革规划阶段，除六维就绪度模型外，还需从以下维度评估变革准备度：

### 6.1 业务战略对齐度
- 本次变革是否来自明确的业务战略需求？
- 高管层是否对"为什么现在要变"有共识？
- 是否存在"Burning Platform"（不变的代价）?

### 6.2 变革愿景清晰度
- 是否能清晰描述变革后的目标状态（彼岸）？
- 愿景是否被翻译成了可操作的具体改变？
- 受影响员工是否能理解变革后对自己意味着什么？

### 6.3 变革规划质量
- 是否完成了As-Is（现状）与To-Be（目标）的分析？
- 是否有清晰的变革路标和阶段划分？
- 变革范围是否合理（不过大也不过小）？

### 6.4 资源保障充足度
- 是否有专职的变革项目团队（不全靠兼职）？
- 预算是否经过ESC或相当层级的审批？
- 外部顾问资源是否到位？

### 6.5 试点条件成熟度
- 是否识别了合适的试点单位（有代表性且愿意配合）？
- 试点范围是否足够小（可控）又足够有代表性（可推广）？
- 试点评估标准是否提前定义清楚？

## 七、变革阻力识别清单（基于华为"七个反对"）

> 补充来源：《常变与长青》郭平著，2024年——2006年华为提出的七个反对

华为2006年总结的"七个反对"不仅是变革原则，也是识别变革阻力来源的重要工具。用以下清单检查变革项目是否存在这些风险：

| 阻力类型 | 识别问题 | 症状表现 | 应对措施 |
|---------|---------|---------|---------|
| **完美主义** | 是否在等"方案完美了"再推行？ | 方案反复修改，迟迟不试点 | 设定"足够好"标准；试点中优化比等待完美更有价值 |
| **繁琐哲学** | 新流程是否比旧流程更复杂？ | 员工抱怨"比以前更麻烦了" | 定期做流程瘦身，合并冗余步骤 |
| **盲目创新** | 是否为了创新而创新？ | 新设计与业务价值脱节 | 每项创新必须能说明对业务的贡献 |
| **局部优化** | 局部优化是否损害了全局效益？ | A部门效率提升，B部门效率下降 | 以端到端流程视角评估变革影响 |
| **无全局观干部主导** | 主导变革的人是否有跨部门视野？ | 变革方案明显偏向某一部门利益 | 确保变革领导组有多部门代表 |
| **无业务经验者参与** | 变革团队是否真正懂业务？ | 方案脱离实际，一线无法执行 | 必须有一线业务经验者参与设计 |
| **未验证直接推广** | 是否跳过试点直接全面推广？ | "只要方案好，不需要试点" | 任何变革必须先试点再推广 |

### 快速自检（变革启动前）

在变革项目启动前，用以下7个问题自检：

- [ ] 我们是否已经定义了"足够好"就可以推进的标准？（防完美主义）
- [ ] 新流程的步骤是否比现有流程更少或持平？（防繁琐）
- [ ] 每项设计创新是否都能说明业务价值？（防盲目创新）
- [ ] 我们是否评估了本次变革对上下游的影响？（防局部优化）
- [ ] 变革领导团队中是否包含受影响最大部门的代表？（防无全局观）
- [ ] 项目团队中是否有至少一名有实操经验的业务专家？（防无经验者）
- [ ] 试点计划是否已经纳入变革路标？（防未验证推广）
