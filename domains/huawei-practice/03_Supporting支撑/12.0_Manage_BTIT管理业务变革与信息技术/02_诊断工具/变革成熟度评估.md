<!-- chunk: 变革成熟度评估.一-变革成熟度-vs-变革就绪度 -->
## 一、变革成熟度 vs 变革就绪度

**变革就绪度**（Change Readiness）：评估"**当前这个变革项目**是否具备成功条件"，是项目维度的评估。

**变革成熟度**（Change Maturity）：评估"**企业整体**推动变革的能力水平"，是组织能力维度的评估。

> 类比：就绪度是"这次开车能否到达目的地"，成熟度是"这位司机的驾驶水平有多高"。

## 二、变革成熟度5级模型

参照华为BT体系和国际CMM标准，薄云咨询将企业变革成熟度划分为5个等级：

```
L5 ■■■■■  引领级  ——  变革是企业的核心竞争力，组织自我更新能力强
L4 ■■■■   优化级  ——  变革体系完整，能持续优化和学习
L3 ■■■    管控级  ——  有规范的变革管理流程，可预期完成目标
L2 ■■     初始级  ——  能完成个别变革，但依赖个人英雄
L1 ■      混沌级  ——  变革以失败为常态，缺乏方法论
```

### L1：混沌级（Ad Hoc）

**特征**：
- 变革项目靠"运动式"推动，完全依赖一把手个人意志
- 没有标准的变革管理方法论，每次都从零开始
- 变革失败率极高，员工对"变革"这个词产生免疫反应
- 前一个变革还没结束，下一个变革又开始了

**典型症状**：
- 变革项目无法量化进展，"永远在推进中"
- 变革启动时声势浩大，实施过程迅速平息
- 没有赞助人体系，一把手亲自抓，但精力有限

**中国企业现状**：大多数成立10年以内、规模300人以下的企业处于L1。

---

### L2：初始级（Repeatable）

**特征**：
- 某些变革项目能成功，但成功的原因是"碰到了对的人"
- 开始使用项目管理工具（甘特图、里程碑），但没有变革管理专项
- 成功项目的经验无法复制，没有文档化和知识传承
- 中层管理者开始有参与变革的经验，但技能不系统

**典型症状**：
- 某个变革由某位强势部门负责人推动成功，但换人就失败
- 项目管理成熟，但"变革管理"（人的部分）严重缺失
- 高管认为"变革就是换个流程系统"，忽略人员管理

**中国企业现状**：大多数500-2000人、有一定管理基础的成长型企业处于L2。

---

### L3：管控级（Defined）

**特征**：
- 有明确的变革管理框架（如船模型），项目团队会使用
- 建立了赞助人体系，关键变革都有高层背书
- 利益关系人管理和沟通培训成为标准动作
- 有变革管理专职团队（BT团队或PMO中有CM专员）

**典型症状**：
- 变革成功率显著提升（≥60%的项目能达到主要目标）
- 但每个项目仍需大量精力从头设计变革管理方案
- 跨项目的协调和知识复用仍然不足

**华为的L3阶段**：约2003-2008年，[[IPD]]成功后开始系统化推广变革管理框架。

---

### L4：优化级（Managed）

**特征**：
- 有企业级的变革管理体系（ESC + BT架构），覆盖所有重大变革
- 变革项目有统一的度量体系（如TAM模型），数据驱动决策
- 变革管理知识得到持续积累和更新，有正式的知识库
- 高管层将"变革领导力"作为干部发展的核心能力

**典型症状**：
- 企业能同时高质量推进3-5个重大变革项目
- 变革失败经验被系统化总结，不会重复犯同类错误
- 组织对外部冲击（如疫情、政策变化）有快速响应能力

**华为的L4阶段**：约2010-2016年，形成完整的BT体系，ESC统一管理。

---

### L5：引领级（Innovating）

**特征**：
- 变革能力成为企业差异化竞争优势
- 变革不是响应外部压力，而是主动引领行业变化
- 组织拥有"持续自我更新"的文化基因，员工主动拥抱变化
- 变革管理实践向行业输出，成为行业标杆

**华为的L5阶段**：约2017年至今，华为将变革能力向产业输出（通过薄云这类合作机构推广华为方法论即是体现）。

## 三、变革成熟度评估问卷（40题完整版）

### 使用说明

- **适用对象**：希望全面了解企业变革能力的管理团队
- **建议评估人**：CEO + 2-3名VP + BT/PMO负责人，分别独立评分后取均值
- **评分方式**：每题1-5分（1=完全不符合，5=完全符合）
- **时间预估**：约20-30分钟
- **结果解读**：各维度均分 × 对照下方成熟度矩阵

---

### 维度一：变革领导力（8题）

| 题号 | 评估内容 | 得分（1-5） |
|------|---------|-----------|
| L1-1 | 每次重大变革都有足够级别的高层领导担任赞助人，且全程可见 | |
| L1-2 | 赞助人会主动解决变革中的资源冲突，而不只是出席启动会 | |
| L1-3 | 中层管理者是变革的主动推动者，而非被动执行者 | |
| L1-4 | 高管团队对变革方向高度一致，不存在明显的"高层分歧" | |
| L1-5 | 变革项目遇到阻力时，领导层不会因为短期压力而退缩 | |
| L1-6 | 公司有正式机制（如ESC）统一管理和优先排序变革项目 | |
| L1-7 | 变革领导力是干部选拔和晋升的重要考量维度 | |
| L1-8 | 高层领导在变革中以身作则，率先改变自身行为 | |

**维度一小计：___/40**

---

### 维度二：变革规划能力（6题）

| 题号 | 评估内容 | 得分（1-5） |
|------|---------|-----------|
| L2-1 | 变革项目启动前会编写高质量的Charter（包含问题陈述、目标、范围、资源） | |
| L2-2 | 变革规划从业务战略出发，能清晰说明变革如何支撑战略目标 | |
| L2-3 | 有明确的变革路标（12-36个月），并定期检视和更新 | |
| L2-4 | 能区分"变革项目"（重构）和"优化项目"（改善），并合理排序 | |
| L2-5 | 变革投资有清晰的ROI预测，并在项目结束时复盘验证 | |
| L2-6 | 变革规划阶段会进行系统的利益关系人识别和影响分析 | |

**维度二小计：___/30**

---

### 维度三：变革管理执行（10题）

| 题号 | 评估内容 | 得分（1-5） |
|------|---------|-----------|
| L3-1 | 重大变革有专职的变革管理团队（而非完全兼职） | |
| L3-2 | 有标准的变革沟通计划，定期向所有相关人员传递进展 | |
| L3-3 | 变革中的培训和发展计划与变革推进节奏同步 | |
| L3-4 | 有系统的利益关系人管理机制，跟踪关键人员的支持度变化 | |
| L3-5 | 新流程/新系统推行时，同步调整了对应的组织架构和岗位职责 | |
| L3-6 | 变革期间的绩效考核体系支持新行为，不存在反向激励 | |
| L3-7 | 有正式的变革阻力识别和处理机制（不只是靠感觉处理） | |
| L3-8 | 变革的文化影响（需要的新文化行为）被明确定义和管理 | |
| L3-9 | 变革项目有明确的试点→推广→固化三阶段执行路径 | |
| L3-10 | "先僵化后优化再固化"的原则在项目中被实际遵守 | |

**维度三小计：___/50**

---

### 维度四：度量与评估（6题）

| 题号 | 评估内容 | 得分（1-5） |
|------|---------|-----------|
| L4-1 | 变革项目有量化的成功指标，且定期监控 | |
| L4-2 | 评估覆盖三个层面：结果（业务KPI）、能力（新行为采纳率）、体系（流程固化度） | |
| L4-3 | 变革成功/失败的经验会被正式总结，形成可复用的知识 | |
| L4-4 | 有变革成熟度的定期评估机制（如每年一次），而非只在项目中评估 | |
| L4-5 | 变革管理工具和模板持续根据项目实践更新迭代 | |
| L4-6 | 变革项目的投入产出数据有系统记录，可用于未来项目的估算参考 | |

**维度四小计：___/30**

---

### 维度五：组织文化与氛围（5题）

| 题号 | 评估内容 | 得分（1-5） |
|------|---------|-----------|
| L5-1 | 员工对变革的整体态度是积极开放的（而非疲惫或抵触） | |
| L5-2 | 公司鼓励员工提出问题和建议，而非只是执行 | |
| L5-3 | 失败的变革尝试会被客观总结，而非追责导致没人敢尝试 | |
| L5-4 | 变革成功的故事和英雄人物会被积极宣扬 | |
| L5-5 | 员工理解"常变与长青"的关系，不认为变革是威胁 | |

**维度五小计：___/25**

---

### 维度六：变革人才体系（5题）

| 题号 | 评估内容 | 得分（1-5） |
|------|---------|-----------|
| L6-1 | 有专职的变革管理人员（BT团队/PMO中的变革经理） | |
| L6-2 | 中层管理者具备基本的变革管理技能（培训或实践获得） | |
| L6-3 | 有"变革Champion"网络，能在基层推动变革 | |
| L6-4 | 变革管理能力被纳入干部能力模型 | |
| L6-5 | 能从外部获取优质的变革管理知识和标杆（如顾问、培训、参访） | |

**维度六小计：___/25**

|------|------|------|------|
| 维度一：变革领导力 | 40 | | |
| 维度二：变革规划能力 | 30 | | |

| 维度四：度量与评估 | 30 | | |

| 总分区间 | 成熟度等级 | 建议 |
|---------|----------|------|
| 0-60分 | L1：混沌级 | 建立基础变革管理意识，引入外部顾问辅导 |
| 61-100分 | L2：初始级 | 建立变革管理方法论框架，培养内部变革骨干 |
| 101-140分 | L3：管控级 | 建立企业级变革管理体系，完善度量机制 |
| 141-170分 | L4：优化级 | 优化ESC治理，强化知识传承和人才体系 |
| 171-200分 | L5：引领级 | 向行业输出变革能力，构建变革生态 |

## 四、快速诊断版（10题，5分钟）

适用于初步摸底或高管层快速自评：

| 题号 | 快速问题 | 得分（1-5） |
|------|---------|-----------|
| Q1 | 公司的变革项目有多大比例能达成既定目标？（1=几乎没有，5=大多数） | |
| Q2 | 变革赞助人的参与度如何？（1=只挂名，5=全程深度参与） | |
| Q3 | 员工对变革的整体态度是什么？（1=抵触疲惫，5=积极支持） | |
| Q4 | 公司有专职的变革管理人员吗？（1=没有，5=有成熟团队） | |
| Q5 | 变革规划是否从战略出发，有清晰的ROI预期？（1=没有，5=每次都有） | |
| Q6 | 变革中的培训是否与推行节奏严格同步？（1=从不，5=每次都是） | |
| Q7 | 失败的变革经验是否被系统总结并传承？（1=从不，5=有完善机制） | |
| Q8 | 中层管理者是变革的推动者还是阻力？（1=主要阻力，5=主要推动者） | |
| Q9 | 新流程推行时是否同步调整了组织和绩效激励？（1=从不，5=每次都是） | |
| Q10 | 公司是否有统一的机制管理全公司的变革项目组合？（1=没有，5=有ESC类机制） | |

**快速版得分对应**：10-20分=L1，21-30分=L2，31-40分=L3，41-45分=L4，46-50分=L5

## 五、诊断结果应用

### 5.1 不同成熟度企业的变革管理建议

**L1企业（得分<60）**：
- 不建议立即启动大型变革项目
- 先做管理基础建设（梳理现状、培养意识）
- 找有丰富经验的外部顾问全程陪跑

**L2企业（得分60-100）**：
- 可以启动变革项目，但需要强力外部支撑
- 重点补课：赞助人发展、利益关系人管理、沟通计划
- 建立内部变革骨干队伍

**L3企业（得分100-140）**：
- 可以独立推进中型变革项目
- 重点提升：变革度量、知识管理、ESC治理
- 从"每次从头来"升级为"有资产可复用"

**L4-L5企业（得分>140）**：
- 有能力同时推进多个变革项目
- 重点方向：向行业输出，建立生态，持续创新

### 5.2 与薄云服务的对接

| 成熟度 | 推荐薄云服务包 |
|-------|-------------|
| L1-L2 | 变革管理全程陪跑服务（Change Accelerator） |
| L2-L3 | 变革蓝图设计服务（Change Blueprint）|
| L3-L4 | 变革体系搭建咨询（BT系统建设） |
| L4以上 | 变革管理能力认证与培训 |
