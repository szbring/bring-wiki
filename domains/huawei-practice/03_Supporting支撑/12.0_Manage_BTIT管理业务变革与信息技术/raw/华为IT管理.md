# 华为IT管理

## 一、流程与IT部门的定位与运作

华为将"流程与IT管理部"定位为公司8大部门之一，拥有1500人，二级部门包括流程变革部、流程管理、架构管理、需求管理、应用开发、IT维护运营等。该部门从辅助部门转型为**业务变革部门**，是华为的投资和利润部门。

### 1.1 流程变革项目运作模式

当公司决定进行业务变革后，成立专门项目组：
- **项目经理**：流程变革部资深人员担任，总体推进变革进程
- **咨询公司顾问**：介绍先进运营流程，制定变革建议方案
- **业务部骨干**：介绍现有流程，制定新流程推进方案
- **需求管理部**：制定由流程变革产生的IT系统需求
- **应用开发部**：实现由流程变革产生的IT需求
- **人力资源部**：制定组织架构和岗位职责调整方案

### 1.2 流程与IT结合的企业价值

1. **敏捷企业、快速变革**：从一开始就考虑IT技术支持，确保战略更好落地
2. **打破部门墙**：IT落地的流程建立跨业务、跨系统的端到端通道
3. **流程优化落到实处**：IT支持的流程积累大量数据，让优化有据可依

### 1.3 核心关系
- 与**战略部门**：流程变革部是把战略转换成流程的第一责任人
- 与**业务部门**：需求管理、应用开发部门负责评估和实现业务IT需求
- 与**HR部门**：流程变革引起组织架构调整需HR发布

## 二、华为信息化四个阶段

| 阶段 | 时间 | 特征 | 关键变革 |
|------|------|------|----------|
| 零散阶段 | 1988-1997 | 分散组织、功能型流程 | OA、MRP II、全球化DDN |
| 集中化阶段 | 1998-2002 | 集中管控 | [[IPD]]变革、财务四统一、采购变革、PDM/HR平台 |
| 国际化阶段 | 2003-2007 | 全球化覆盖 | [[ISC]]变革、ERP全球覆盖(100+国家)、全球协同办公 |
| 全球化阶段 | 2008-2013 | 端到端打通 | [[IFS]]变革、CRM变革、[[LTC]] E2E打通、全球移动办公 |
| IT2.0阶段 | 2014- | 数字化转型 | "5个1"战略、云数据中心、数据治理、CRM天兔/罗盘 |

### 2.1 华为IT2.0：全球联合作战系统

IT2.0以**云计算和大数据**为双轮驱动，目标：
1. 建立面向全球的联合作战系统
2. 构建数字化作战平台，提升运营效率

**三层架构**：
- **IaaS层**：全球云化数据中心
- **PaaS层**：H.A.E云平台（软件包平台+大数据平台）
- **SaaS层**：以人为中心的IT应用层

**应用云**：
- 营销三朵云：体验云、知识云、方案云
- 生产五朵云：销售云、财经云、交付云、供应云、采购云
- 研发七朵云：设计云、桌面云、持续集成云、仿真云、测试云、调测云、杀毒云
- 企业社交云：基于eSpace的协同办公平台

### 2.2 "5个1"战略目标

- 软件1分钟下载
- PO 1天处理完成
- 生产模块1周完成
- 1个月内货物发送到全球任何地方
- 安装完成后1个月内完成验收

## 三、业务变革与IT治理体系

### 3.1 变革四大维度

| 维度 | 要点 |
|------|------|
| 流程 | 基于业务本质；端到端拉通跨部门集成；数据与流程匹配 |
| 组织 | 按角色画流程；围绕端到端流程建立决策体系；流程责任人 |
| IT | 固化流程、组织及管理体系；规范化和提升运作效率 |
| 行为改变 | 使公司上下有意愿接受变化且有能力改变 |

### 3.2 IT治理体系五大部分

1. **战略与规划**：每年看3-5年业务变革与信息化建设规划
2. **企业架构管控**：业务架构、信息架构、技术架构
3. **项目管理**：软件包开发、应用实施、基础设施建设
4. **IT运维管理**：基于ITIL v3.0的BSM体系
5. **人力资源管理**

### 3.3 业务流程架构分层

| 层级 | 名称 | 聚焦 |
|------|------|------|
| L1 | 流程分类 | 支撑战略和业务目标，覆盖全部业务 |
| L2 | 流程组 | 体现创造客户价值的主要业务流程 |
| L3 | 流程 | 战略执行的具体业务流程 |
| L4 | 子流程 | 价值创造的子流程 |
| L5 | 活动 | 完成流程目标的具体活动 |
| L6 | 任务 | 具体任务，体现业务灵活性和多样性 |

**原则**：三层以上流程要固定不变，五层六层强调灵活可因地制宜。

### 3.4 IT预算原则：谁受益，谁承担

公共基础设施和公司级应用由公司投资，费用按BG收益分摊到业务部门。业务部门自建系统自出钱但需备案。

## 四、华为数字化变革三大转变

华为CIO陶景文提出的IT架构三大转变：

### 转变一：封闭式架构 → 云化服务化开放架构
解决传统"烟囱式"IT系统慢、散、贵的问题，实现业务快速响应。

### 转变二：关注内部流程运作 → 关注用户ROADS体验
ROADS = **Real-time（实时）、On-demand（按需）、All-online（全在线）、DIY（自助）、Social（社交化）**

### 转变三：内部作业IT系统 → 与用户连接的实时智能系统
IT系统从仅服务内部管理，转向端到端业务流程的在线、实时、自动化运作。

### 4.1 数字化实践成果

| 领域 | 实践 | 成果 |
|------|------|------|
| 研发 | 作业上云（7种服务） | 编译从94分钟→31分钟；测试从2天→8小时；构建成功率40%→95%+ |
| 销售 | 数字化销售 | 销售主管手机查看170+国家实时经营情况 |
| 交付 | 一站式服务交付平台 | 从打开20+系统→一个入口 |
| 制造 | 全球制造运营指挥中心 | 实时决策、质量预测 |
| 物流 | 物流可视平台 | 帐实一致率大幅提升，100+仓库实时监控 |
| 办公 | Welink APP | 17万用户，会议/业务/文件全连接 |

### 4.2 数字化转型五点体会

1. 业务部门主导，IT部门使能
2. ROADS体验为变革目标和成功标准
3. 面向5类用户（客户/消费者/合作伙伴/员工/供应商）构建全连接平台
4. "业务对象数字化、业务过程数字化、业务规则数字化"是转型手段
5. 确定性业务追求标准化自动化；不确定性业务通过智能化社交化实现

## 五、华为管理核心理念（IT视角）

1. **IT是企业的核心竞争要素**，是技术创新引导者
2. **集中控制、分散资源**是华为IT管治的基础选择
3. 与业务流程变革紧密结合的IT给业务带来最大价值
4. 建立与公司发展相适应的流程与IT变革管理架构与决策体系
5. **流程管理以规则的确定性对付结果的不确定**
6. **先僵化、再固化、再优化**
7. 架构和软件包驱动，坚持**主干简洁、末端灵活**
8. 基础和平台建设适度超前
9. **谁受益，谁承担**
10. IT价值和运营绩效管理

## 六、业务流程管理七大关键措施

1. **端到端流程责任人（GPO）**：负责主干流程优生优育
2. **流程IT部门为GPO配置专业资源**：指导和协助设计优化流程
3. **主干流程的仲裁和决策机构**：站在全局设计流程
4. **组织与流程相适配**：对组织结构提出明确要求
5. **对一线末端流程有效授权**：匹配末端业务场景
6. **流程版本和生命周期管理**：正确的流程设计与运营评估
7. **一线质量运营组织**：确保流程像产品一样得到运行维护

## 七、任正非"七个反对"原则

1. 反对完美主义
2. 反对繁琐哲学
3. 反对盲目创新
4. 反对没有全局效益提升的局部优化
5. 反对没有全局观的干部主导变革
6. 反对没有业务实践经验的员工参加变革
7. 反对没有充分论证的流程进入实用

## 八、华为信息化规模数据

- 流程IT人员约2000人，工资奖金占销售收入2.2%
- 15个L1流程支撑全公司，800+业务应用流程
- 全球70+数据中心
- 200000+虚拟机
- 网络带宽70Gbps
- 存储60PB
- 支持年度处理合同数630000+
- 100000名员工活跃在移动端
- IT投资占销售收入1%-1.5%，年花费约30多亿人民币

## 九、IT三阶段演进

1. **成本中心**：老板只看到花钱没看到赚钱
2. **信息中心/价值中心**：CIO为公司创造价值
3. **端到端数据智能**：全数字化转型，工业4.0/移动互联
