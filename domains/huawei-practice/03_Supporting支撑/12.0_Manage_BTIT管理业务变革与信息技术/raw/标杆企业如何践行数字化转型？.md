# 标杆企业如何践行数字化转型？

标杆公司认为数字化转型是企业利用先进技术来优化或创新的业务模式，[[以客户为中心]]，以数据为驱动，打破传统的组织效能边界和行业边界，提升企业竞争力，为企业创造新价值的过程。

标杆公司数字化转型，是基于其20多年来持续构建以流程为核心的管理体系的坚实基础上提出的。

该过程可以分两个大的阶段：信息化阶段和数字化阶段该过程可以分两个大的阶段：信息化阶段和数字化阶段。

2005年，标杆公司业务流程化程度90%，2005~2010年期间，形成了完备的数据治理，2010年前后流程信息化程度90%。

因此，我们认为数字化转型是基于坚实的数据治理基础上，它是流程信息化的高级阶段。

01阶段一：标杆公司信息化阶段的实践在业务流程信息化、筑牢数据治理的地基阶段，业务、流程、IT、质量、运营的关系如下。

标杆公司用了10年以上的时间完成了各个主要业务域端到端流程的建设，例如从1998年开始的研发业务域的[[IPD]]（集成产品开发）建设，2008年在营销业务域开始的[[LTC]]（线索到回款）变革。

同时，在该阶段，在中长期战略指导下，IT战略支撑业务战略，并驱动企业架构和IT架构的构建与更新，最终指导IT解决方案。

企业架构与IT实施之间的关系如图02阶段二：标杆公司数字化阶段的实践数字化不同于信息化，它是在信息化的基础上，通过引入云、人工智能等数字技术，把数据作为新的生产要素，同时打破物理与数字空间界限，以全量全要素的联接和实时反馈为基础，围绕快速变化的客户需求进行业务模式重构和创新。

数字化转型的本质是用数据产生业务价值，数字化转型的基础需要长期和艰巨的投入。

例如，在业务流程、应用建设、数据治理、数据湖建设和数据应用等领域都需要长期的投入。

流程是对业务最佳实践的结构化呈现，流程贯通高效后，基于流程构建应用系统。

应用系统产生及时、准确、一致的数据后，需要对数据汇聚、连接、可视和分析，最终用数据产生业务价值。

这些建设的支撑，依然是第一阶段的4A（业务架构、信息架构、应用架构与技术架构）。

当业务流程化、流程信息化的程度都已经较高，同时数据治理的坚实地基已经筑牢。

标杆公司开始了数字化转型，并形成了自身的转型框架，如下图所示。

数字化转型有连接、在线、共享、智能四大特征数字化转型有连接、在线、共享、智能四大特征。

数字化可以实现人与人、人与物、物与物、客户、员工、伙伴、数据之间的连接，让流程在线、组织在线，搭建共性平台、实现数据共享、业务协作，让作业自动化、决策智能化。

数字化平台通过“作业即记录、记录即数据”的方式，能给企业带来：1. 使得企业对广泛的业务的态势感知能力大幅加强，成本大幅降低；

2. 让不同层级、不同部门实时看到相同的信息，减少沟通成本，加快响应速度；

4. 通过对异常情况的判定与实时警报，让监管更加有的放矢4. 通过对异常情况的判定与实时警报，让监管更加有的放矢。

数字化转型是一场全新的变革，之所以是变革，在于其改变的不仅仅是生产力，还包括企业内部和企业之间的生产关系；

数字化转型影响的不仅仅是作业人员，还包括其他管理者在内的各个层级，其影响的深度和广度远大于“机器取代人”。

具体表现在："1. 打破企业内的组织层级：基于清洁、实时的数字化运营能帮助企业通过不同的层级同时看到相同的信息，从而使得组织更加扁平化。

2. 融合业务与IT能力：传统的信息化建设方式是业务部门提需求，IT部门负责实现。

最后结果是业务抱怨IT“慢、贵、难”，IT则抱怨业务讲不清楚需求、朝令夕改。

数字化所达到的也就是业务即IT,IT即业务数字化所达到的也就是业务即IT,IT即业务。

3. 对准体验提升、效率提升、模式创新三大目标：关注客户满意度和营收增长、运营效率提升、决策效率提升，进而通过运营模式、商业模式的创新为企业创造新的价值。

"在经历多年的数字化转型后，参与项目的实战团队总结出如下经验：1、数字化转型是一把手工程、变革的决心和领导力是关键；

2、数字化转型变革是长期、艰巨、复杂的举措、需建立专职、专业的变革团队；

3、数字化是手段，为客户创造价值，公司内部效率提升，成本下降等才是目标；

4、数字化转型对数据治理投入度的要求非常高，费时费力4、数字化转型对数据治理投入度的要求非常高，费时费力。

它属于短期看不到成绩的事情，需要战略定力的持续投入它属于短期看不到成绩的事情，需要战略定力的持续投入；

5、数字化转型要依据业务复杂性和数据复杂性排定工作优先级，效率模式优先；

6、数字化转型，要构建业务/数据/IT三位一体的产品团队，通过产品化的方式沉淀业务能力；

7、上一套IT系统来解决业务问题是错的，没有业务设计的IT系统是没有灵魂的。
