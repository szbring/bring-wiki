# 干部管理体系导入路径

<!-- chunk: 干部管理体系导入路径.导入总览 -->
## 导入总览

```
阶段一：诊断与设计（4-6周）→ 阶段二：机制建立（6-8周）→ 阶段三：运行固化（4-6周）
```

总周期：3-4个月（不含长期陪跑）

---

<!-- chunk: 干部管理体系导入路径.阶段一-诊断与设计-4-6周 -->
## 阶段一：诊断与设计（4-6周）

### Week 1-2：干部队伍现状诊断

**核心动作**：
- 干部花名册梳理（年龄/级别/司龄/绩效分布）
- 高管访谈（CEO/HR VP，各2小时）
- 干部能力评估（360度反馈，覆盖TOP20干部）
- 关键岗位继任者盘点

**诊断输出**：

| 维度 | 诊断内容 | 典型发现 |
|---|---|---|
| 干部结构 | 年龄/级别/司龄分布 | 老化/年轻化/断层 |
| 干部能力 | 360度评估结果 | 普遍短板在哪 |
| 干部活力 | 近3年升降级数据 | 只上不下→僵化 |
| 继任储备 | 关键岗位后备情况 | X个岗位无人可接 |

### Week 3-4：干部标准设计

**核心交付**：
- 干部定义与分类（确认哪些人算干部）
- 干部选拔「三看」标准
  - 贡献标准：PBC等级要求、关键经历
  - 能力标准：各层级管理能力模型
  - 价值观标准：行为红线
- 干部选拔流程设计

**三看标准细化示例**：

| 干部层级 | 贡献要求 | 能力要求 | 价值观要求 |
|---|---|---|---|
| 基层干部(M2) | PBC连续B+以上 | 有带3人以上经验 | 客户导向+奋斗精神 |
| 中层干部(M3-M4) | PBC连续A或B+ | 有跨部门项目经验 | +团队协作+自我批判 |
| 高层干部(M5+) | 有突破性业务贡献 | 有之字形轮岗经历 | +使命感+大局观 |

### Week 5-6：评价与淘汰机制设计

**核心交付**：
- 干部三维评价体系
- 干部年度述职制度
- 末位淘汰规则
- 民主评议制度

**末位淘汰规则设计要点**：

| 设计项 | 选择范围 | 推荐 |
|---|---|---|
| 淘汰比例 | 5%/10%/15% | 10%（华为标准） |
| 考核周期 | 年度/半年度 | 年度 |
| 处理方式 | 降级/转岗/免职 | 分层处理 |
| 再上机制 | 有/无 | 必须有（否则不敢下） |

**里程碑**：[[华为干部管理]]制度设计方案评审通过

## 阶段二：机制建立（6-8周）

### Step 1：干部评审委员会组建（1-2周）

**组成**：
- 主任：CEO
- 成员：HR VP + 业务VP + 资深专家
- 秘书：HR干部部

**运作规则**：
- 季度例会（评审选拔/淘汰/轮岗）
- 紧急事项可临时召开
- 决策需2/3以上成员同意

### Step 2：现有干部盘点与定级（2-3周）

**盘点方法**：
1. 每位干部填写干部信息表（经历/绩效/能力自评）
2. 直属上级初评（对照三看标准）
3. 评审委员会校准（确保标准一致）
4. 输出干部九宫格

**干部九宫格**：

```
           高潜力
             │  ☆重点培养   ☆重点培养   ★破格提拔
             │
        中潜力
             │  ○关注发展   ○稳定贡献   ☆重点培养
             │
        低潜力
             │  △警示观察   △维持使用   ○稳定贡献
             └──────────────────────────────→
              低绩效      中绩效      高绩效
```

### Step 3：继任者计划启动（2-3周）

**核心动作**：
- 确定关键岗位清单（通常TOP15-20个岗位）
- 每个岗位确定2名后备人选
- 后备人选差距分析
- 制定个人发展计划（IDP）

**继任者评估维度**：

| 维度 | 就绪度 | 预计到位时间 |
|---|---|---|
| 完全就绪 | 能力+经验均达标 | 0-6个月 |
| 1-2年就绪 | 有差距但可弥补 | 1-2年 |
| 长期培养 | 差距较大 | 2-3年 |

### Step 4：首轮干部选拔评审（1-2周）

**流程**：
```
组织推荐/自荐 → 材料审核 → 评审委员会评审 → 任前公示 → 试用期
```

**评审方式**：
- 查阅PBC记录和绩效数据（贡献维度）
- 结构化面试（能力维度，30分钟/人）
- 民主评议汇总（价值观维度）
- 委员会讨论投票

**里程碑**：干部评审委员会成立并完成首轮评审，继任者计划启动

## 阶段三：运行固化（4-6周）

### 制度发布

**发布内容**：
- 干部管理制度正式文件
- 干部选拔标准与流程
- 干部述职制度
- 末位淘汰操作细则
- 轮岗管理制度
- 继任者计划管理办法

### 干部培养项目启动

| 项目 | 对象 | 首期人数 | 时长 |
|---|---|---|---|
| 青训班 | M2/M3新任干部 | 15-20人 | 2个月（在职） |
| 高研班 | M4/M5骨干干部 | 10-15人 | 3个月（在职） |

### 固化机制

| 机制 | 频率 | 内容 |
|---|---|---|
| 干部例会 | 季度 | 选拔/淘汰/轮岗决策 |
| 述职评审 | 半年 | 干部三维评价 |
| 人才盘点 | 年度 | 九宫格更新/继任者调整 |
| 民主评议 | 年度 | 价值观维度评估 |

## 常见阻力与应对

| 阻力 | 应对 |
|---|---|
| 「末位淘汰太冷血」 | 强调「能上能下」，下来不是终点，可再上去 |
| 「干部都是我培养的，不能淘汰」 | 用三看标准说话，不是针对个人 |
| 「轮岗影响业务连续性」 | 轮岗有交接期，短期阵痛vs长期健康 |
| 「评审委员会形同虚设」 | CEO必须亲自主持，否则无权威 |
| 「继任者计划没用，人随时走」 | 正因为没有储备才更脆弱，必须开始 |

## 验收标准

- [ ] 干部管理制度正式发布
- [ ] 干部评审委员会成立并运作
- [ ] 干部选拔标准与流程明确
- [ ] 现有干部完成盘点，输出九宫格
- [ ] 关键岗位继任者计划启动（覆盖TOP15岗位）
- [ ] 末位淘汰规则明确并首次执行
- [ ] 首期干部培养项目启动
- [ ] 至少完成1轮干部述职评审
