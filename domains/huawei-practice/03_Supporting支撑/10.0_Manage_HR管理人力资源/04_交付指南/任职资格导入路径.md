# 任职资格体系导入路径

<!-- chunk: 任职资格导入路径.三阶段导入 -->
## 三阶段导入

### 阶段一：设计（4-6周）

**Week 1-2：调研与设计准备**
- 收集现有岗位信息（JD、汇报关系）
- 访谈CEO/HR/业务VP（各1小时）
- 确认岗位族分类（技术/管理/销售/职能）
- 确定级别数量（通常4-6级，不宜过多）

**Week 3-4：标准编写**
- 优先编写TOP10核心岗位的任职标准
- 每个标准包含：知识/技能/行为/经验四维度
- 组织业务专家评审，确保标准准确

**Week 5-6：机制设计**
- 晋升评审委员会组建（HR VP + 业务VP + 资深专家）
- 评审流程设计（申请→评审→公示）
- 薪酬带宽与级别挂钩方案

**里程碑**：[[华为任职资格体系]]框架 + TOP10岗位标准定稿

---

### 阶段二：定级（4-6周）

**核心动作**：给现有员工套级

**定级原则**：
- 宁低不高（错误定高了很难降）
- 一次定级可挂钩小幅薪酬调整（正向激励）
- 定级结果不完全公开（避免攀比）

**定级方法**：
1. 员工自评（填写自评表）
2. 主管初审（结合日常观察）
3. 委员会审核（高级别需评审委审核）
4. HR统一校验（防止内部不公平）

**常见问题**：
- 员工高估自己 → 用具体事例对照标准
- 主管为保人才故意定高 → 委员会独立审核
- 员工不认可定级 → 解释标准，接受申诉

**里程碑**：全员定级完成，结果通知到个人

---

### 阶段三：固化（4-6周）

**制度化**：
- 任职资格管理办法正式发布
- 晋升评审制度（半年/年度）
- 薪酬管理办法更新（与级别挂钩）

**配套机制**：
- 晋升宣传：「升到T4意味着什么」可视化
- 差距辅导：让员工知道「我缺什么」
- 培训体系与任职资格对接

<!-- chunk: 任职资格导入路径.常见阻力与应对 -->
## 常见阻力与应对

| 阻力 | 应对 |
|---|---|
| 「这是给人贴标签」 | 强调是发展工具，不是淘汰工具；先说晋升路径 |
| 「标准脱离实际」 | 业务专家参与标准编写，不只是HR制定 |
| 「执行不了」 | 先覆盖TOP30%关键岗位，从小范围开始 |
| 「薪酬要涨」 | 定级调薪控制在总薪酬包3-5%范围内 |

<!-- chunk: 任职资格导入路径.验收标准 -->
## 验收标准

- [ ] 全部岗位族有级别定义
- [ ] TOP50%岗位有任职标准文件
- [ ] 全员完成定级，结果通知到个人
- [ ] 晋升评审委员会成立并运作
- [ ] 任职资格管理制度正式发布
- [ ] 完成至少一轮晋升评审
