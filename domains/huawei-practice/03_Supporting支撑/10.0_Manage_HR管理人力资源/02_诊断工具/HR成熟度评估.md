# HR管理成熟度评估

<!-- chunk: hr成熟度评估.五级成熟度 -->
## 五级成熟度

| 级别 | 名称 | 特征 |
|---|---|---|
| L1 | 行政管理 | HR=办理入离职+发工资，无职业通道，晋升靠关系 |
| L2 | 基础人事 | 有岗位体系和薪酬体系，但标准模糊，绩效凭印象 |
| L3 | 规范化管理 | 任职资格建立，绩效有PBC，HR开始支持业务 |
| L4 | 战略伙伴 | HR三支柱运作，人才数据驱动决策，HRBP融入业务 |
| L5 | 持续卓越 | 人才管理成为竞争优势，HR是战略资产 |

<!-- chunk: hr成熟度评估.六维诊断模型 -->
## 六维诊断模型

### 1. 职业发展通道
- [ ] 是否有明确的职级体系？
- [ ] 技术和管理双通道是否清晰？
- [ ] 每级晋升标准是否员工可查？
- [ ] 晋升评审是否有客观流程？

### 2. 绩效管理
- [ ] 是否有年度绩效目标承诺？
- [ ] 绩效结果是否有差异化分布？
- [ ] 绩效是否与薪酬/晋升直接挂钩？
- [ ] 是否有绩效改进计划（PIP）机制？

### 3. 薪酬激励
- [ ] 薪酬带宽是否基于任职级别？
- [ ] 高绩效者与低绩效者薪酬差距是否≥1.5倍？
- [ ] 是否有长期激励（股权/期权）？
- [ ] 薪酬结构是否清晰透明？

### 4. 人才招聘与保留
- [ ] 是否有岗位画像和招聘标准？
- [ ] 核心岗位离职率是否可接受（<15%/年）？
- [ ] 是否有离职分析机制？
- [ ] 是否有人才储备（继任者计划）？

### 5. 学习与发展
- [ ] 是否有系统的新员工培训？
- [ ] 管理者是否有领导力培训？
- [ ] 是否有技术/专业能力培训体系？
- [ ] 培训效果是否被评估？

### 6. HR角色与能力
- [ ] HR是否深度参与业务决策？
- [ ] HR是否使用数据支撑建议？
- [ ] HR是否主动识别组织问题？
- [ ] HR能力是否与业务需求匹配？

## 快速诊断问卷（售前5分钟版）

1. **「员工知道自己现在是什么级别吗？下一步晋升需要什么条件？」**
   - 不清楚 → L1-L2
   - 有职级但标准模糊 → L2
   - 有清晰标准和评审 → L3+

2. **「绩效考核结果，最好的员工和最差的员工薪酬差多少？」**
   - 基本一样 → L1-L2（大锅饭）
   - 差20-30% → L2-L3
   - 差50%以上 → L3+

3. **「过去一年，核心研发/技术骨干流失了几个？主要原因是什么？」**
   - 流失率>15%或原因是「看不到前途」 → [[华为任职资格体系]]缺失
   - 原因是「薪酬竞争力不够」 → [[华为的激励机制]]问题
   - 原因是「被挖走」 → 相对正常，聚焦留人机制

4. **「HR部门主要在做什么工作？花时间最多的是？」**
   - 入离职/考勤/社保 → SSC级别，HR价值未释放
   - 招聘+培训 → L2，开始关注人才
   - 参与业务决策 → L3+，HRBP转型中
