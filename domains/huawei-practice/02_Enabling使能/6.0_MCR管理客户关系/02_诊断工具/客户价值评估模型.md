# 客户价值评估模型

客户价值评估是MCR流程中客户选择和分级的基础工具。B2B场景下，传统的RFM模型需要适配，同时需要叠加战略价值和成长价值维度，形成适合B2B的复合评估模型。

---

<!-- chunk: 客户价值评估模型.一-b2b客户价值评估框架 -->
## 一、B2B客户价值评估框架

### 1.1 三维价值模型

| 维度 | 定义 | 核心问题 |
|---|---|---|
| 当前价值 | 客户当前对企业的经济贡献 | 这个客户现在值多少钱？ |
| 战略价值 | 客户对企业战略目标的支持程度 | 这个客户对我们的战略有多重要？ |
| 成长价值 | 客户未来的增长潜力 | 这个客户未来值多少钱？ |

### 1.2 为什么B2B不能直接用RFM

RFM（Recency-近度、Frequency-频度、Monetary-金额）是B2C的经典模型，但在B2B场景存在局限：

| RFM维度 | B2C场景 | B2B场景差异 |
|---|---|---|
| Recency（最近购买时间） | 高频消费，时效敏感 | B2B采购周期长，半年前购买也算"近" |
| Frequency（购买频率） | 每月/每周购买 | B2B按项目采购，年度1-2次也正常 |
| Monetary（购买金额） | 单次金额小 | 单次金额大，波动大，需看年度总额 |

因此，B2B需要对RFM进行适配改造。

<!-- chunk: 客户价值评估模型.二-适配b2b的rfm模型 -->
## 二、适配B2B的RFM模型

### 2.1 改造后的B-RFM模型

| 维度 | B2B定义 | 评分标准（5分制） |
|---|---|---|
| **B-Recency**（业务近度） | 最近一次合作/项目的时间 | 1分：>2年 → 5分：<3月 |
| **B-Frequency**（合作频度） | 年度合作项目数/合同数 | 1分：0个 → 5分：≥5个 |
| **B-Monetary**（业务金额） | 年度营收贡献 | 1分：<50万 → 5分：>500万 |

### 2.2 B-RFM评分细则

**B-Recency（业务近度）**

| 评分 | 标准 | 含义 |
|---|---|---|
| 5 | 3个月内有合作 | 活跃客户 |
| 4 | 3-6个月 | 正常节奏 |
| 3 | 6-12个月 | 需关注 |
| 2 | 1-2年 | 流失风险 |
| 1 | 超过2年 | 已流失或休眠 |

**B-Frequency（合作频度）**

| 评分 | 标准 | 含义 |
|---|---|---|
| 5 | 年度≥5个项目 | 高频合作 |
| 4 | 3-4个项目 | 稳定合作 |
| 3 | 2个项目 | 正常合作 |
| 2 | 1个项目 | 低频合作 |
| 1 | 0个项目（有历史合作） | 休眠客户 |

**B-Monetary（业务金额）**

| 评分 | 标准 | 含义 |
|---|---|---|
| 5 | 年度>500万 | 头部客户 |
| 4 | 200-500万 | 重要客户 |
| 3 | 100-200万 | 中等客户 |
| 2 | 50-100万 | 一般客户 |
| 1 | <50万 | 长尾客户 |

> 注：金额阈值需根据行业和企业规模调整。

### 2.3 B-RFM客户分类

| B-RFM组合 | 客户类型 | 特征 | 策略 |
|---|---|---|---|
| 高高高（555） | 黄金客户 | 活跃、高频、高金额 | 顶级服务，防流失 |
| 低高高（155） | 重要休眠 | 曾经大客户但已不活跃 | 重新激活，分析流失原因 |
| 高低高（515） | 大单客户 | 偶尔大项目 | 提升频次，交叉销售 |
| 高高低（551） | 高频小客 | 经常合作但金额小 | 提升客单价 |
| 低低高（115） | 沉睡大客 | 曾经大客户已流失 | 评估是否值得挽回 |
| 低低低（111） | 长尾客户 | 低价值 | 标准化服务，低成本维护 |

## 三、战略价值评估

### 3.1 评估维度

| 维度 | 定义 | 评分（1-5分） |
|---|---|---|
| 品牌背书 | 客户是否为行业标杆，合作能否带来品牌效应 | 1=无名 → 5=行业龙头 |
| 技术引领 | 与客户合作能否推动技术进步 | 1=无 → 5=前沿引领 |
| 市场准入 | 客户能否帮助进入新市场或新领域 | 1=无 → 5=关键通道 |
| 生态价值 | 客户生态圈是否带来间接机会 | 1=无 → 5=生态核心 |
| 人才吸引 | 客户能否帮助吸引人才 | 1=无 → 5=显著帮助 |

### 3.2 战略价值判断标准

| 战略价值总分 | 评级 | 说明 |
|---|---|---|
| 20-25分 | S级战略 | 必须全力争取和维持 |
| 15-19分 | A级重要 | 值得重点投入 |
| 10-14分 | B级一般 | 有一定战略意义 |
| 5-9分 | C级较低 | 战略意义有限 |

<!-- chunk: 客户价值评估模型.四-成长价值评估 -->
## 四、成长价值评估

### 4.1 评估维度

| 维度 | 定义 | 评分（1-5分） |
|---|---|---|
| 客户行业增长率 | 客户所在行业的增长速度 | 1=萎缩 → 5=高增长 |
| 客户自身增长 | 客户营收和利润增长趋势 | 1=下滑 → 5=高增长 |
| 交叉销售潜力 | 可向客户销售的产品/服务线数 | 1=1条 → 5=5条以上 |
| 增购空间 | 客户预算和需求增长空间 | 1=饱和 → 5=空间大 |
| 合作深化空间 | 从供应商到伙伴到战略伙伴的升级空间 | 1=已到顶 → 5=大空间 |

### 4.2 成长价值判断

| 成长价值总分 | 评级 | 说明 |
|---|---|---|
| 20-25分 | 高成长 | 值得提前投入，抢占未来 |
| 15-19分 | 中成长 | 关注并适度投入 |
| 10-14分 | 低成长 | 维持现状 |
| 5-9分 | 衰退型 | 控制投入，考虑退出 |

## 五、综合评估与客户分级

### 5.1 三维加权模型

**综合价值 = 当前价值 x 40% + 战略价值 x 30% + 成长价值 x 30%**

| 维度 | 原始分 | 标准化（百分制） | 权重 | 加权分 |
|---|---|---|---|---|
| 当前价值（B-RFM） | /15 |  | 40% |  |
| 战略价值 | /25 |  | 30% |  |
| 成长价值 | /25 |  | 30% |  |
| **综合** |  |  |  | **/100** |

### 5.2 综合分级标准

| 综合得分 | 客户级别 | 投入策略 |
|---|---|---|
| 80-100分 | S类战略客户 | 最高优先，高层直管，专属团队 |
| 60-79分 | A类伙伴客户 | 重点投入，专职客户经理 |
| 40-59分 | B类普通客户 | 标准服务，定期维护 |
| <40分 | C类长尾客户 | 标准化服务，低成本运营 |

### 5.3 动态调整规则

| 触发条件 | 调整动作 |
|---|---|
| 客户连续2个QBR评分上升 | 考虑升级（B→A） |
| 客户连续2个QBR评分下降 | 考虑降级（A→B） |
| 客户流失风险预警 | 启动挽留计划 |
| 客户行业发生重大变化 | 重新评估成长价值 |
| 客户关键人变动 | 重新评估关系覆盖 |

## 六、应用场景

| 场景 | 如何使用 |
|---|---|
| 年度客户规划 | 年初全面评估，确定分级和投入策略 |
| 资源分配决策 | 按客户价值分配销售资源和预算 |
| 流失预警 | 监测价值变化趋势，提前预警 |
| 新客户开拓 | 评估目标客户的价值，决定是否投入 |
| 续约谈判 | 了解客户价值，制定续约策略 |
