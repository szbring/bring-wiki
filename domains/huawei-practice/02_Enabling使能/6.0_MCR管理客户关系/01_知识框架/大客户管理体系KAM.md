# 大客户管理体系KAM

KAM（Key Account Management，大客户管理）是[[MCR]]流程在战略客户层面的落地体系。华为的KAM不是简单的"大客户销售"，而是一套从组织、流程、工具到能力的完整管理体系，确保战略客户关系的深度和可持续性。

---

<!-- chunk: 大客户管理体系kam.一-kam体系总览 -->
## 一、KAM体系总览

### 1.1 为什么需要KAM

| 没有KAM | 有KAM |
|---|---|
| 大客户依赖个别销售，人员变动即流失 | 客户关系建在组织层面，人员变动影响可控 |
| 大小客户一视同仁，资源撒胡椒面 | 差异化投入，战略客户获得专属资源 |
| 客户需求被动响应 | 主动洞察需求，提前布局 |
| 各产品线各自为战，客户感知碎片化 | 统一客户界面，一体化交付价值 |
| 客户满意度靠感觉判断 | 可量化的客户健康度评分 |

### 1.2 KAM体系四要素

```
┌──────────────────────────────────────────┐
│              KAM 体系                     │
│                                          │
│   ┌──────────┐    ┌──────────────┐       │

│   │          │    │ 计划         │       │
│   └──────────┘    └──────────────┘       │
│                                          │
│   ┌──────────┐    ┌──────────────┐       │
│   │ 客户经理  │    │ 运营机制     │       │
│   │ 能力模型  │    │             │       │
│   └──────────┘    └──────────────┘       │
│                                          │
└──────────────────────────────────────────┘
```

## 二、KAM组织设计

### 2.1 三层组织架构

| 层级 | 角色 | 职责 | 汇报关系 |
|---|---|---|---|
| 战略层 | 客户赞助人（Sponsor） | 高层关系维护、战略方向把控、资源协调 | 公司高管/CEO |
| 管理层 | 客户总监（Account Director） | 客户经营计划制定、跨部门协调、业绩负责 | 销售副总裁 |
| 执行层 | 客户经理（Account Manager） | 日常关系维护、商机推进、交付协调 | 客户总监 |

### 2.2 铁三角协同

华为铁三角是KAM的核心作战单元：

| 角色 | 职责 | 考核重点 |
|---|---|---|
| AR（客户经理） | 客户关系、商机挖掘、项目运作 | 客户满意度、商机转化率 |
| SR（解决方案专家） | 技术方案、产品匹配、差异化竞争 | 方案竞争力、技术满意度 |
| FR（交付专家） | 交付承诺、项目落地、售后保障 | 交付质量、客户续约率 |

铁三角不是三个人的简单组合，而是一个面向客户的协同作战组织，以AR为核心牵引。

### 2.3 客户团队配置标准

| 客户级别 | 客户经理 | 解决方案 | 交付 | 高层Sponsor |
|---|---|---|---|---|
| S类战略客户 | 专职1人 | 专职1人 | 专职1人 | CEO/VP级 |
| A类伙伴客户 | 专职1人 | 兼职1人 | 兼职1人 | 总监级 |
| B类普通客户 | 兼职（管2-3家） | 按需调配 | 按需调配 | 部门经理 |
| C类长尾客户 | 区域团队统一覆盖 | 远程支持 | 标准交付 | 无 |

## 三、战略客户经营计划

### 3.1 定义与价值

战略客户经营计划（Account Business Plan，ABP）是KAM的核心工具，是客户经理对战略客户的"年度作战计划"。华为要求S类和A类客户必须制定ABP，并每季度回顾更新。

### 3.2 ABP七要素

| 要素 | 内容 | 输出 |
|---|---|---|
| 客户洞察 | 行业、企业、决策链、竞争态势 | 客户画像报告 |
| 关系现状 | 三层客户关系现状评估 | 客户关系地图 |
| 业绩目标 | 营收、利润、份额、新项目 | 量化目标表 |
| 关系目标 | 各关键客户关系提升目标 | 关系提升计划 |
| 机会清单 | 潜在项目、时间窗口、竞争态势 | 商机Pipeline |
| 行动计划 | 月度关键动作、责任人、时间节点 | 行动甘特图 |
| 资源需求 | 需要公司投入的资源和支持 | 资源申请清单 |

### 3.3 ABP制定流程

```
客户洞察 → 关系评估 → 目标设定 → 策略制定 → 行动规划 → 资源匹配
    ↑                                                    │
    └────────────── 季度回顾调整 ←──────────────────────┘
```

### 3.4 季度业务回顾（QBR）

QBR是KAM最重要的运营节奏：

| QBR内容 | 目的 |
|---|---|
| 上季度业绩回顾 | 目标达成情况、偏差分析 |
| 客户健康度评分 | 满意度、关系深度、竞争态势 |
| 下季度机会与风险 | 重点商机、关键风险、应对策略 |
| 客户需求与反馈 | 新需求、痛点、改进建议 |
| 关系提升进展 | 三层客户关系提升情况 |
| 资源调整需求 | 需要增减的资源 |

## 四、客户经理能力模型

### 4.1 五力模型

| 能力维度 | 定义 | 关键行为 |
|---|---|---|
| 客户洞察力 | 深度理解客户的能力 | 行业分析、决策链梳理、需求预判 |
| 关系构建力 | 建立和维护客户关系的能力 | 多层级接触、信任建立、教练培养 |
| 商机运作力 | 从线索到成单的能力 | 机会识别、竞争策略、赢单动作 |
| 资源整合力 | 调动内部资源服务客户的能力 | 铁三角协同、跨部门协调、高层资源调用 |
| 价值交付力 | 确保客户价值实现的能力 | 交付质量、客户成功、续约增购 |

### 4.2 能力分级标准

| 级别 | 客户洞察力 | 关系构建力 | 商机运作力 | 资源整合力 | 价值交付力 |
|---|---|---|---|---|---|
| 初级（1-2年） | 能收集基础信息 | 能维护对接人关系 | 能跟进明确商机 | 能协调内部1-2个部门 | 能完成标准交付 |
| 中级（3-5年） | 能分析决策链 | 能构建关键客户关系 | 能独立运作中型项目 | 能协调铁三角 | 能确保客户满意 |
| 高级（5-8年） | 能预判客户需求 | 能构建组织客户关系 | 能运作战略级项目 | 能调动公司高层资源 | 能推动客户成功 |
| 专家（8年+） | 能洞察行业趋势 | 能主导战略合作 | 能运作复杂大单 | 能整合生态资源 | 能共创客户价值 |

### 4.3 客户经理考核指标

| 指标类别 | 指标 | 权重 |
|---|---|---|
| 业绩指标 | 营收达成率 | 30% |
| 业绩指标 | 利润率 | 10% |
| 客户指标 | 客户满意度 | 15% |
| 客户指标 | 客户留存率 | 10% |
| 关系指标 | 关系覆盖度 | 10% |
| 关系指标 | 商机Pipeline健康度 | 10% |
| 过程指标 | QBR执行率 | 5% |
| 过程指标 | ABP完成度 | 5% |
| 成长指标 | 新客户开拓 | 5% |

## 五、KAM运营机制

### 5.1 会议体系

| 会议 | 频次 | 参与人 | 目的 |
|---|---|---|---|
| 周例会 | 每周 | 客户团队 | 商机进展、问题协调 |
| 月度经营分析 | 每月 | 客户总监+团队 | 月度目标回顾、策略调整 |
| QBR | 每季 | 高层+客户团队 | 战略对齐、资源调整 |
| 年度规划会 | 每年 | CEO+管理层 | 年度ABP制定、目标设定 |

### 5.2 信息流转

```
客户接触 → 信息录入CRM → 客户画像更新 → ABP调整 → 行动计划更新 → 执行反馈
    ↑                                                              │
    └──────────────────── 持续循环 ←─────────────────────────────┘
```

### 5.3 常见问题与对策

| 问题 | 根因 | 对策 |
|---|---|---|
| ABP流于形式 | 只定不执行，缺乏回顾 | 强制QBR机制，与考核挂钩 |
| 客户经理单打独斗 | 铁三角形同虚设 | 明确三角色职责，联合考核 |
| 资源争抢 | 多客户经理争夺有限资源 | 基于客户分级的资源分配机制 |
| 高层参与不足 | Sponsor名存实亡 | 设定高层见客户最低频次，纳入考核 |
| 客户信息分散 | 各系统不打通 | 统一CRM平台，强制信息录入 |
