# WBS工作分解模板

WBS（Work Breakdown Structure）是将项目可交付成果和工作分解为更小、更易于管理的组件的结构化方法。

## 二、WBS分解结构

### 第1层：项目

```
项目名称
├── 1. 阶段一：[名称]
│   ├── 1.1 [工作包]
│   │   ├── 1.1.1 [活动]
│   │   └── 1.1.2 [活动]
│   └── 1.2 [工作包]
│       ├── 1.2.1 [活动]
│       └── 1.2.2 [活动]
├── 2. 阶段二：[名称]
│   ├── 2.1 [工作包]
│   └── 2.2 [工作包]
├── 3. 阶段三：[名称]
│   ├── 3.1 [工作包]
│   └── 3.2 [工作包]
└── 4. 项目管理
    ├── 4.1 项目规划
    ├── 4.2 项目监控
    └── 4.3 项目收尾
```

<!-- chunk: wbs工作分解模板.三-wbs分解原则 -->
## 三、WBS分解原则

| 原则 | 说明 |
|---|---|
| 100%原则 | WBS必须覆盖项目100%的工作范围 |
| 互斥原则 | 各WBS元素之间不能有重叠 |
| 8/80法则 | 单个工工作包工时在8-80小时之间 |
| 3-6层深度 | WBS层级不超过6层 |
| 名词定义 | WBS元素用名词（交付物），不用动词（活动） |

## 四、责任分配矩阵（RACI）

| WBS元素 | PM | 技术经理 | 交付经理 | 客户代表 | 质量经理 |
|---|---|---|---|---|---|
| 1.1 工作包 | A | R | C | I | C |
| 1.2 工作包 | A | C | R | I | C |
| 2.1 工作包 | A | R | C | C | I |

> R=执行 Responsible A=批准 Accountable C=咨询 Consulted I=知会 Informed
