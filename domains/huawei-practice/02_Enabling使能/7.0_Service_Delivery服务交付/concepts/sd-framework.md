# 服务交付管理体系

## 定义
华为服务交付管理体系是以项目为中心、结构化流程驱动的端到端交付框架。经历了四代演进：PM 1.0 专业化（2004-2012，PMO 成立、PMP1.0 发布）、PM 2.0 体系化（2013-2016，PMP 与 LTC/ISC/财经集成）、PM 3.0 数字化（2017-2020，ISDP 平台数字化作战）、PM 4.0 价值化（2021 至今，项目经理成为 CEO、服务标准化外溢）。核心理念：项目经理即项目 CEO，对商业成功负责，充分授权、责权统一，平台赋能加精兵作战。

## 输入 / 输出
- **输入**：销售合同/PO、交付可行性报告、客户需求与 SLA
- **输出**：验收通过的交付物、决算报告、复盘报告与知识沉淀
- **核心子模块**：[[id:huawei.sd.concept.c8-team]] C8 团队 / [[id:huawei.sd.concept.gme-governance]] GME 决策 / [[id:huawei.sd.concept.service-quality]] 服务质量 / [[id:huawei.sd.concept.evm]] EVM 挣值 / [[id:huawei.sd.concept.pm-maturity]] 成熟度模型

## 角色
- **决策方**：GME 三层决策体系中的 G 层（Sponsor + 客户 CXO）
- **管理方**：项目经理（PM）作为项目 CEO，统筹 C8 核心团队
- **执行方**：C8 团队拉动功能部门资源执行交付
- **监督方**：PMO、质量经理（QM）

## 上下游
- **上游**：[[id:huawei.ltc.concept.ltc-framework]] 销售签约后转入交付，合同/PO 为交付起点
- **下游**：交付验收 → 移交运维 → [[id:huawei.itr.concept.itr-framework]] 售后问题处理
- **平行**：IPD（产品开发支撑交付）、DSTE（战略对齐交付资源）

## 典型误区
- **交付与经营脱钩**：项目经理只管交付不管损益，忽视项目经营指标
- **PM 当执行者不当 CEO**：被动执行计划而非主动经营客户满意与商业价值
- **四代演进跳步**：未夯实体系化（2.0）就追求数字化（3.0），根基不稳
- **DR 评审走过场**：六个交付评审点形同虚设，未起到质量闸口作用
