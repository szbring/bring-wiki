# 华为的项目管理

## 一、华为项目管理概述

华为的项目管理是一种业务运作模式，"以项目为中心"的运作不仅仅是实践或工具，更是一套完整的管理体系，包括**政策、规则、流程、方法和IT工具平台、组织运作和评价**等要素。通过三层的管控机制，有效开展项目、项目集和项目组合管理，实现商业价值。

华为"以项目为中心"的运作是确保实现"[[以客户为中心_Ch1_为客户服务是唯一理由|以客户为中心]]，[[以奋斗者为本_Ch1_全力创造价值|以奋斗者为本]]，长期坚持艰苦奋斗"核心价值观的具体承载。

### 关键里程碑

| 时间 | 事件 |
|------|------|
| 2000年 | 首名工程师获PMP认证，鼓励全员参加PMP |
| 2001年 | 孙亚芳提出培养5名总监级项目经理管理海外交钥匙项目；研发进入产品经理和项目经理的"双轨时代" |
| 2005年 | 开发《华为高级项目管理研讨》课程，VP级领导系统学习项目管理 |
| 2008年 | 开发《交付项目管理高级研讨》，全球代表系统学习 |
| 2009年 | [[IFS]]变革推进项目"四算"（概预核决），项目管理进入项目CEO经营时代 |
| 2011年 | ISD变革确立项目管理作为核心能力的显性化地位 |
| 2013年 | 成立公司级PMCoE（项目管理能力中心），系统开展"以项目为中心"运作变革 |

### 人才成长路径

华为项目管理人才遵循"士兵(基层) -> 英雄(骨干) -> 班长(基层管理者) -> 将军(中高层)"的职业路径，分为三个阶段：

1. **基层历练阶段**：干一行、爱一行、专一行，强调从最基层项目做起
2. **战训结合阶段**：循环轮换("之"字成长) + 赋能培训（青训班、FLMP）
3. **理论收敛阶段**：高级管理研讨（高研班/华为"抗大"），传递管理哲学和核心价值观

## 二、华为项目管理的七大特点

### 2.1 基于流程的产品开发项目管理

华为产品开发流程分为6个阶段：概念 -> 计划 -> 开发 -> 验证 -> 发布 -> 生命周期管理。以LPDT(产品开发项目领导)管理项目工作，按公司定义的流程完成目标。

**关键发现**：概念阶段和计划阶段虽然周期更长，但产品定义清晰、方案具体，反而缩短了整体开发周期。

### 2.2 端到端管理

"端到端"从市场中来，通过项目活动满足市场需求。产品开发不仅是技术部门的工作，需要市场、销售、注册、技术、制造等跨部门团队共同完成。

### 2.3 跨部门项目管理模式

采用**重度矩阵式**管理模式：
- PDT成员由LPDT和部门经理共同协商确定
- 职能部门经理由"既管事又管人"转变为"只管人"（能力培养、人力资源规划）
- LPDT对团队成员具有考核权

### 2.4 研发项目分类管理

| 类型 | 特点 | 考核重点 |
|------|------|----------|
| 产品预研 | 风险大、结果难预知 | 进度/结果考核权重小 |
| 产品开发 | 进度可预知、质量可控 | 进度、质量、财务 |
| 技术预研 | 技术倾向明显 | 技术突破 |
| 技术开发 | 工程化倾向明显 | 工程交付 |

### 2.5 过程审计保障

PQA(产品质量保证)承担过程审计，定位于项目中的流程专家：
- 流程引导者：培训团队熟悉流程和管理制度
- 过程组织者：组织技术评审
- 独立审计者：审计团队是否按流程执行

### 2.6 项目经理培养体系

从**知识、技能、行为和素质**四个方面认证项目经理胜任力，规定五个等级。成立项目管理能力建设组，制定培养规划。

### 2.7 技术管理与项目管理分离

- **PDT经理**（管理专家）：协调各部门关系，类似CEO
- **系统工程师**（技术专家）：需求转化、架构设计、技术把控

## 三、项目型组织

### 本质定义

华为项目型组织：为顺利完成项目或项目群目标而建立的工作团队，通常独立于功能型组织，成员来自一个或多个功能型组织。

核心本质：**激发组织活力，改善运作效率，增强项目盈利，提升客户满意度**。

### 大平台下的精兵作战

华为从2007年推进项目型组织建设，2015年明确"从以功能型组织为中心，向以项目型组织为中心转变"。

核心思路：
- 指挥权前移，减少决策层级
- 一线"战区主战"自主作战
- 服务、资源、能力云化，构建平台化支撑
- **项目型组织必须对应平台型组织**，没有平台支撑就是伪命题

### 打造项目型组织的五个方面

1. **组织的定义**：项目分级、生成、变更、关闭及标准模型
2. **授权管理**：人、财、事上对项目经理的授权（权力进项目）
3. **资源的调度与使用**："机关资源池化、资源市场化"，定价、上架、使用
4. **评价与激励**：项目奖的来源/分配/发放，加大进项目人员和平台人员的奖金差距
5. **IT支撑**：工时系统、运营可视化系统等

### 三个关键要素

1. **资源调配**（最难）：建立资源买卖机制，保证人钱分离——经营单元有钱无人，资源部门有人无钱。资源部门养兵，项目经理用兵，市场机制调兵
2. **项目经理领导力**：从职务影响力转向品牌和影响力，通过授权、服务进行沟通协调
3. **项目HR转型**：从V-cross模型转向"人"字模型（责权管理、资源管理、评价管理），先于业务流程准备HR动作

## 四、项目管理文化与沟通

### 三种项目管理文化

| 文化层次 | 内容 |
|----------|------|
| 理念文化 | 公司价值观、项目管理价值观、高层领导支持 |
| 制度文化 | 规范的项目管理体系、能力框架、匹配的组织架构 |
| 行为文化 | 职业化行为、卓越领导力、健康生态圈 |

### 文化精髓

1. **[[以客户为中心_Ch1_为客户服务是唯一理由|以客户为中心]]**：支点建在离客户最近的地方，让听得见炮声的人呼唤炮火
2. **契约精神**：与公司立约(政策/流程)、与客户立约(信任)、与员工立约(KPI)
3. **结果导向**：项目KPI纳入个人绩效承诺书，以结果为导向的任职资格
4. **团队协同作战**："胜则举杯相庆，败则拼死相救"，狼性文化=拼搏+团队意识

### 沟通三原则

1. 与关键者沟通：客户、高层领导、相关方
2. 与项目团队沟通：信息透明、及时同步
3. 学会倾听：人人必会的技能

## 五、项目流程管理

### 流程发展历程

- **[[IPD]]流程**：概念 -> 计划 -> 开发 -> 验证 -> 发布 -> 生命周期管理
- **[[IPD]]+CMMI**：在IPD基础上增加CMMI过程域，覆盖项目管理、需求管理、配置管理等

### 项目四算（概预核决）

| 环节 | 定位 | 作用 |
|------|------|------|
| 概算 | 设计利润 | 80%项目成本在此阶段确定 |
| 预算 | 管理增收节支 | 基于概算，设定财务基准 |
| 核算 | 温度计 | 准确记录历史、描述现在、预测未来 |
| 决算 | 秋后算账 | 经验教训总结，改善后续运作，刷新基线 |

## 六、组织级项目管理体系

### 三层管控

1. **项目组合**（代表处）：客户组合和产品组合管理，优化资源配置
2. **项目集**（系统部）：对经营目标实现和客户满意度负责
3. **项目**（项目组）：基于契约开展优质高效交付

### 成熟度模型（5级）

| 级别 | 描述 |
|------|------|
| 1 初级 | 不具备或未使用项目管理技能 |
| 2 推行 | 有基本技能，偶尔使用，不连贯 |
| 3 运行 | 技能健全并全面运行，标准统一流程 |
| 4 集成 | 技能娴熟，高度集成，持续使用，结果可预测 |
| 5 领先 | 世界领先水平，业界最佳实践 |

### 项目健康七大关键

工作和进度可预测、范围可实现可管理、实现业务收益、相关方做出承诺、团队高绩效、风险可规避、项目财经管理良好。

## 七、大项目运作方法论

### 营销四要素

1. 客户关系
2. 解决方案
3. 交付
4. 商务

### 关键要点

- 大项目拓展是**科学**而非艺术，可复制可学习
- "三板斧和一指禅"：每个阶段有核心突破点
- 流程的威力：标准化、可复制
- 个人战斗力与团队战斗力并重

## 九、项目HRBP培养

华为项目HRBP角色的转变：
- 从V-cross模型（6个角色认知）转向"人"字模型
- **人字模型三要素**：责权管理、资源管理、评价管理
- 核心要求：先于业务流程准备HR动作，在业务流程每个节点自发启动人力资源适配

### 培养路径

1. 理论学习：项目管理知识体系
2. 实战轮岗：跨部门、跨区域岗位轮换
3. 认证评估：知识、技能、行为、素质四维认证
