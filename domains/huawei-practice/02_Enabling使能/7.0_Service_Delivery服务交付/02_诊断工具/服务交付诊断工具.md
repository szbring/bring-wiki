# 服务交付诊断工具

<!-- chunk: 服务交付诊断工具.成熟度评估 -->
## 成熟度评估

| 级别 | 特征 |
|---|---|
| L1 | 项目全靠经验，无标准流程，质量不稳定，延期常见 |
| L2 | 有基本项目管理，但工具不统一，知识不沉淀 |
| L3 | 标准化交付流程，里程碑清晰，质量有检查点 |
| L4 | 数据驱动项目管理，资源利用率可量化，预测准确 |
| L5 | 交付能力成为竞争优势，客户NPS持续领先 |

---

<!-- chunk: 服务交付诊断工具.快速诊断问题 -->
## 快速诊断问题

1. **「你们的项目按时按质完成的比例是多少？」**
   - 不知道/低于60% → L1-L2
   - 60-80% → L2-L3
   - 80%以上 → L3+

2. **「一个新顾问加入项目，需要多长时间能独立上手？」**
   - 一两个月靠老带新 → 知识没有系统化沉淀
   - 有手册但不完整 → L2
   - 有完整onboarding和项目手册 → L3+

3. **「项目结束后，有没有系统性的复盘和知识沉淀？」**
   - 基本没有 → L1-L2
   - 有复盘但不落文字 → L2
   - 有标准复盘模板且沉淀到知识库 → L3+

4. **「客户满意度如何收集和跟踪？」**
   - 靠感觉判断 → L1
   - 有问卷但没系统追踪 → L2
   - 有NPS/CSAT且定期回顾 → L3+
