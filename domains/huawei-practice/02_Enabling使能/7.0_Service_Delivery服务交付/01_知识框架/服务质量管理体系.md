# 服务质量管理体系

服务质量是交付体系的核心保障。华为基于ITIL和ISO20000，结合自身交付实践，建立了覆盖服务全生命周期的质量管理体系。

---

<!-- chunk: 服务质量管理体系.一-服务质量框架 -->
## 一、服务质量框架

### 1.1 三层质量保障

| 层级 | 内容 | 责任人 |
|---|---|---|
| 事前预防 | 标准化流程、评审机制、能力认证 | 质量经理/PM |
| 事中控制 | 过程审计、里程碑评审、缺陷管理 | 质量经理 |
| 事后改进 | 客户满意度、复盘改进、经验固化 | PM/质量部 |

### 1.2 质量管理PDCA循环

```
Plan（策划）→ Do（执行）→ Check（检查）→ Act（改进）
    ↑                                      │
    └──────────── 持续循环 ←──────────────┘
```

## 二、服务质量指标体系

### 2.1 过程质量指标

| 指标 | 定义 | 目标值 | 计算方法 |
|---|---|---|---|
| 里程碑按时达成率 | 里程碑按计划完成的比例 | ≥90% | 按时完成数/总里程碑数 |
| 评审通过率 | 评审一次通过的比例 | ≥85% | 一次通过数/评审总数 |
| 缺陷密度 | 每千行代码/每百页文档的缺陷数 | 行业对标 | 缺陷数/规模单位 |
| 变更率 | 范围变更占总需求的比例 | ≤15% | 变更需求数/初始需求数 |

### 2.2 交付质量指标

| 指标 | 定义 | 目标值 | 计算方法 |
|---|---|---|---|
| 交付物合格率 | 客户验收合格的比例 | ≥95% | 合格交付物数/总交付物数 |
| 验收一次通过率 | 客户一次验收通过的比例 | ≥80% | 一次通过数/验收总数 |
| 交付延期率 | 超计划交付的比例 | ≤10% | 延期项目数/总项目数 |
| 客户投诉率 | 客户投诉占总项目的比例 | ≤5% | 投诉项目数/总项目数 |

### 2.3 客户满意度指标

| 指标 | 定义 | 目标值 | 采集方式 |
|---|---|---|---|
| CSAT | 客户满意度评分 | ≥8分（10分制） | 项目验收后调查 |
| NPS | 净推荐值 | ≥30 | 项目验收后调查 |
| 重复购买率 | 客户续约/增购比例 | ≥70% | 年度统计 |
| 客户留存率 | 客户持续合作比例 | ≥85% | 年度统计 |

## 三、质量评审体系

### 3.1 评审类型

| 评审类型 | 目的 | 时机 | 参与人 |
|---|---|---|---|
| 同行评审 | 技术质量保障 | 交付物发出前 | 技术专家 |
| 里程碑评审 | 阶段质量把关 | 里程碑节点 | PM+质量经理+客户 |
| 管理评审 | 项目健康度评估 | 月度 | PMO+管理层 |
| 客户评审 | 客户确认验收 | 验收节点 | 客户+项目团队 |

### 3.2 评审流程

```
评审准备 → 评审执行 → 缺陷记录 → 缺陷修复 → 修复验证 → 评审关闭
```

### 3.3 评审原则

| 原则 | 说明 |
|---|---|
| 越早越好 | 问题发现越早，修复成本越低 |
| 零缺陷出口 | 交付物发出前必须通过评审 |
| 独立评审 | 评审人不能是交付物作者 |
| 数据驱动 | 用评审数据指导改进 |

## 四、持续改进机制

### 4.1 改进来源

| 来源 | 触发条件 | 改进动作 |
|---|---|---|
| 项目复盘 | 项目结束 | 总结经验教训，更新知识库 |
| 缺陷分析 | 缺陷趋势异常 | 根因分析，流程改进 |
| 客户反馈 | CSAT/NPS下降 | 识别痛点，服务改进 |
| 审计发现 | 质量审计 | 纠正措施，预防措施 |
| 行业对标 | 定期对标 | 差距分析，目标调整 |

### 4.2 改进闭环

```
发现问题 → 根因分析 → 制定措施 → 执行改进 → 效果验证 → 固化流程
    ↑                                          │
    └──────────── 持续循环 ←──────────────────┘
```

### 4.3 知识沉淀

| 知识类型 | 沉淀方式 | 应用场景 |
|---|---|---|
| 项目经验 | 复盘报告入知识库 | 新项目参考 |
| 最佳实践 | 标准化成模板/Checklist | 流程复用 |
| 失败教训 | 案例化并培训传播 | 避免重犯 |
| 质量标准 | 更新质量基线 | 评审依据 |

## 五、ITR（从问题到解决）关联

服务质量体系与[[ITR]]流程紧密关联，形成服务交付的完整闭环：

| [[ITR]]阶段 | 质量关联 |
|---|---|
| 问题受理 | 问题分级、SLA确认 |
| 问题处理 | 根因分析、解决方案质量 |
| 问题解决 | 解决验证、客户确认 |
| 问题关闭 | 满意度确认、知识沉淀 |
