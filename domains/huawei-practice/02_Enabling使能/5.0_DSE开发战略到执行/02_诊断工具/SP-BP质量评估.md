# SP/BP 质量评估

<!-- chunk: sp-bp质量评估.一句话定义 -->
## 一句话定义

对企业的 SP（中长期[[DSTE]]）和 BP（年度业务计划）进行内容质量与流程质量双维评估，识别战略规划文档"好不好"和战略规划流程"对不对"。

---

<!-- chunk: sp-bp质量评估.一-为什么需要-sp-bp-质量评估 -->
## 一、为什么需要 SP/BP 质量评估

| 常见问题 | 后果 | 评估价值 |
|---------|------|---------|
| SP空话套话多 | 无法指导BP制定 | 检查SP内容是否具体可操作 |
| BP变成工作罗列 | 战略无法落地 | 检查BP是否承接SP |
| SP和BP两张皮 | 资源配置偏离战略 | 检查SP-BP衔接质量 |
| 年年做年年不变 | 战略管理走形式 | 检查流程是否产生实质改变 |

<!-- chunk: sp-bp质量评估.二-sp-质量评估 -->
## 二、SP 质量评估

### 2.1 评估维度与评分

| # | 评估维度 | 1分（不合格） | 3分（合格） | 5分（优秀） | 得分 |
|---|---------|-------------|------------|------------|------|
| 1 | 差距分析 | 无差距分析或仅列数据 | 有差距识别但缺乏根因 | 差距量化+根因+改善方向 | |
| 2 | 市场洞察 | 泛泛而谈行业趋势 | 有数据有观点 | 五看完整+结论明确+数据支撑 | |
| 3 | 战略意图 | "成为行业领先"等空话 | 有方向但不够量化 | 3年目标量化+必赢战役明确 | |
| 4 | 业务设计 | 未涉及 | 有描述但不完整 | 五要素齐全且有竞争力分析 | |
| 5 | 创新焦点 | 无创新规划 | 有方向但无资源匹配 | 突破方向+资源配置+时间表 | |
| 6 | 里程碑 | 无里程碑或仅有年度大目标 | 有年度节点 | 季度级里程碑+可追踪 | |
| 7 | 纵向对齐 | 各层级SP各自为政 | 有向下传达但无双向沟通 | 上下双向确认+横向协同 | |
| 8 | 可执行性 | SP无法指导BP制定 | 部分可指导 | BP可直接承接SP展开 | |

**SP总分**：____/40分

**评分解读**：
- 32-40分：高质量SP，可有效指导BP
- 20-31分：基本合格，部分环节需强化
- 20分以下：SP质量不达标，需重新制定

### 2.2 SP 典型问题检查清单

- [ ] 战略意图是否用了"领先"、"一流"等无法量化的词？
- [ ] 市场洞察是否只说了"行业在增长"而没有结构化分析？
- [ ] 业务设计是否只描述了现状而非目标状态？
- [ ] 是否有明确的"不做什么"的选择？
- [ ] 战略举措是否太多（超过8项），失去聚焦？
- [ ] 是否有对上年度SP执行偏差的反思？

<!-- chunk: sp-bp质量评估.三-bp-质量评估 -->
## 三、BP 质量评估

### 3.1 评估维度与评分

| # | 评估维度 | 1分（不合格） | 3分（合格） | 5分（优秀） | 得分 |
|---|---------|-------------|------------|------------|------|
| 1 | SP承接 | 与SP无关 | 部分承接SP | 每个优先行动可追溯至SP | |
| 2 | 优先行动 | 罗列日常工作 | 有重点但太多 | 3-5个战略优先行动 | |
| 3 | KPI设计 | KPI与战略无关 | 部分对齐 | KPI完整承接优先行动+SMART | |
| 4 | 资源匹配 | 预算与战略脱节 | 基本匹配 | 战略优先项资源优先保障 | |
| 5 | 组织配套 | 未考虑组织调整 | 有考虑但不具体 | 组织调整方案+时间表 | |
| 6 | 人才配套 | 未考虑人才需求 | 有计划但不具体 | 关键岗位盘点+招聘/培养计划 | |
| 7 | 风险预案 | 无风险识别 | 有风险但无应对 | 风险清单+应对措施+触发条件 | |
| 8 | 可追踪性 | 无法追踪进展 | 有月度检查 | 里程碑+红黄绿灯+QBR机制 | |

**BP总分**：____/40分

**评分解读**：
- 32-40分：高质量BP，执行基础扎实
- 20-31分：基本合格，需强化关键环节
- 20分以下：BP质量不达标，无法有效执行

### 3.2 BP 典型问题检查清单

- [ ] 优先行动是否只是"正常工作的加总"，没有战略突破性？
- [ ] KPI是否太多（超过8个），什么都重要等于什么都不重要？
- [ ] 是否缺少领先型KPI（只有滞后型财务指标）？
- [ ] 预算分配是否"撒胡椒面"，战略优先项没有额外资源？
- [ ] 是否缺少跨部门协同的机制设计？
- [ ] 个人PBC是否与BP中的KPI关联？

## 四、SP-BP 衔接质量评估

### 4.1 衔接评估矩阵

| SP中的战略意图 | 对应的BP优先行动 | 对应的KPI | 资源匹配 | 衔接评价 |
|---------------|-----------------|----------|---------|---------|
| 战略意图1 | ？ | ？ | ？ | 完整/部分/断裂 |
| 战略意图2 | ？ | ？ | ？ | 完整/部分/断裂 |
| ... | | | | |

### 4.2 衔接断裂的信号

- SP中的战略意图在BP中找不到对应行动 → **解码缺失**
- BP中有大量行动但与SP无关 → **BP自说自话**
- KPI存在但无法追溯到SP → **KPI脱节**
- 预算没有向SP中的优先方向倾斜 → **资源错配**

<!-- chunk: sp-bp质量评估.五-流程质量评估 -->
## 五、流程质量评估

除了文档内容，还需评估SP/BP制定流程本身的质量：

| 流程维度 | 1分 | 3分 | 5分 |
|---------|-----|-----|-----|
| 参与度 | 仅战略部编写 | 管理层参与讨论 | 高管团队深度参与 |
| 方法论 | 无标准方法论 | 有方法但执行不一致 | 统一方法论（BLM/BEM） |
| 节奏性 | 想起来才做 | 有大致时间表 | 严格按年度节奏执行 |
| 评审质量 | 无评审或走过场 | 有评审但缺实质讨论 | 评审有决策有输出 |
| 持续性 | 做完就归档 | 年度回顾 | 滚动更新+季度追踪 |

<!-- chunk: sp-bp质量评估.六-评估交付物 -->
## 六、评估交付物

评估完成后交付以下内容：

1. **SP/BP质量评分表**（量化打分）
2. **SP-BP衔接矩阵**（可视化断裂点）
3. **TOP5改进建议**（按影响度排序）
4. **标杆对比**（与行业领先企业的差距）
