# 华为战略解码工具BEM

> 业务执行模型（Business Execution Model）：从战略到年度业务计划的完整解码路径

---

<!-- chunk: 华为战略解码工具bem.一-bem的背景与定位 -->
## 一、BEM的背景与定位

### 1.1 战略执行的核心难题

麦肯锡研究表明，企业制定的战略中，超过70%无法有效执行。失败的原因往往不是战略方向错了，而是战略与执行之间存在巨大的"鸿沟"：
- 战略停留在PPT层面，没有转化为可操作的行动
- 战略目标无法分解到部门和个人
- 战略执行进展无法有效追踪和评估

### 1.2 BEM的核心价值

**BEM（Business Execution Model，业务执行模型）**是华为DSTE（Develop Strategy to Execute）体系中战略解码的核心工具。它解决的核心问题是：

> **如何将3-5年的中长期战略（SP），系统性地拆解为当年可执行的年度业务计划（BP），并确保执行过程可追踪？**

BEM本质上是一个**战略解码框架**，帮助企业在战略与执行之间建立清晰的逻辑链条：

```
战略意图（Strategic Intent）
     ↓ BEM解码
战略举措（Strategic Initiatives）
     ↓
年度优先项（Annual Priorities）
     ↓
部门KPI和行动计划
     ↓
个人PBC（绩效承诺）
```

### 1.3 BEM与BLM的关系

BEM是BLM（Business Leadership Model，业务领导力模型）的配套工具：
- **BLM**：用于战略分析和战略制定（回答"战略是什么"）
- **BEM**：用于战略解码和执行落地（回答"战略怎么执行"）

两者是"一套牌"，通常在年度战略规划（SP/BP）流程中配套使用。

## 二、BEM的框架结构

### 2.1 BEM的核心模块

BEM框架包含以下六个核心模块，形成一个完整的战略解码系统：

```
┌─────────────────────────────────────────────────────────────┐
│                     BEM业务执行模型                          │
├──────────────┬──────────────────────┬───────────────────────┤
│  战略意图    │    关键成功要素       │    业务设计           │
│  (Strategic  │    (CSF)             │    (Business Design)  │
│   Intent)    │                      │                       │
├──────────────┼──────────────────────┼───────────────────────┤
│  优先行动    │    组织与人才         │    氛围与文化          │
│  (Priority   │    (Org & Talent)    │    (Climate &         │
│   Actions)   │                      │     Culture)          │
└──────────────┴──────────────────────┴───────────────────────┘
```

### 2.2 模块一：战略意图（Strategic Intent）

**核心问题**：我们未来3-5年要达到什么目标状态？

战略意图是整个BEM的出发点，需要清晰描述：
- **业务愿景**：未来3-5年要达到的业务状态（市场地位、收入规模、技术领先性等）
- **差距识别（Gap Analysis）**：当前状态与目标状态之间的核心差距在哪里
- **优先方向**：基于差距，未来1-3年最重要的战略方向是什么

**关键工具**：
- **SWOT分析**：识别内外部优劣势和机会威胁
- **差距分析**（Gap Chart）：量化当前位置与目标位置的差距
- **战略优先级矩阵**：基于重要性×紧迫性对战略方向排序

**华为实践**：
华为在年度SP（战略规划）中，要求各业务部门明确陈述"3年战略意图"——不是宏大叙事，而是具体的、可量化的目标描述，例如："3年内在中东非洲企业市场进入前三，收入达到XX亿美元，市场份额达到XX%。"

---

### 2.3 模块二：关键成功要素（Critical Success Factors, CSF）

**核心问题**：要实现战略意图，必须在哪些关键事项上做得出色？

CSF是战略意图的"必要条件"分析——实现目标必须做好哪几件事，少了哪一件都会失败。

**CSF的识别方法**：
1. 列出实现战略意图的所有重要因素（Brainstorming）
2. 筛选出"必要"因素（少了这个，目标一定无法实现）
3. 通常将CSF限定在3-5个，避免"什么都重要"

**CSF示例（以企业软件公司为例）**：

| # | 关键成功要素 | 当前状态 | 目标状态 |
|---|------------|---------|---------|
| 1 | 产品竞争力（核心功能领先） | 功能与竞品持平 | 3个核心场景功能领先 |
| 2 | 销售能力（行业解决方案销售） | 通用产品销售为主 | 建立3个行业顾问型销售团队 |
| 3 | 合作伙伴生态 | 仅5家集成商合作 | 建立50家认证合作伙伴网络 |
| 4 | 客户成功能力 | 客户续约率65% | 提升至85%以上 |

---

### 2.4 模块三：业务设计（Business Design）

**核心问题**：我们用什么样的商业模式和业务方式实现战略意图？

业务设计是对商业模式的核心要素进行明确和优化：

**业务设计的五要素**：
1. **目标客户**：主要服务哪些细分客户群体，优先级如何？
2. **价值主张**：为目标客户提供什么独特价值？差异化在哪里？
3. **盈利模式**：通过什么方式盈利？收入结构是什么？
4. **战略控制点**：在价值链上哪个环节具有不可替代性？
5. **关键资源**：支撑业务模式所需的关键资源和能力是什么？

**与战略意图的联动**：
业务设计需要回答：现有的商业模式能否支撑战略意图的实现？如果不能，商业模式需要哪些调整？

---

### 2.5 模块四：优先行动（Priority Actions）

**核心问题**：今年（当期）最重要的3-5件事是什么？

优先行动是BEM的"执行核心"，将战略意图转化为当期可执行的重点工作：

**优先行动的定义标准**：
- 直接支撑CSF的实现
- 在当期（年度）内可以启动并取得阶段性成果
- 有明确的负责人和可量化的完成标准
- 资源已经或可以落实到位

**优先行动的格式要求**（华为标准）：

```
优先行动示例：
行动名称：建立行业解决方案销售团队
战略连接：支撑CSF#2（销售能力提升）
目标成果：Q4前完成3个行业（制造/零售/金融）销售团队组建
           和首批销售人员的行业能力认证
负责人：销售VP
关键里程碑：
  Q1：完成岗位设计和招聘计划
  Q2：完成招募和入职培训
  Q3：完成行业能力认证
  Q4：首个季度行业销售目标完成率≥80%
```

**避免"假优先行动"**：
优先行动的常见误区是把日常工作包装成"优先行动"——这样的"行动"没有突破性，对战略无实质推动。真正的优先行动必须是**今年必须做、不做就无法实现战略意图**的事情。

---

### 2.6 模块五：组织与人才（Organization & Talent）

**核心问题**：实现战略意图，组织和人才是否做好了准备？

这个模块关注两个子问题：

**组织设计是否匹配战略？**
- 组织结构是否支持战略执行（例如：战略要进入新市场，是否成立了对应的业务单元？）
- 关键岗位是否有合适的人担任？
- 决策机制是否有助于快速响应市场变化？

**人才能力是否满足战略需求？**
- 战略执行需要什么关键能力？
- 当前人才储备有哪些能力缺口？
- 如何通过招聘、培养、外部合作填补缺口？

华为在这个模块特别强调**"关键岗位盘点"**——对于支撑战略的核心岗位，必须确认有合适人选，并制定备份方案。

---

### 2.7 模块六：氛围与文化（Climate & Culture）

**核心问题**：组织文化和内部氛围是否支持战略执行？

这是BEM中最软性但往往最关键的模块：

**评估维度**：
- 员工是否理解并认同公司战略方向？
- 组织内部是否有支持创新和冒险的文化？
- 是否存在阻碍战略执行的文化障碍（例如：官僚主义、部门壁垒、短期主义）？
- 领导层是否以身作则地体现了期望的文化价值观？

**华为实践**：
在战略变革时期，任正非会专门针对文化阻力发表内部演讲（即"总裁电视电话会"），通过领导层的高度重视和持续沟通，打破文化惰性。

## 三、BEM在DSTE流程中的应用节奏

### 3.1 年度SP/BP流程中的BEM应用

```
时间线         活动                    BEM模块
─────────────────────────────────────────────────────
7-8月         战略研讨（SP Workshop）  战略意图 + CSF + 业务设计
─────────────────────────────────────────────────────
9月           战略意图对齐             战略意图定稿
─────────────────────────────────────────────────────
10月          优先行动规划             优先行动 + 组织人才
─────────────────────────────────────────────────────
11月          BP（年度业务计划）制定   全模块整合，数字化落地
─────────────────────────────────────────────────────
12月          BP审批与发布             领导层审批，全员宣贯
─────────────────────────────────────────────────────
次年Q1        执行启动                 PBC设定与对齐
─────────────────────────────────────────────────────
次年Q2/Q3/Q4  季度回顾（QBR）          优先行动进展追踪
```

### 3.2 BEM的季度追踪机制

优先行动不是"一设了之"，需要建立季度追踪机制：

**季度业务回顾（QBR，Quarterly Business Review）**
- 回顾上季度优先行动完成情况
- 分析达成或未达成的原因
- 调整下季度的重点和资源配置
- 识别新出现的战略威胁或机会

## 四、BEM实施常见问题

### 问题1：BEM走形式，与日常工作脱节

**表现**：年初做了BEM，定了优先行动，但日常工作仍按照原来方式运转，无人追踪优先行动进展。

**解决方案**：
- 将优先行动的里程碑纳入管理例会议程
- 指定每个优先行动有专属负责人和追踪人
- 季度QBR中，优先行动进展是核心议题

### 问题2：优先行动太多，失去聚焦

**表现**：列出了20+个"优先"行动，结果什么都是优先，等于什么都不优先。

**解决方案**：
- 严格控制优先行动数量（通常3-5个）
- 引入"If-Then"原则：如果完成了这个行动，就能显著推进CSF；如果没有这个行动，CSF一定无法实现
- 高层必须做取舍，明确"不做什么"

### 问题3：战略意图不清晰，导致CSF和优先行动跑偏

**表现**：战略意图描述模糊（如"成为行业领先者"），导致无法有效推导CSF。

**解决方案**：
- 战略意图必须量化（具体到市场份额、收入规模、客户数量等）
- 进行差距分析（Gap Analysis），明确当前状态与目标状态的具体差距
- 用"3年后我们在哪里"的具体场景描述来明确战略意图

### 问题4：BEM是领导层的工具，基层员工不了解

**表现**：BEM做完后只在高层流传，普通员工不知道公司今年最重要的事是什么。

**解决方案**：
- 将BEM的核心结论（战略意图+优先行动）进行全员宣贯
- 确保每个员工的个人PBC（绩效承诺）能够追溯到BEM的某个优先行动
- 建立"战略可视化墙"，让战略在日常工作环境中可见

## 五、薄云咨询的BEM辅导方法

### 5.1 导入BEM的典型项目设计

**阶段1：诊断（2-3周）**
- 访谈高管，了解现有战略规划流程
- 评估战略执行的主要障碍
- 分析历年SP/BP与实际执行的偏差

**阶段2：首次BEM工作坊（1-2天）**
- 教学BEM框架和方法
- 引导核心管理团队完成战略意图对齐
- 产出：初步的战略意图和CSF草稿

**阶段3：深化与完善（4-6周）**
- 各模块深化（优先行动规划、人才盘点等）
- 跨部门对齐和资源协商
- 产出：完整BEM框架和年度BP

**阶段4：执行机制建立（持续）**
- 建立季度QBR机制
- 培训内部战略规划骨干
- 辅导第一次完整的QBR执行

### 5.2 BEM的快速诊断问卷

在与客户沟通时，可以用以下问题快速评估战略解码成熟度：

1. 贵公司是否有明确的3-5年战略目标？能用一句话说清楚吗？
2. 今年公司最重要的3件事是什么？（考察是否有清晰的优先行动）
3. 您的团队目标是否明确对应到公司战略的某个方向？
4. 季度末会专门回顾战略执行进展吗？
5. 员工是否知道公司今年最重要的战略目标是什么？

如果大多数问题的答案是"不确定"或"没有"，则说明战略解码体系有重大改进空间。
