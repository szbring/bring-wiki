# DSTE 流程架构 — 开发战略到执行

<!-- chunk: dste流程架构.一句话定义 -->
## 一句话定义

DSTE（Develop Strategy to Execute）是华为战略管理的端到端流程体系，覆盖战略规划、战略解码、战略执行与监控、战略评估四个阶段，形成"规划—解码—执行—评估"的完整闭环。

---

<!-- chunk: dste流程架构.一-dste-的总体架构 -->
## 一、DSTE 的总体架构

```
┌─────────────────────────────────────────────────────────────────┐
│                     DSTE 端到端战略管理流程                       │
├──────────────┬───────────────┬──────────────┬──────────────────┤
│  战略规划     │  战略解码      │  战略执行     │  战略评估        │
│  (SP)        │  (Decode)     │  (Execute)   │  (Evaluate)     │
├──────────────┼───────────────┼──────────────┼──────────────────┤
│ 市场洞察      │ 战略举措分解   │ BP执行与监控  │ 战略复盘         │
│ 战略意图      │ KPI/PBC设定   │ 资源动态调配  │ 绩效评价         │
│ 创新焦点      │ 年度BP制定    │ 经营分析会    │ 战略调整         │
│ 业务设计      │ 预算分配      │ 管理闭环      │ 经验沉淀         │
└──────────────┴───────────────┴──────────────┴──────────────────┘
       4-8月                    9-10月              次年1-12月        次年12月-后年1月
```

DSTE 不是一个"单次活动"，而是一个**年度循环、滚动迭代**的管理节奏。华为每年从 4 月启动战略规划，到次年完成评估，形成完整的 PDCA 闭环。

## 二、阶段一：战略规划（SP，Strategic Planning）

### 2.1 SP 的定位与周期

SP 回答的核心问题：**未来 3-5 年，我们要去哪里？怎么去？**

- 周期：3-5 年滚动规划，每年更新
- 启动时间：每年 4-8 月
- 主要工具：[[BLM]]（业务领先模型）

### 2.2 SP 的核心产出

| 产出 | 内容 | 责任主体 |
|------|------|---------|
| 市场洞察报告 | 行业趋势、竞争格局、客户需求变化 | 战略部/市场部 |
| 战略意图描述 | 3-5 年愿景、战略目标、必赢战役 | CEO + 高管团队 |
| 业务设计 | 客户选择、价值主张、盈利模式、战略控制点 | 各 BU 负责人 |
| 创新焦点 | 需要突破的领域和资源配置方向 | CEO + CTO |

### 2.3 SP 制定的关键步骤

1. **差距分析**：审视上期战略执行结果，识别业绩差距与机会差距
2. **市场洞察**：五看（看行业、看市场、看竞争、看自己、看机会）
3. **战略意图设定**：基于洞察和差距，明确 3-5 年战略方向
4. **创新焦点选择**：确定资源配置的优先领域
5. **业务设计优化**：围绕战略意图重新审视和调整商业模式

### 2.4 华为实践要点

华为要求 SP 必须做到"三个对齐"：
- **上下对齐**：公司 SP → BU SP → 区域 SP，层层承接
- **左右对齐**：各 BU 之间的战略协同，避免资源内耗
- **内外对齐**：战略方向与外部市场变化保持一致

## 三、阶段二：战略解码（Decode）

### 3.1 战略解码的定位

战略解码回答的核心问题：**战略如何转化为每个人可执行的目标？**

- 时间：每年 9-10 月，承接 SP 制定 BP
- 主要工具：[[BEM]]（业务执行模型）
- 核心逻辑：SP → CSF（关键成功要素）→ 战略举措 → 年度优先项 → KPI/PBC

### 3.2 解码的四个层级

```
第一层：公司战略意图
       ↓
第二层：关键成功要素（CSF，3-5个）
       ↓
第三层：战略举措与年度优先项（每个CSF展开2-3个举措）
       ↓
第四层：部门KPI + 个人PBC（绩效承诺）
```

### 3.3 战略解码的核心产出

| 产出 | 内容 | 责任主体 |
|------|------|---------|
| CSF 清单 | 实现战略意图的关键成功要素 | 高管团队 |
| 战略举措 | 每个CSF对应的行动方案 | 各部门负责人 |
| 年度KPI | 可量化的关键绩效指标 | 战略部 + HR |
| 个人PBC | 个人绩效承诺书 | 全体员工 |

### 3.4 解码质量的关键标准

- **可衡量**：每个KPI必须有量化目标和衡量方法
- **可追溯**：个人PBC能够追溯到某个战略举措，战略举措追溯到某个CSF
- **可追踪**：有明确的里程碑和检查点
- **有聚焦**：KPI数量控制在合理范围（公司级5-8个，部门级3-5个）

## 四、阶段三：战略执行与监控（Execute & Monitor）

### 4.1 执行监控的定位

战略执行监控回答的核心问题：**战略在执行中是否偏航？如何纠偏？**

- 时间：全年持续
- 核心机制：经营分析会 + 战略执行追踪看板

### 4.2 执行监控的三大机制

**机制一：经营分析会（月度/季度）**
- 月度经营分析：跟踪KPI达成和关键举措进展
- 季度战略回顾（QBR）：评估战略执行偏差，进行资源调配
- 半年战略审视：基于环境变化评估战略是否需要调整

**机制二：战略执行追踪看板**
- 可视化展示各战略举措的进展状态（红/黄/绿）
- 关键里程碑达成率
- 风险预警和升级机制

**机制三：红黄绿灯管理**

| 状态 | 定义 | 管理动作 |
|------|------|---------|
| 绿灯 | 进展正常，按计划推进 | 常规跟踪 |
| 黄灯 | 存在风险，可能延期 | 升级至部门负责人，制定应对方案 |
| 红灯 | 严重偏离，无法按期完成 | 升级至高管层，启动专项整改 |

### 4.3 执行中的动态调整

战略不是一成不变的。华为强调"战略是打出来的，不是规划出来的"，执行中需要根据市场变化动态调整：

- **微调**：措施和方法调整，目标不变（季度QBR中决策）
- **中调**：部分目标调整，战略方向不变（半年审视中决策）
- **大调**：战略方向变更，需启动SP修订流程（需EMT审批）

## 五、阶段四：战略评估（Evaluate）

### 5.1 战略评估的定位

战略评估回答的核心问题：**战略执行的结果如何？学到了什么？**

- 时间：年底至次年初
- 核心活动：战略复盘 + 绩效评价

### 5.2 战略评估的三个维度

| 维度 | 评估内容 | 方法 |
|------|---------|------|
| 目标达成 | KPI是否完成？差距在哪里？ | 定量分析 |
| 过程质量 | 战略制定和解码过程是否科学？ | 过程审计 |
| 能力提升 | 组织战略管理能力是否进步？ | 成熟度评估 |

### 5.3 战略复盘的输出

- 上年度战略执行总结报告
- 关键成功经验和失败教训
- 下年度SP制定的输入（差距分析的起点）
- 战略管理流程优化建议

## 六、DSTE 流程的时间节奏

| 月份 | 主要活动 | 关键产出 |
|------|---------|---------|
| 4-5月 | 战略规划启动，市场洞察 | 市场洞察报告 |
| 6-7月 | SP 研讨与制定 | SP 初稿 |
| 8月 | SP 评审与定稿 | SP 定稿 |
| 9-10月 | 战略解码，BP 制定 | BP + KPI + PBC |
| 11-12月 | 预算分配，执行准备 | 年度预算 |
| 次年1-12月 | 战略执行与监控 | QBR 报告 |
| 次年12月 | 战略评估与复盘 | 战略复盘报告 |

## 八、薄云咨询的 DSTE 应用场景

**场景一：DSTE 体系从零建立**
- 适用于：从未有过系统化战略管理流程的企业
- 切入点：先做 SP/BP 流程，再逐步建设监控和评估机制
- 周期：6-12 个月

**场景二：战略执行断层修复**
- 适用于：有战略规划但无法落地的企业
- 切入点：重点加强战略解码（[[BEM]]）和执行追踪机制
- 周期：3-6 个月

**场景三：战略管理成熟度提升**
- 适用于：已有基础流程但执行质量不高的企业
- 切入点：优化 DSTE 各阶段的关键输出质量
- 周期：持续陪跑
