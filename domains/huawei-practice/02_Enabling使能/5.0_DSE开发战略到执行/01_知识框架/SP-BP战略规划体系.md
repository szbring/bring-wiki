# SP/BP 战略规划体系

<!-- chunk: sp-bp战略规划体系.一句话定义 -->
## 一句话定义

SP（Strategic Plan，中长期战略规划）与 BP（Business Plan，年度业务计划）构成华为战略规划的"双轮驱动"——SP 定方向（3-5 年），BP 定行动（1 年），确保战略既有长远视野又有当期落点。

---

<!-- chunk: sp-bp战略规划体系.一-sp-与-bp-的定位与关系 -->
## 一、SP 与 BP 的定位与关系

```
┌────────────────────────────┐     ┌────────────────────────────┐
│    SP 中长期战略规划        │     │    BP 年度业务计划          │
│    (3-5年)                 │     │    (1年)                    │
├────────────────────────────┤     ├────────────────────────────┤
│ · 使命与愿景               │ ←── │ · 年度目标与KPI            │
│ · 战略方向与业务组合        │ 承接 │ · 重点项目与资源分配        │
│ · 核心能力建设路径          │ ──→ │ · 组织调整与人才计划        │
│ · 关键里程碑（年度分解）    │ 约束 │ · 预算与经营目标           │
└────────────────────────────┘     └────────────────────────────┘
         4-8月制定                         9-11月制定
```

**核心关系**：SP 是 BP 的"天花板"——BP 不能偏离 SP 的方向；BP 是 SP 的"地基层"——SP 的目标必须通过年度 BP 来兑现。

<!-- chunk: sp-bp战略规划体系.二-sp-中长期战略规划 -->
## 二、SP（中长期战略规划）

### 2.1 SP 的核心内容框架

| 模块 | 核心问题 | 关键产出 |
|------|---------|---------|
| 差距分析 | 我们现在在哪？差距在哪？ | 业绩差距 + 机会差距清单 |
| 市场洞察 | 外部环境将发生什么变化？ | 五看分析报告 |
| 战略意图 | 3-5年后我们要到哪里？ | 愿景、战略目标、必赢战役 |
| 创新焦点 | 哪些领域需要突破性创新？ | 创新方向与资源配置策略 |
| 业务设计 | 用什么模式实现战略？ | 客户选择、价值主张、盈利模式、战略控制点 |
| 战略里程碑 | 关键节点如何设定？ | 3-5年里程碑计划 |

### 2.2 SP 的制定流程

```
步骤1：差距审视
  │   审视上年度SP执行偏差，识别核心差距
  ↓
步骤2：市场洞察（五看）
  │   看行业/看市场/看竞争/看自己/看机会
  ↓
步骤3：战略意图研讨
  │   高管团队对齐3-5年战略方向
  ↓
步骤4：业务设计优化
  │   调整客户选择、价值主张、盈利模式
  ↓
步骤5：创新焦点确定
  │   明确资源配置优先级和突破方向
  ↓
步骤6：SP评审与发布
      EMT（执行管理团队）评审，正式发布
```

### 2.3 SP 制定的关键原则

- **不追求完美**：SP 不需要100%精确预测未来，而是建立方向性共识和灵活应变能力
- **滚动更新**：每年审视更新，而非每3-5年重写一次
- **上下贯通**：公司SP → 各BU SP → 各区域SP，确保纵向对齐
- **横向协同**：BU之间的依赖关系和协同点必须明确

## 三、BP（年度业务计划）

### 3.1 BP 的核心内容框架

| 模块 | 核心问题 | 关键产出 |
|------|---------|---------|
| 年度目标 | 今年要达成什么？ | 营收/利润/市占率等量化目标 |
| 战略举措落地 | SP中的战略今年做什么？ | 年度TOP5优先事项 |
| KPI设定 | 如何衡量成功？ | 公司级/部门级KPI体系 |
| 资源计划 | 需要什么资源？ | 人力/财务/技术资源计划 |
| 组织与人才 | 组织需要怎样调整？ | 组织调整方案、人才计划 |
| 预算 | 钱怎么花？ | 年度预算表 |

### 3.2 BP 的制定流程

```
步骤1：SP回顾与目标分解
  │   将SP的年度里程碑转化为当年目标
  ↓
步骤2：BEM战略解码
  │   CSF提取 → 战略举措 → 优先行动
  ↓
步骤3：KPI设定与分解
  │   公司级KPI → 部门KPI → 个人PBC
  ↓
步骤4：资源与预算匹配
  │   根据优先级分配预算和人力
  ↓
步骤5：BP评审与审批
      预算委员会审批，正式下达
```

### 3.3 BP 与预算的关系

华为强调"没有BP就没有预算"——预算必须服务于战略：

| 预算类型 | 与BP的关系 | 审批层级 |
|---------|-----------|---------|
| 战略预算 | 对应SP中的战略举措，跨年度投入 | EMT审批 |
| 运营预算 | 对应BP中的年度运营目标 | BU负责人审批 |
| 专项预算 | 对应BP中的重点项目 | 职能VP审批 |

## 四、SP 与 BP 的衔接机制

### 4.1 常见衔接断裂问题

| 问题 | 表现 | 根因 |
|------|------|------|
| SP与BP脱节 | SP定了宏大方向，BP却是日常工作的罗列 | 缺乏[[BEM]]解码环节 |
| BP无法支撑SP | BP中的项目与SP战略意图无关 | KPI体系未对齐战略 |
| SP形同虚设 | SP做完后束之高阁，无人回顾 | 缺乏执行追踪机制 |

### 4.2 华为的衔接保障机制

1. **SP指引BP**：SP必须在BP制定前完成定稿，作为BP的输入
2. **[[BEM]]强制解码**：BP制定必须经过BEM流程，确保战略举措可执行
3. **PBC承接**：个人绩效承诺必须追溯到BP中的战略举措
4. **季度QBR**：定期审视BP执行是否偏离SP方向

## 五、SP/BP 的组织保障

### 5.1 关键角色与职责

| 角色 | 在SP/BP中的职责 |
|------|----------------|
| EMT（执行管理团队） | 审批SP/BP，决策战略方向 |
| 战略委员会 | 组织SP研讨，协调跨BU对齐 |
| 各BU负责人 | 制定本BU的SP和BP |
| 战略部 | 流程组织、方法论支持、质量把关 |
| HR | PBC体系设计、人才盘点支撑 |
| 财经 | 预算分配、ROI评估 |

### 5.2 SP/BP 制定的时间节奏

| 时间 | 活动 | 产出 |
|------|------|------|
| 4月 | SP启动会 | SP制定计划 |
| 5-6月 | 市场洞察与差距分析 | 洞察报告 |
| 7月 | SP研讨（[[BLM]]工作坊） | SP初稿 |
| 8月 | SP评审定稿 | SP定稿 |
| 9月 | 战略解码启动 | CSF + 战略举措 |
| 10月 | BP制定 | BP初稿 |
| 11月 | BP评审与预算分配 | BP定稿 + 预算 |
| 12月 | PBC签署 | 个人绩效承诺 |
| 次年1月 | 执行启动 | 全面执行 |

## 六、SP/BP 质量评估标准

### 6.1 SP 质量检查清单

- [ ] 是否有清晰的差距分析（业绩差距+机会差距）？
- [ ] 市场洞察是否有数据支撑，而非泛泛而谈？
- [ ] 战略意图是否量化且可衡量？
- [ ] 业务设计是否回答了"客户选择、价值主张、盈利模式、战略控制点"？
- [ ] 是否有明确的3-5年里程碑？
- [ ] 各BU的SP是否横向对齐？

### 6.2 BP 质量检查清单

- [ ] BP的目标是否承接SP的年度里程碑？
- [ ] 是否有3-5个年度优先行动（而非罗列日常工作）？
- [ ] KPI是否SMART（具体、可衡量、可达成、相关、有时限）？
- [ ] 预算是否与优先行动对齐？
- [ ] 是否有组织/人才/文化的配套计划？

- KPI体系设计与分解辅导

- 建立SP/BP年度节奏
- 设计评审流程和决策机制
