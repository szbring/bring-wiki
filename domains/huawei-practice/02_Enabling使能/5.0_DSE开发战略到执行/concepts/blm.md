# BLM 业务领先模型

## 定义
华为引自 IBM 的战略制定与执行模型，由"领导力 + 价值观"驱动，覆盖战略制定（左半）与战略执行（右半）八大模块，落地差距驱动的战略闭环。

## 输入 / 输出
- **输入**：业绩差距 + 机会差距（从财务实绩与目标对比、市场机会与现状对比识别）
- **输出**：完整 SP 文件（战略意图/市场洞察/创新焦点/业务设计 + 关键任务/正式组织/人才/氛围与文化）
- **核心结构**：
  - 战略制定（左）：差距 → 战略意图 → 市场洞察 → 创新焦点 → 业务设计
  - 战略执行（右）：关键任务 → 正式组织 → 人才 → 氛围与文化
  - 贯穿驱动：领导力 + 价值观

## 角色
- **主持者**：业务单元一把手亲自牵头研讨
- **结构化引导**：战略 / 顾问 团队
- **数据提供**：市场情报、财经、研究机构

## 上下游
- **上游**：差距识别（业绩差距 + 机会差距）作为 BLM 起点
- **下游**：BLM 输出 → [[id:huawei.dste.concept.sp-bp]] SP 文件
- **配套工具**：五看三定（看行业/市场/竞争/自己/机会 + 定目标/策略/计划）作为市场洞察模块的方法论；BEM 业务执行模型用于将战略拆解为关键任务

## 典型误区
- **只做左半边**：研讨完战略意图就结束，不进入"关键任务/组织/人才"——战略执行无抓手
- **跳过差距分析**：直接谈愿景，不分析差距——战略悬空
- **市场洞察走形式**：五看做成报告堆砌，不用于挑战业务假设
- **业务设计 ≠ 收入目标**：业务设计是"谁是客户、卖什么、怎么赚钱、靠什么挡住竞争"——不是数字游戏
