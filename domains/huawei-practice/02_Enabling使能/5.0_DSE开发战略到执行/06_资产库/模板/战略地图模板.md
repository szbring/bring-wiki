# 战略地图模板

<!-- chunk: 战略地图模板.使用说明 -->
## 使用说明

战略地图是 Robert Kaplan 和 David Norton 在平衡计分卡基础上开发的战略可视化工具，将战略从财务、客户、内部流程、学习与成长四个层面进行因果逻辑呈现。在华为DSTE体系中，战略地图用于BLM工作坊的产出整理和SP/BP的可视化表达。

## 三、因果逻辑连线

战略地图的核心价值在于呈现因果关系。填写完四层内容后，用箭头标注因果链：

```
学习成长目标 → 内部流程目标 → 客户目标 → 财务目标

示例：
关键人才引进 → 解决方案交付能力提升 → 客户满意度提高 → 客户续约率提升 → 收入增长
```

### 因果逻辑验证

| 因果链 | 逻辑是否成立？ | 验证方式 |
|--------|-------------|---------|
| [成长目标] → [流程目标] | | 有无数据支撑？ |
| [流程目标] → [客户目标] | | 客户是否确实关注？ |
| [客户目标] → [财务目标] | | 能否带来实际收入？ |

## 四、战略地图与BLM的对应关系

| 战略地图层面 | BLM对应要素 |
|------------|------------|

| 客户层面 | 业务设计（客户选择+价值主张） |
| 内部流程层面 | 业务设计（价值链）+ 关键任务 |
| 学习与成长层面 | 人才 + 企业文化 |

## 五、战略地图一页纸

将完整的战略地图浓缩为一页纸，用于：

1. **高管共识**：BLM工作坊产出整理
2. **全员宣贯**：战略一页纸的核心内容
3. **追踪看板**：QBR回顾的核心框架

```
┌─────────────────────────────────────────┐
│         我们要达成什么（财务）             │
│         ___________________________      │
│                  ↑                       │
│         客户为什么选择我们（客户）          │
│         ___________________________      │
│                  ↑                       │
│         我们必须擅长什么（内部流程）        │
│         ___________________________      │
│                  ↑                       │
│         我们如何持续提升（学习成长）        │
│         ___________________________      │
└─────────────────────────────────────────┘
```
