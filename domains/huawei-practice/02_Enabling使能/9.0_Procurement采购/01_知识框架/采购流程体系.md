# 采购流程体系

采购流程体系覆盖从需求到付款的端到端全过程，确保采购活动规范、透明、高效。

---

<!-- chunk: 采购流程体系.一-端到端采购流程 -->
## 一、端到端采购流程

```
需求管理 → 寻源认证 → 招标谈判 → 合同管理 → 订单执行 → 验收付款
   │          │          │          │          │          │
  需求确认   供应商认证  比价/招标  合同签署   订单跟踪   来料检验
  需求审批   样品测试   谈判定价   条款确认   催交管理   入库结算
```

---

<!-- chunk: 采购流程体系.二-六阶段详解 -->
## 二、六阶段详解

### 2.1 需求管理

| 活动 | 内容 | 输出 |
|---|---|---|
| 需求收集 | 收集各部门采购需求 | 需求清单 |
| 需求审核 | 审核需求合理性和必要性 | 审核通过/退回 |
| 需求分类 | 按品类和紧急度分类 | 分类的需求 |
| 需求合并 | 相同/相似需求合并 | 合并采购需求 |
| 需求释放 | 释放到寻源或直接采购 | 采购申请 |

### 2.2 寻源认证

| 活动 | 内容 | 输出 |
|---|---|---|
| 供应市场分析 | 市场格局、价格趋势、风险 | 市场分析报告 |
| 供应商发现 | 寻找潜在供应商 | 候选供应商名单 |
| 供应商调查 | 资质调查和问卷 | 供应商档案 |
| 现场考察 | 实地考察供应商能力 | 考察报告 |
| 样品测试 | 技术验证和质量测试 | 测试报告 |
| 认证批准 | 合格供应商认证 | 认证供应商名单 |

### 2.3 招标谈判

| 方式 | 适用场景 | 流程 |
|---|---|---|
| 公开招标 | 大额标准化采购 | 发布公告→资格预审→投标→评标→定标 |
| 邀请招标 | 特定品类采购 | 邀请供应商→投标→评标→定标 |
| 竞争性谈判 | 复杂/定制化采购 | 谈判邀请→多轮谈判→确定 |
| 单一来源 | 唯一供应商 | 论证→审批→谈判 |
| 询价比价 | 小额/常规采购 | 询价→比价→选定 |

### 2.4 合同管理

| 活动 | 内容 | 输出 |
|---|---|---|
| 合同起草 | 拟定合同条款和条件 | 合同草案 |
| 法务审核 | 法律风险审核 | 审核意见 |
| 合同谈判 | 与供应商谈判关键条款 | 谈判纪要 |
| 合同签署 | 双方签字生效 | 正式合同 |
| 合同归档 | 合同存档和管理 | 合同台账 |

### 2.5 订单执行

| 活动 | 内容 | 输出 |
|---|---|---|
| 订单下达 | 根据合同下发采购订单 | 采购订单 |
| 供应商确认 | 供应商确认交期和数量 | 确认回执 |
| 过程跟踪 | 跟踪生产进度和发运 | 进度报告 |
| 催交管理 | 催促延迟交货 | 催交记录 |
| 变更管理 | 处理数量/交期变更 | 变更通知 |

### 2.6 验收付款

| 活动 | 内容 | 输出 |
|---|---|---|
| 来料检验 | 按标准检验来料质量 | 检验报告 |
| 入库管理 | 合格品入库登记 | 入库单 |
| 发票核对 | 三单匹配（PO+收货+发票） | 对账确认 |
| 付款审批 | 按合同条款审批付款 | 付款申请 |
| 付款执行 | 财务执行付款 | 付款凭证 |

<!-- chunk: 采购流程体系.三-关键控制点-kcp -->
## 三、关键控制点（KCP）

| 控制点 | 位置 | 控制内容 |
|---|---|---|
| KCP1 | 需求审批 | 需求的合理性和预算 |
| KCP2 | 供应商认证 | 供应商资质和能力 |
| KCP3 | 招标合规 | 招标流程合规性 |
| KCP4 | 合同审批 | 合同条款和风险 |
| KCP5 | 三单匹配 | 订单/收货/发票一致 |
| KCP6 | 付款审批 | 付款条件和审批链 |

## 四、采购流程数字化

| 数字化模块 | 功能 | 价值 |
|---|---|---|
| SRM供应商关系管理 | 供应商全生命周期管理 | 供应商信息统一管理 |
| e-Procurement电子采购 | 在线寻源/招标/订单 | 流程透明、效率提升 |
| P2P采购到付款 | 从申请到付款自动化 | 减少人工、降低差错 |
| 支出分析 | 采购支出可视化和分析 | 支出优化、合规监控 |
