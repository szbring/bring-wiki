# 集成供应链ISC方法论

ISC（Integrated Supply Chain，集成供应链）是华为供应链管理的核心方法论，由IBM顾问引入，基于SCOR模型，历时数年变革而成。任正非曾说："集成供应链（ISC）解决了，公司的管理问题基本上就解决了。"

---

<!-- chunk: 集成供应链isc方法论.一-isc核心理念 -->
## 一、ISC核心理念

### 1.1 三个"集成"

| 集成维度 | 内容 | 目的 |
|---|---|---|
| 流程集成 | 打通计划-采购-制造-交付-退货端到端流程 | 消除断点，信息共享 |
| 组织集成 | 合并制造/计划/采购/物流为供应链管理部 | 跨部门协同 |
| IT集成 | ERP/MRP/WMS/TMS/SRM系统打通 | 数据一致，实时可视 |

### 1.2 变革目标

**质量好、成本低、服务好、对客户需求反应快**

<!-- chunk: 集成供应链isc方法论.二-isc流程框架 -->
## 二、ISC流程框架

### 2.1 基于SCOR的流程架构

```
供应商端 ←───────────────────────────────────────→ 客户端

         ┌──────┐   ┌──────┐   ┌──────┐   ┌──────┐
         │ 采购  │ → │ 制造  │ → │ 交付  │ → │ 退货  │
         │Source│   │ Make │   │Deliver│   │Return│
         └──┬───┘   └──┬───┘   └──┬───┘   └──┬───┘
            │          │          │          │
         ┌──┴──────────┴──────────┴──────────┴──┐
         │              计划 Plan                  │
         │  需求计划 · 供应计划 · 产能计划 · 库存计划 │
         └───────────────────────────────────────┘
```

### 2.2 L2到L3流程分解

| L2流程 | L3流程示例 | 业务模式适配 |
|---|---|---|
| 计划 | S&OP、需求预测、库存计划、产能规划 | 统一 |
| 采购 | 寻源认证、采购执行、供应商管理 | 按物料分类 |
| 制造 | MTS按库存生产、MTO按订单生产、CTO按订单配置 | 按业务模式 |
| 交付 | 订单管理、仓储管理、运输管理、安装调试 | 按客户类型 |
| 退货 | 退货接收、检测维修、返工报废 | 统一 |

<!-- chunk: 集成供应链isc方法论.三-isc变革七步法 -->
## 三、ISC变革七步法

| 步骤 | 内容 | 关键动作 |
|---|---|---|
| 1. 现状诊断 | 识别供应链问题 | IBM梳理78个问题，归为6大核心问题 |
| 2. 流程设计 | 基于SCOR模型设计新流程 | 50+子流程，数百个下级流程 |
| 3. 组织变革 | 合并职能部门为供应链管理部 | 制造+计划+采购+物流→供应链管理部 |
| 4. IT系统建设 | 系统集成和打通 | MRP→ERP→SRM/WMS/TMS |
| 5. KPI体系 | 建立供应链绩效评估 | 五大角度+量化指标 |
| 6. 试点运行 | 选取业务试点 | 深圳基地先行 |
| 7. 全面推广 | 全球推广和持续优化 | GSC→CISC→ISC+ |

<!-- chunk: 集成供应链isc方法论.四-华为isc-数字化转型 -->
## 四、华为ISC+数字化转型

### 4.1 六大转变

| 序号 | 转变方向 | 说明 |
|---|---|---|
| 1 | 被动→主动 | 从被动响应到主动预测和预防 |
| 2 | 人找异常→异常找人 | IOC设置300+探针自动发现异常 |
| 3 | 经验驱动→数据驱动 | 决策基于数据和算法 |
| 4 | 局部优化→全局优化 | 端到端供应链视角优化 |
| 5 | 人工执行→智能执行 | 灵蜂智能引擎自动化作业 |
| 6 | 功能割裂→场景贯通 | 跨功能跨流程场景化运营 |

### 4.2 两层智能业务体系

| 层次 | 名称 | 功能 |
|---|---|---|
| 决策层 | 灵鲲数智云脑 | 全局数据分析、模拟仿真、预案生成、决策指挥 |
| 执行层 | 灵蜂智能引擎 | 敏捷高效、即插即用、蜂群式现场作业 |

## 五、ISC与IPD/LTC的界面

| 界面 | ISC与[[IPD]] | ISC与LTC |
|---|---|---|
| 交互点 | 采购介入研发、BOM传递、可供应性评估 | 订单传递、交付承诺、物流跟踪 |
| 数据流 | 产品数据→BOM→供应方案 | 商机→订单→交付计划 |
| 协同机制 | 采购早期介入研发（EPI） | 交付方案评审（DRB） |
