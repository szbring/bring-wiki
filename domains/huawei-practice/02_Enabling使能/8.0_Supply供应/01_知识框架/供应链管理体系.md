# 供应链管理体系

供应链管理是从原材料到最终产品的端到端价值流管理，华为基于SCOR模型构建了集成供应链（[[ISC]]）体系，实现了从"烟囱式独立模块"到"端到端集成运作"的变革。

---

<!-- chunk: 供应链管理体系.一-scor模型 -->
## 一、SCOR模型

### 1.1 五大流程领域

SCOR（Supply-Chain Operations Reference Model）是供应链运作参考模型，围绕五大基本管理流程：

| 流程 | 英文 | 核心内容 |
|---|---|---|
| 计划 | Plan | 需求与供应规划，平衡供需，制定行动方案 |
| 采购 | Source | 寻源、供应商选择、采购执行、物料获取 |
| 制造 | Make | 生产执行、质量控制、在制品管理 |
| 交付 | Deliver | 订单管理、仓储、运输、安装调试 |
| 退货 | Return | 退货接收、检测、处置（维修/返工/报废） |

### 1.2 SCOR四大要素

| 要素 | 内容 |
|---|---|
| Performance（绩效） | 供应链KPI体系——可靠性、响应性、柔性、成本、资产效率 |
| Process（流程） | 各领域流程和流程关系的标准描述 |
| Practice（实践） | 供应链领域最佳实践指引 |
| People（人员） | 执行供应链流程所需技能的标准定义 |

## 二、华为ISC变革历程

| 阶段 | 时间 | 内容 | 标志性成果 |
|---|---|---|---|
| 建设期 | 1999-2003 | [[ISC]]建设，基于SCOR模型重构流程 | 50+子流程、数百个下级流程 |
| 全球化期 | 2005-2007 | GSC全球供应链建设 | 5大全球供应中心 |
| 集成期 | 2008-2011 | 打通交付，海外多供应中心 | 大交付概念形成 |
| 终端期 | 2012-2014 | CISC终端供应链建设 | ToC供应链体系 |
| 数字化期 | 2015至今 | [[ISC]]+数字化转型 | 灵鲲数智云脑+灵蜂智能引擎 |

### 变革前后对比

| 指标 | 变革前 | 变革后 |
|---|---|---|
| 及时齐套发货率 | 20-30% | 65%+ |
| 库存周转率 | 3.6次/年 | 5.7次/年 |
| 订单履行周期 | 25天 | 17天 |
| 成本 | 基线 | 降低25% |
| 客户满意度 | 基线 | 提升15-30% |

## 三、三大端到端链条

[[ISC]]核心是三条端到端的业务链条：

| 链条 | 内容 | 核心流程 |
|---|---|---|
| 需求到交付 | 从客户需求到订单交付 | 需求管理→计划→采购→制造→交付 |
| 采购到付款 | 从采购需求到供应商付款 | 寻源→招标→合同→采购执行→验收→付款 |
| 退货到处置 | 从退货触发到最终处置 | 退货接收→检测→处置决策→执行 |

<!-- chunk: 供应链管理体系.四-s-op销售与运营计划 -->
## 四、S&OP销售与运营计划

S&OP是[[ISC]]的核心运营机制，将销售预测与供应能力平衡，每月滚动：

```
销售预测 → 需求汇总 → 供应能力评估 → 差距分析 → 行动方案 → 各部门计划更新
    ↑                                                          │
    └────────────── 月度滚动 ←────────────────────────────────┘
```

| S&OP输出 | 内容 |
|---|---|
| 客户订单计划 | 需求分类和优先级 |
| 生产计划 | 排产计划和产能分配 |
| 采购计划 | 采购需求和到货计划 |
| 库存管理计划 | 安全库存和补货策略 |
| 交付计划 | 发货和物流计划 |

## 五、供应链绩效评价五大角度

| 角度 | 关键指标 | 目标方向 |
|---|---|---|
| 可靠性 | 订单准时交付率、齐套发货率、订单完美执行率 | 越高越好 |
| 响应性 | 订单履行周期、紧急订单响应时间 | 越短越好 |
| 柔性 | 需求波动适应性、产能弹性、供应商柔性 | 越强越好 |
| 成本 | 供应链总成本、采购成本、物流成本 | 越低越好 |
| 资产效率 | 库存周转率、现金周转天数、资产利用率 | 越高越好 |
