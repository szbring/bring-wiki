# 华为组织架构演进

<!-- chunk: 华为组织架构演进.一句话定义 -->
## 一句话定义
华为组织架构从职能制→矩阵制→BG制→军团制，每一次演进都与流程变革和战略转型深度耦合。

---

<!-- chunk: 华为组织架构演进.一-组织架构演进四阶段 -->
## 一、组织架构演进四阶段

### 阶段一：职能制（1988-1998）

```
        总裁
    ┌───┼───┐
    │   │   │
  研发 销售 制造
```

| 特征 | 说明 |
|---|---|
| 结构 | 简单职能分工 |
| 决策 | 集中于创始人 |
| 优势 | 快速响应，灵活高效 |
| 问题 | 部门墙，缺乏跨部门协作 |
| 典型问题 | 研发不了解客户需求，销售卖不出研发的产品 |

### 阶段二：矩阵制（1999-2011）

```
           总裁
       ┌───┼───┐
       │   │   │
    职能部门（纵向）
       │   │   │
    ───┼───┼───┼─── PDT（横向）
       │   │   │
```

| 特征 | 说明 |
|---|---|
| 结构 | 职能部门+跨部门团队（PDT/IPT） |
| 驱动 | IPD变革引入PDT（产品开发团队） |
| 优势 | 打破部门墙，端到端协同 |
| 问题 | 双线汇报，考核复杂 |
| 关键机制 | PDT经理对产品全生命周期负责，职能部门提供资源 |

**IPD带来的关键变化**：
- PDT（产品开发团队）：跨部门组建，对产品商业成功负责
- IPMT（集成组合管理团队）：投资决策
- PDT经理有权从职能部门"调动"资源

### 阶段三：BG制（2012-2020）

```
           EMT
       ┌───┼───┐
       │   │   │
    运营商  企业  消费者
    BG     BG    BG
       │   │   │
    ───┼───┼───┼─── 职能平台
```

| 特征 | 说明 |
|---|---|
| 结构 | 三大BG（业务集团）+ 职能平台 |
| 驱动 | 业务多元化，客户差异化 |
| 优势 | BG自主经营，客户导向 |
| 问题 | 平台重复建设，资源分散 |
| 关键机制 | BG有独立的经营决策权，平台提供共享服务 |

**三大BG**：
- 运营商BG：面向电信运营商
- 企业BG：面向企业客户
- 消费者BG：面向消费者

### 阶段四：军团制（2021-至今）

```
           EMT
       ┌───┼───┐
       │   │   │
    BG/BU  军团  军团
    (稳态)  (灵活)
       │
    职能平台
```

| 特征 | 说明 |
|---|---|
| 结构 | BG/BU + 军团组织 |
| 驱动 | 特定行业深度突破，应对制裁挑战 |
| 优势 | 小团队灵活作战，行业深度聚焦 |
| 问题 | 与BG的边界和协作 |
| 关键机制 | 军团按行业组建，包含研发/销售/交付全链条 |

**军团制核心**：
- 按行业组建（煤矿军团、海关港口军团、智能光伏军团等）
- 包含研发+销售+交付全链条，减少内部协同环节
- 类似"特种部队"，快速响应行业需求

## 二、组织与流程的对应关系

| 流程 | 对应组织 | 组织形态 |
|---|---|---|
| IPD | PDT/IPMT/PDT经理 | 跨部门团队 |
| LTC | 客户团队/销售项目组 | 客户导向团队 |
| ITR | 服务交付团队 | 服务团队 |
| ISC | 供应链管理团队 | 端到端供应链 |
| DSE | 战略规划团队 | 跨职能团队 |
| MCR | 客户关系团队 | 客户经理制 |

### 核心原则：组织跟着流程走

```
流程定义价值创造路径 → 组织适配流程执行需要
    ↓                        ↓
流程确定角色和活动     → 组织确定岗位和职责
    ↓                        ↓
流程定义接口和协同     → 组织确定汇报和协作关系
```

**不是组织决定流程，而是流程决定组织。**

<!-- chunk: 华为组织架构演进.三-组织设计的关键决策 -->
## 三、组织设计的关键决策

| 决策维度 | 选项 | 华为选择 | 理由 |
|---|---|---|---|
| 集权 vs 分权 | 集中/分散/联邦 | 联邦型（前台灵活+后台统一） | 平衡效率与管控 |
| 职能 vs 流程 | 部门导向/流程导向 | 流程导向 | 端到端价值创造 |
| 产品 vs 客户 | 产品线/客户线/矩阵 | 矩阵（不同层级不同侧重） | 产品+客户双维度 |
| 稳态 vs 敏捷 | 大组织/小团队 | BG稳态+军团敏捷 | 稳定与灵活兼顾 |

## 四、对咨询客户的启示

1. **组织跟着流程走** — 先定流程，再调组织，不要本末倒置
2. **跨部门团队是关键** — PDT/项目组是打破部门墙的有效手段
3. **分权不是放任** — 前台灵活作战，后台统一管控
4. **组织形态要匹配发展阶段** — 不要小公司学大公司，但要提前规划演进路径
5. **变革要同步** — 流程变革+组织调整+IT建设要同步推进
