# 华为流程治理体系

<!-- chunk: 华为流程治理体系.一句话定义 -->
## 一句话定义
GPO/PO/BPE三层治理架构，确保流程从规划、设计、运行到持续优化的全生命周期管理，实现"流程责任人"对流程结果全权负责。

---

<!-- chunk: 华为流程治理体系.一-流程治理三层架构 -->
## 一、流程治理三层架构

```
┌──────────────────────────────────────────────┐
│     GPO（Global Process Owner）全球流程Owner   │
│     对一级流程规划、设计、运行和结果承担全责        │
├──────────────────────────────────────────────┤
│     PO（Process Owner）流程Owner               │
│     对二/三级流程的设计和运行负责                  │
├──────────────────────────────────────────────┤
│     BPE（Business Process Expert）流程专家      │
│     流程设计、优化、培训的执行者                   │
└──────────────────────────────────────────────┘
```

### 各层职责

| 角色 | 职责 | 任命方式 | 考核挂钩 |
|---|---|---|---|
| GPO | 流程战略规划、重大变更审批、流程绩效管理 | EMT任命 | 流程绩效纳入个人PBC |
| PO | 流程设计维护、流程优化、流程遵从管理 | 业务领域任命 | 流程KPI达标率 |
| BPE | 流程建模、SOP编写、流程培训、问题分析 | 部门任命 | 流程质量指标 |

<!-- chunk: 华为流程治理体系.二-流程绩效管理 -->
## 二、流程绩效管理

### 2.1 流程KPI体系

| KPI类型 | 示例指标 | 数据来源 | 报告频率 |
|---|---|---|---|
| 效率指标 | 流程周期时间、一次通过率 | IT系统 | 月度 |
| 质量指标 | 返工率、差错率、客户投诉率 | 质量系统 | 月度 |
| 成本指标 | 单次流程执行成本、人力投入 | 财务系统 | 季度 |
| 满意度 | 内部用户满意度、外部客户满意度 | 调研 | 季度/半年度 |
| 合规性 | 流程遵从率、审计发现数 | 审计系统 | 季度 |

### 2.2 流程绩效报告

```
GPO季度述职 ──→ 流程绩效报告 ──→ EMT审视
     │                              │
     ▼                              ▼
  改进措施                    资源调整/优先级变更
```

<!-- chunk: 华为流程治理体系.三-流程变更管理 -->
## 三、流程变更管理

### 3.1 变更分类

| 变更类型 | 定义 | 审批层级 | 示例 |
|---|---|---|---|
| 重大变更 | 影响流程架构或跨域接口 | EMT/GPO | 新增一级流程、流程重组 |
| 一般变更 | 流程内部优化 | PO | 流程步骤调整、角色变更 |
| 微调 | 表单/模板/操作说明更新 | BPE | 表单字段增加、SOP细化 |

### 3.2 变更流程

```
变更申请 → 影响评估 → 变更设计 → 审批 → 试运行 → 发布 → 培训 → 监控
```

### 3.3 变更控制要点

- 流程变更必须经过正式审批
- 评估对上下游流程的接口影响
- 重大变更必须试运行验证
- 变更后更新流程文档和IT系统
- 培训相关人员后正式发布

## 四、流程遵从管理

### 4.1 流程遵从审计

| 审计类型 | 频率 | 审计方 | 内容 |
|---|---|---|---|
| 流程自检 | 月度 | 流程Owner团队 | 关键控制点遵从情况 |
| 流程审计 | 季度 | 内控部门 | 流程执行合规性 |
| 专项审计 | 按需 | 内审部门 | 特定流程/事件深入审计 |

### 4.2 流程遵从度指标

| 指标 | 定义 | 目标值 |
|---|---|---|
| 流程执行率 | 实际按流程执行的案件数/总案件数 | >95% |
| 关键控制点遵从率 | 关键控制点实际执行数/应执行数 | 100% |
| 流程偏差率 | 偏离标准流程的案件比例 | <5% |
| 流程文档覆盖率 | 有SOP的流程步骤/总步骤 | 100% |

<!-- chunk: 华为流程治理体系.五-流程资产管理 -->
## 五、流程资产管理

### 5.1 流程资产清单

| 资产类型 | 内容 | 管理方式 |
|---|---|---|
| 流程架构 | BPA L1-L6 | 版本化管理，GPO审批 |
| 流程模型 | 流程图、角色、规则 | ARIS/流程管理平台 |
| 流程文档 | SOP、操作指南、模板 | 知识库管理 |
| 流程指标 | KPI定义、目标值、实际值 | 绩效系统 |
| 流程改进记录 | 优化建议、变更历史 | 变更管理系统 |

### 5.2 流程版本管理

- 每次变更创建新版本
- 版本号规则：V主版本.次版本.修订号
- 历史版本可追溯
- 发布版本与IT系统版本同步

<!-- chunk: 华为流程治理体系.六-华为与一般企业流程治理的差异 -->
## 六、华为与一般企业流程治理的差异

| 维度 | 一般企业 | 华为 |
|---|---|---|
| 流程Owner | 兼职/无明确Owner | GPO全职，对流程全权负责 |
| 流程变更 | 随意变更 | 严格变更管理流程 |
| 流程遵从 | 靠人自觉 | 流程内嵌IT系统强制 |
| 流程绩效 | 无度量 | 量化指标纳入考核 |
| 流程资产 | 散落在各部门 | 集中管理，版本化 |
