# 华为的流程管理

## 核心概念与定义

华为流程管理的核心理念是"流程反映业务流"，流程不是权力的载体，而是业务运作的轨道。华为理解的流程是：从客户需求出发，端到端拉通所有相关活动，以客户价值创造为目标，把跨部门的离散活动集成为有序的业务流。流程化组织就是沿着流程分配权力、资源和责任的组织。"班长的战争"——战术指挥权下沉到一线作战单元，是流程化组织的核心形态。

## 方法论框架

### 一、华为三大业务流程体系

| 流程 | 全称 | 起点 | 终点 | 核心组织 |
|------|------|------|------|---------|
| [[IPD]] | 集成产品开发 | 市场需求/Charter | 产品上市/GA | PDT重量级团队 |
| [[LTC]] | 线索到回款 | 线索/机会点 | 合同关闭/回款 | 铁三角（CC3） |
| [[ITR]] | 问题到解决 | 客户问题/故障 | 问题关闭/客户确认 | 服务交付团队 |

### 二、流程设计五原则

| 原则 | 内容 |
|------|------|
| 主干清晰，末端灵活 | 主流程标准化，末端允许因地制宜 |
| 目标导向建设，结果导向优化 | 流程建设对准客户价值，优化对准业务结果 |
| 流程不可以彰显权力 | 流程是业务轨道，不是权力的舞台 |
| 效率优先，兼顾管控 | 先跑通再管控，控制点不超过5个 |
| 流程体系化管理 | 流程分层分级（Level1-6），有owner有审计有优化机制 |

### 三、流程变革道法器术

| 层面 | 内容 | 华为实践 |

### 四、流程变革四关键因素

| 因素 | 内容 |
|------|------|
| 变革认知 | 为什么变？流程的战略价值是什么？正确的变革意识 |
| 变革策略 | 先易后难，控制节奏，管理期望，分阶段推进 |
| 变革能力 | 有没有能力做？顾问引入+内部能力建设 |
| 变革机制 | 变革组织保障+运作流程+激励机制+流程owner制度 |

### 五、流程型组织建设核心

1. **前端拉动为主，后端推动为辅**：从下往上设计，客户需要什么组织就建什么
2. **让听得见炮声的人呼唤炮火**：一线呼唤需求，后方联勤化响应
3. **后方联勤化**：一线呼唤一个点，机关之间自行协调，不让一线反复对接多个部门
4. **倒推流程**：从客户端往回梳理，平台只服务前线需要

### 六、流程变革成功十六条经验（核心8条）

1. 变革是一把手工程，最高层坚定支持
2. 先僵化后优化再固化（削足适履）
3. 引入世界最佳实践，不闭门造车
4. 没有一线业务人员主导，不可能做出好流程
5. 变革目标清晰：多打粮食+增强土地肥力
6. 控制变革节奏，管理变革期望
7. 变革和业务运作不脱节，边变革边运作
8. 流程owner负责制，持续优化不回退

### 七、日落法（简化流程）

任正非提出学习"日落法"：每新增一个流程节点，必须关闭另两个流程节点。主业务流程和能力解耦，能力模块化、微服务化。

## 关键流程/步骤

1. **流程建设流程**：业务流梳理→流程现状诊断→差距分析→目标流程设计→流程文件编写→试点推行→全面切换→持续优化
2. **流程变革推行**：变革立项→现状诊断→方案设计（顾问+业务专家）→试点验证→推行实施→效果评估→制度化固化
3. **流程管理闭环**：流程owner指定→流程绩效指标设定→流程执行监控→流程审计→流程优化需求收集→流程迭代发布

## 核心原则

1. **流程反映业务流**：流程是业务流的描述，不是人为设计的约束。业务流是客观存在的，流程要和业务流合拍——流程特别顺畅说明合拍，特别别扭说明业务流被扭曲。

2. **端到端拉通**：流程必须从客户需求出发到客户满意结束，中间不能断开。跨部门的断点是流程设计最大的问题。

3. **从下往上设计**：先看客户需要什么，再倒推后方需要提供什么支撑。不给力的就剪掉。不是从上往下设计权力结构。

4. **流程不能彰显权力**：流程是业务的轨道，不是权力的载体。很多企业的流程成了部门权力宣示——"必须经过我审批"，这是流程的异化。

5. **IT建设前提是流程稳定**：业务流不稳定时不要急着做IT系统。流程没捋顺就做IT是错的，会付出两三倍的时间和金钱。

## 典型案例与实践要点

**案例1：[[LTC]]流程变革**。2007年启动CRM规划，2008年确定10大问题，分4个子项目推进（核心CRM/以客户为中心能力/卓越一线/解决方案优化）。LTC是1.0阶段重点，目标让财务更健康/客户更满意/工作更高效。整个项目华为花数亿美金。

**案例2：班长的战争**。美军从越南战争（拳头打跳蚤被拖死）到伊拉克战争（联合作战仅1人非战斗死亡）到刺杀本拉登（23人38分钟），核心转变是战术指挥权下沉。华为学习这种模式：项目经理有战术指挥权，总部保留干部任命/审计/资金/财务四权。流程化组织就是支持这种作战模式的基础设施。

**实践要点**：
- 流程分6层：L1一级流程→L2→L3→L4→L5→L6操作指导
- 每个流程有owner（流程所有者），对流程绩效负责
- 过程审计确保流程遵从，不合规会被流程审计发现问题
- 变革节奏控制：先易后难，先核心后外围，试点再推行
- 业务流不稳定时用纸质流程/Excel，稳定后再做IT系统

## 与华为其他体系的关联

- [[IPD]]流程与研发组织联动：PDT重量级团队是IPD的组织载体
- [[LTC]]流程与销售组织联动：铁三角（CC3）是LTC的组织载体
- [[ITR]]流程与服务组织联动：GTS是ITR的组织载体
- 流程与绩效管理联动：流程KPI纳入组织绩效，流程遵从度纳入个人PBC
- 流程与财经体系联动：四算（概算/预算/核算/决算）嵌入流程控制点
- 流程与质量管理联动：流程审计=质量审计，过程合规是质量的基础
