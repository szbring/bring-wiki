# IPD智枢 · 企业智能化产品开发平台
**薄云咨询 · 企业产品方案**
**版本：V4.0（企业客户版）**
**日期：2026年3月**

---

> **一句话定位：**
> IPD智枢是薄云咨询为正在运行 IPD 体系的企业团队打造的 AI Agent 工具套件，让产品经理、PDT Leader、项目经理用 AI 把每一个 IPD 流程环节跑得更快、更准、更有质量。

---

## 1. 产品定位与价值主张

### 产品名称

**IPD智枢**（IPD ZhiShu · Intelligent Hub for Product Development）

副标题：让每一个 IPD 流程环节都有 AI 在你身边

---

### 目标客户画像

| 角色 | 他是谁 | 他最头疼什么 |
|------|--------|-------------|
| **CPO / 产品VP** | 负责整个产品组合的战略方向，对研发ROI负责 | 看不清哪些产品该投、哪些该退；数据分散，决策靠感觉；不知道自己的IPD到底跑到什么水平 |
| **PDT Leader / 产品经理** | 负责一条产品线从立项到上市，对商业成功负责 | Charter写起来很费劲；DCP评审前材料准备要花好几天；不知道本产品处于哪个生命周期阶段 |
| **项目经理** | 负责研发项目的计划、执行和风险管理 | 风险识别不系统；跨部门进度难追踪；周报写了没人看；DCP节点交付物经常漏项 |
| **IPMT成员** | 参与立项评审、阶段评审的高层决策者 | 评审材料厚重难读；各产品数据口径不统一，无法横向比较；Go/Kill/Hold决策缺乏数据支撑 |
| **需求分析师 / 市场产品经理** | 收集用户需求、做竞品分析、输出需求包 | 访谈完笔记整理要半天；需求散乱，很难结构化；需求优先级靠强势方主导 |

**典型客户企业特征：**
- 研发投入占营收 15% 以上
- 有 2 条以上产品线并行开发
- 已引入或正在导入 IPD 体系（导入 6 个月以上）
- 团队规模：研发 50-500 人

---

### 核心价值主张

**对产品经理/PDT Leader：**
> 你不再需要对着空白文档发呆——告诉 AI 你的产品想法，它帮你把 Charter 五大模块（市场机会/产品定义/商业计划/开发计划/成功标准）全部搭出来，DCP 评审前一天就能搞定所有材料。

**对项目经理：**
> 不再靠经验猜风险——AI 帮你系统识别每个阶段的典型风险，风险登记册一键生成，周报自动整理成管理层要看的格式。

**对 CPO/产品VP：**
> 不再用 Excel 拼产品组合数据——每个产品的健康度（财务40%+市场30%+技术20%+运营10%）实时可见，BCG矩阵自动生成，该投资的、该收割的、该退市的，AI 直接告诉你建议和理由。

**对 IPMT：**
> 评审不再是走过场——每次 DCP 评审，AI 提前帮你备好核心问题清单和 Go/Kill/Hold 决策建议书，让每一个决策都有数据支撑。

---

### 与传统 IPD 咨询的核心差异

| 维度 | 传统 IPD 咨询 | IPD智枢 |
|------|------------|---------|
| **服务形式** | 顾问到场辅导，人走知识走 | AI Agent 常驻，随时可用 |
| **响应速度** | 等顾问排期，通常 3-7 天 | 当下即用，分钟级输出 |
| **覆盖深度** | 重点项目重点辅导，其他项目自生自灭 | 每个项目每个节点都有 AI 支持 |
| **知识沉淀** | 靠顾问的 Word 文档，难以复用 | 结构化知识库，每次用都更懂你 |
| **成本结构** | 按天计费，规模越大成本越高 | 订阅制，边际成本接近零 |
| **使用门槛** | 需要顾问协助才能用好 | 产品经理自己就能上手 |
| **诊断能力** | 顾问诊断，主观性强，每次重来 | AI 基于标准框架，客观输出成熟度基线，持续追踪进步 |

## 2. 客户 IPD 痛点分析

### 2.1 立项阶段（DCP0 前）— 最常见的浪费

**真实场景：**
产品经理接到任务："我们要做一个 XX 产品，你把 Charter 写一下，下周 DCP0。"
然后他花了 3 天，写出一份"看起来像 Charter"的文档——市场数据是估的，财务模型是拍的，竞品分析是网上随便搜的。

**核心痛点：**
- Charter 五大模块（市场机会/产品定义/商业计划/开发计划/成功标准）结构复杂，格式不熟，从零写要耗时 2-3 天
- 市场数据、竞品信息分散在各处，整合耗时
- 财务回报模型（IRR/NPV/回本周期）很多产品经理根本不会建
- 写完了也不知道 DCP0 评审会问哪些问题，被打回来重写

**失败模式：**
→ Charter 质量低 → DCP0 评审走形式 → 立项靠"老板感觉" → 方向错了发现太晚 → 浪费 6-12 个月研发资源

---

### 2.2 计划与开发阶段（DCP1-DCP2）— 最容易出错的阶段

**真实场景：**
PDT 团队在推进项目，但跨部门协作像打仗：研发说市场需求不清，市场说研发不理解客户，供应链说来不及，财务说成本超了。

**核心痛点：**
- 需求变更无序：客户需求改了，没人评估影响范围，直接就改了
- 风险识别靠经验：项目经理凭感觉列风险，容易漏掉行业特有的坑
- 跨部门协调靠催：PDT 会议纪要没人写，行动项没人追
- DCP 评审材料临时抱佛脚：DCP1 要准备什么？DCP2 要核查哪些指标？不清楚

**失败模式：**
→ 需求蔓延 → 开发周期越拉越长 → 成本越来越高 → DCP2 时发现方向跑偏 → 要么强行上市要么中途夭折

---

### 2.3 产品上市后（DCP4/PLM）— 最容易被忽视的阶段

**真实场景：**
产品上市了，皆大欢喜。然后……就没有然后了。没有人系统地跟踪这个产品的健康状况，直到销售反馈"卖不动了"才发现产品已经进入衰退期，但研发还在往里投资。

**核心痛点：**
- 没有系统化的产品健康度监测（财务/市场/技术/运营四维），靠销售感觉判断产品好不好
- 年度产品生命周期评审（PLCR）准备工作量巨大，各产品数据要分别收集
- 不知道什么时候该退市：产品"撑着死"，大量维护资源被吃掉
- 产品组合失衡：成熟产品和新产品资源分配没有科学依据；S 曲线接替时机判断不准

**失败模式：**
→ 产品"自然死亡"而非"有序退市" → 维护成本失控 → 研发 70% 资源在维护，只有 30% 在创新 → 新产品青黄不接

---

### 2.4 IPD 体系本身的诊断盲区 — 最被忽视的问题

**真实场景：**
企业导入 IPD 已经 1 年了，Charter 和 DCP 都在跑，但感觉"不太对劲"——评审走过场，Charter 质量参差不齐，跨部门协作没多大改善。但没有人能说清楚：**我们到底处于哪个成熟度阶段？差距在哪里？下一步该改什么？**

**核心痛点：**
- 没有客观的基线评估，不知道当前 IPD 处于 L1/L2/L3 哪个级别
- 六个维度（立项决策/需求管理/跨部门协作/项目过程/研发ROI/工具系统）哪些弱、哪些强，没有全貌
- 不知道 PLM 成熟度如何——生命周期管理、退市流程、产品组合管理是否到位
- 改进方向太多，不知道优先改哪个

**失败模式：**
→ IPD 变成形式主义 → 顾问离场后流程退化 → 再次请顾问，重头再来 → IPD 落地成本越来越高

---

### 2.5 团队能力差距 — 最根本的问题

- 新加入 PDT 的成员不了解 IPD 流程，学习成本高
- 产品经理懂产品但不懂财务模型（IRR/NPV/回本周期）
- 项目经理会做计划但不懂 IPD 的阶段评审逻辑
- 每次流程执行质量参差不齐，取决于哪个顾问在辅导

---

## 3. Agent 产品体系设计

IPD智枢包含 **9 个核心 Agent 产品**，按使用场景组织，覆盖 IPD 全流程的关键节点。

> **Agent 设计原则：**
> - **入口工具先行**：成熟度自诊断是所有客户的起点，也是最好的销售钩子
> - **流程节点对齐**：每个 Agent 对应 IPD 流程的一个或多个关键节点
> - **知识库内嵌**：所有 Agent 内嵌薄云华为系 IPD 知识库，输出质量远超通用 AI

---

### Agent 1：IPD成熟度自诊断

| 属性 | 内容 |
|------|------|
| **产品名称** | IPD成熟度自诊断 |
| **Skill 名称** | `ipd-maturity-assessor` |
| **目标用户** | CPO、研发VP、CTO、IPD项目负责人 |
| **解决什么问题** | 不知道企业 IPD 处于哪个成熟度级别（L1-L5）；不知道六个维度里哪些是短板；改进方向太多，不知道先做什么；PLM 成熟度没有客观评估 |
| **开发优先级** | **P0（入口产品，也是销售钩子）** |

**核心功能：**

1. **快速诊断（5分钟版）**：4 个关键判断题（基于薄云华为系问卷），即时输出 L1-L5 成熟度判断和"你们最痛的1个点"
   - 「你们每年多少个新产品立项？靠什么标准决定？」
   - 「研发和市场之间最常见的冲突是什么？」
   - 「一个研发项目，谁对商业成功负责？」
   - 「上一个失败的研发项目，主要原因是什么？」

<!-- chunk: consulting-ipd-agent智能化方案.part1 -->

2. **六维深度评估（30分钟版）**：覆盖 IPD 六大维度（立项决策机制 / 需求管理 / 跨部门协作 / 项目过程管理 / 研发ROI管理 / 工具与系统支撑），每维度 4 题，共 24 项评估，输出雷达图

3. **PLM成熟度专项评估（20分钟版）**：基于薄云标准 30 题 PLM 问卷，六维度评估（生命周期阶段划分 / DCP决策机制 / 产品组合管理 / 绩效指标体系 / 退市流程规范度 / 数字化支撑），输出加权总分和成熟度级别

4. **GAP 差距分析报告**：对照华为标准，识别当前级别与目标级别（通常 L3）的 TOP3 核心差距，每个差距附具体表现和改进建议

5. **定制访谈问题生成**：基于诊断结果，为后续深度访谈（研发VP/产品经理/项目经理/CEO）生成定制化问题清单，帮助顾问/团队开展精准访谈

6. **推荐路径输出**：基于成熟度评估结果，推荐最适合的 IPD智枢 套餐和薄云咨询服务组合

**典型使用场景：**
> 某工业装备企业 CTO 接触 IPD智枢 后，先用 5 分钟完成快速诊断。系统判断：「你们处于 L2 受管理级——有基本项目管理，但研发与市场脱节，立项靠经验。」并指出核心差距是「立项决策」和「跨部门协作」。
>
> CTO 感觉"说到心里去了"，进一步完成 30 分钟六维深度评估，得到完整的雷达图：立项决策 L1.5、需求管理 L2、跨部门协作 L1、项目过程管理 L2.5、研发ROI管理 L1、工具支撑 L2。
>
> 系统推荐：「优先改立项决策和跨部门协作，建议从 IPD智枢 标准版开始，同步考虑薄云 IPD 轻量化导入（SP2）。」

| | |
|---|---|
| **输入** | 企业基本信息（规模/行业/IPD导入时长）+ 诊断问卷回答 |
| **输出** | 成熟度级别评定（L1-L5）+ 六维雷达图 + TOP3差距分析 + 改进路标 + 推荐套餐 + 定制访谈问题清单（可选）|
| **使用频率** | 项目节点（首次使用时/每半年复测一次/顾问诊断前）|

### Agent 2：立项决策助手

| 属性 | 内容 |
|------|------|
| **产品名称** | 立项决策助手 |
| **Skill 名称** | `ipd-charter-generator` |
| **目标用户** | PDT Leader、产品经理 |
| **解决什么问题** | Charter 五大模块从零写要 2-3 天，格式繁琐，质量参差不齐；财务回报模型（IRR/NPV）不会建；DCP0 评审被打回的概率高 |
| **开发优先级** | **P0** |

**核心功能：**

1. **Charter 智能生成**：告诉 AI 你的产品想法（几句话也行），它帮你生成符合薄云 Charter 标准模板的完整五大模块初稿：
   - **市场机会模块**：目标市场细分（TAM/SAM/SOM）、客户痛点TOP3、竞争分析表格
   - **产品定义模块**：一句话产品定位、核心功能特性TOP5、差异化卖点
   - **商业计划模块**：3年收入预测表格、成本估算结构、IRR/回本周期初步预测
   - **开发计划模块**：DCP0-DCP4 里程碑时间线、资源需求估算、PDT 团队建议构成
   - **成功标准模块**：可量化的成功指标（市场份额/营收目标/客户满意度）

2. **财务模型辅助**：自动建立收入预测、成本估算、IRR/NPV/回本周期的基础模型框架，引导产品经理逐步填入关键参数

3. **DCP0 交付物核查**：生成 Charter 后，自动对照 DCP0 评审标准核查"5个必须回答的问题"（市场机会是真实的吗？产品定义是清晰的吗？商业回报是合理的吗？计划是可执行的吗？风险是可接受的吗？），告诉你哪里不达标

4. **信息缺口提示**：识别 Charter 中信息不足的部分（如缺少市场数据支撑、竞品分析不够），给出具体的补全建议和数据来源参考

5. **评审问题预测**：预测 IPMT 在 DCP0 评审时最可能追问的 5-8 个问题，帮你提前准备答案

6. **CBB 复用建议**：根据产品功能描述，自动提示可能存在的可复用模块（CBB），降低重复研发风险

**典型使用场景：**
> 某制造企业产品经理王总，周一接到任务：本周五要进行新产品 X 的 DCP0 立项评审，需要提交 Charter。
> 他打开立项决策助手，用 15 分钟填写了产品基本信息问卷（目标市场、核心功能、竞品、预算），助手在 30 分钟内生成了完整 Charter 初稿（含五大模块），并标出了 3 处数据缺口需要补充。王总花了半天完善数据，周三就完成了一份高质量的 Charter，比以往提前 2 天。

| | |
|---|---|
| **输入** | 产品名称、目标市场描述、核心价值卖点、竞品信息、预算范围、计划上市时间 |
| **输出** | 完整 Charter 文档（含五大模块，对齐薄云Charter标准模板）+ DCP0 交付物核查清单 + 评审问题预测清单 + CBB 复用建议 |
| **使用频率** | 项目节点（每次新产品立项） |

### Agent 3：竞品情报雷达

| 属性 | 内容 |
|------|------|
| **产品名称** | 竞品情报雷达 |
| **Skill 名称** | `ipd-market-analyzer` |
| **目标用户** | 产品经理、市场经理、PDT Leader |
| **解决什么问题** | 竞品信息散落各处，整合一份竞品分析报告要 2-3 天；分析框架不统一，每次格式都不一样；Charter 竞争分析模块数据不支撑 |
| **开发优先级** | **P0** |

**核心功能：**

1. **竞品全景扫描**：输入竞品名单，AI 自动搜集并整理各竞品的核心信息（产品特性、价格定位、市场份额、近期动态、技术路线）

2. **差异化定位分析**：生成竞品对比矩阵（华为标准竞品分析格式），清晰标注你的产品在哪些维度领先、哪些落后、差异化在哪里

3. **市场规模估算**：给出 TAM/SAM/SOM 的基础估算框架和参考数据，直接对应 Charter 市场机会模块

4. **SWOT 自动生成**：基于竞品分析结果，输出产品的 SWOT 草稿（与 Charter 格式兼容）

5. **竞争动态预警**：识别竞品近期的重大动作（新产品发布、价格调整、技术专利），提示需要调整差异化策略的信号

6. **Charter 直接对接**：输出格式与立项决策助手的 Charter 模板（竞争分析模块）完全兼容，可直接粘贴使用

**典型使用场景：**
> 在做 DCP0 准备时，产品经理需要填写 Charter 的"竞争分析"模块，平时需要找 3-5 个信息源手动整理。现在只需输入 5 个竞品名称，20 分钟后得到一份结构化的竞品对比表，可以直接粘贴进 Charter，还附带发现竞品上月刚发布了类似功能，需要调整差异化定位的预警。

| | |
|---|---|
| **输入** | 目标市场关键词、竞品名称列表（3-8个）、分析维度偏好 |

### Agent 5：DCP评审助手

| 属性 | 内容 |
|------|------|
| **产品名称** | DCP评审助手 |
| **Skill 名称** | `ipd-review-preparer` |
| **目标用户** | PDT Leader、项目经理、IPMT 成员 |
| **解决什么问题** | 每次 DCP 评审前材料准备耗时长（1-3天）；不清楚每个 DCP 节点应该完成哪些交付物；评审问题没有系统化准备，容易被打回；IPMT 审查材料时抓不住重点 |
| **开发优先级** | **P1** |

**核心功能：**

1. **分节点评审材料生成**：根据评审类型（DCP0/DCP1/DCP2/DCP3/DCP4），自动生成评审汇报大纲和核心材料框架，每个 DCP 节点的核心问题不同：
   - **DCP0**：这个产品值得做吗？市场机会是否存在？
   - **DCP1**：计划可行吗？资源是否到位？
   - **DCP2**：开发目标达成了吗？可以测试了吗？
   - **DCP3**：产品质量达标了吗？可以上市了吗？（上市就绪度评审：功能+质量+市场准备）
   - **DCP4**：产品是否继续投资？何时退市？（年度生命周期评审触发条件核查）

2. **交付物完成度核查**：自动列出本 DCP 阶段应该完成的所有交付物，标注已完成/未完成状态，未完成项自动给出紧急程度和补充建议

4. **遗留问题清单**：自动识别项目中尚未解决的关键问题（技术问题/市场问题/资源问题），并建议处理优先级和 Owner

5. **财务门核查**：每个 DCP 节点都嵌入财务评审（DCP0-IRR初步预测/DCP1-5年P&L模型/DCP2-成本基线/DCP3-定价毛利率/DCP4-实际vs预测偏差），提醒哪些财务指标需要准备

6. **评审问题预测**：预测 IPMT 在本次评审中最可能提出的质疑点，帮助 PDT 提前准备答案

**典型使用场景：**
> PDT Leader 李总，DCP2 评审在 3 天后。他把项目当前状态告诉 DCP 评审助手（开发进度 92%，3 个里程碑完成，1 个延迟，主要测试指标达标情况）。

| **输入** | 评审类型（DCP0-4）、项目当前状态摘要、里程碑完成情况、主要风险清单 |

6. **季度风险更新提醒**：系统自动提醒每季度更新风险状态，标注已关闭、新增、升级的风险项

### Agent 7：产品健康监测仪

| 属性 | 内容 |
|------|------|
| **产品名称** | 产品健康监测仪 |
| **Skill 名称** | `plm-health-scorer` |
| **目标用户** | 产品经理、CPO、IPMT 成员 |
| **解决什么问题** | 产品上市后没有系统化的健康度监测，靠销售感觉判断好不好；年度 PLCR 评审数据收集整合耗时；不知道产品处于哪个生命周期阶段，投资策略混乱 |
| **开发优先级** | **P1** |

**核心功能：**

1. **四维健康度评分**（华为标准模型）：
   - **财务健康（权重40%）**：毛利率趋势 / 收入增长率 / 与 Charter 预测偏差程度 / IRR 实际 vs 预测
   - **市场健康（权重30%）**：市场份额变化 / NPS/CSAT 客户满意度 / 竞争态势（相对位置）
   - **技术健康（权重20%）**：代码质量评分（缺陷密度）/ 架构负债程度 / 与新技术兼容性
   - **运营健康（权重10%）**：版本发布准时率 / 客户服务响应效率 / 供应链健康度

2. **红/黄/绿三色预警体系**：
   - 80分以上 🟢 绿灯（健康，按计划推进）
   - 60-79分 🟡 黄灯（关注，制定改善计划）
   - 40-59分 🟠 橙灯（预警，启动专项诊断）
   - 40分以下 🔴 红灯（危机，IPMT 紧急介入）

3. **生命周期阶段识别**：根据财务和市场数据，自动判断产品处于导入期/成长期/成熟期/衰退期/退市期哪个阶段，并与各阶段标准 KPI 基线对比（如：成长期应达到市场增速的 1.5 倍以上）

4. **产品经营看板生成**：生成一页式产品经营看板（华为标准格式）：当前阶段/四维健康度得分/本季度重点动作（已完成/进行中/风险）/下季度决策点提示

5. **阶段偏差分析**：对标当前生命周期阶段的标准 KPI（导入期：首年收入达成率 >70% / 成长期：收入增长 >行业均值1.5倍 / 成熟期：NPS>30 / 衰退期：维护成本占收入比预警），指出哪些指标偏离了正常范围

6. **下季度决策建议**：基于健康度评分和生命周期阶段，输出"继续（Go）/ 调整（Adjust）/ 加速退市（Accelerate EoL）"的具体建议和理由

7. **PLCR 年度评审包**：一键生成年度产品生命周期评审所需的全套材料（产品年度经营报告 + 竞争态势摘要 + 决策建议），减少 PLCR 前的数据整合工作量

**典型使用场景：**
> CPO 每个季度要对 5 条产品线做经营评估，以前要让 5 个产品经理各自准备材料，花一周时间。现在每个产品经理只需填写健康度数据表（20-30分钟），产品健康监测仪自动生成每个产品的四维健康度评分（财务/市场/技术/运营）和一页式看板。CPO 开会时直接看 5 张看板，30 分钟完成原本需要 3 小时的评审会议。

| | |
|---|---|
| **输入** | 产品名称、当前生命周期阶段（或由系统判断）、财务数据（毛利率/收入增长/IRR偏差）、市场数据（市场份额/NPS）、技术数据（缺陷率/架构评估）、运营数据（交付准时率/客户响应）|
| **输出** | 四维健康度评分 + 综合预警等级 + 生命周期阶段判断 + 产品经营看板（一页式）+ 阶段偏差分析 + 下季度决策建议 + PLCR 评审包（年度模式）|
| **使用频率** | 月/季度（日常监测）+ 项目节点（年度 PLCR 评审前） |

### Agent 8：产品路标规划助手

| 属性 | 内容 |
|------|------|
| **产品名称** | 产品路标规划助手 |
| **Skill 名称** | `ipd-roadmap-planner` |
| **目标用户** | CPO、产品VP、PDT Leader |
| **解决什么问题** | 产品路标规划靠经验排序，缺乏系统化的多产品投资优先级方法（BCG/组合管理）；S 曲线接替时机判断不准，新老产品交接出现断档；资源分配没有量化依据 |
| **开发优先级** | **P2** |

1. **BCG 矩阵自动分析**：将现有产品组合可视化为 BCG 矩阵（明星/现金牛/问题/瘦狗），自动给出每类产品的投资策略建议：
   - **明星产品**：大力投资，巩固领先地位，准备向现金牛过渡
   - **现金牛产品**：控制研发投入，最大化利润提取，资金反哺明星/问题产品
   - **问题产品**：评估市场潜力，设定观察期，选择重点投入或快速退出
   - **瘦狗产品**：设定退市时间表，最小化维护成本

3. **跨产品线资源优化建议**：
   - 从现金牛产品抽取资金，投入明星/问题产品研发的具体方案
   - 从衰退产品释放人员，充实成长产品团队的排期建议
   - 共享 CBB 平台能力，降低跨产品重复投入的识别

4. **多方案路标对比**：生成 2-3 个不同的资源分配路标方案，对比各方案的预期收益、风险和战略匹配度

5. **产品接替时间线**：生成新老产品交接的具体时间线，包括：新产品何时立项、何时可以承接老产品客户、老产品何时触发退市评审

6. **CBB 平台化建设建议**：识别多产品线共用功能，提出平台化/CBB 化建议，提升整体研发效率

**典型使用场景：**
> 某装备企业 CPO 面临年度产品路标规划，手上有 7 条产品线、有限的研发资源。她把每条产品线的基本经营数据和市场数据输入路标规划助手，得到一张 BCG 矩阵（2 个明星、3 个现金牛、1 个问题、1 个瘦狗），S 曲线分析显示成熟度最高的现金牛产品已到接替窗口，建议今年 Q3 启动下一代立项；瘦狗产品建议加速退市释放约 15 名研发人力充实明星产品。

| **输入** | 产品列表 + 每条产品线的市场份额/增长率/当前投入 + 战略优先级权重 + 总资源约束 |
| **输出** | BCG 矩阵分类及策略建议 + S 曲线接替时间表 + 跨产品线资源优化方案 + 多方案路标对比 + CBB 平台化建议 |

### Agent 9：退市决策助手

| 属性 | 内容 |

| **Skill 名称** | `plm-eol-planner` |
| **目标用户** | 产品经理、IPMT 成员、CPO |
| **解决什么问题** | 产品"撑着死"而不是"有序退出"；退市决策拖延，维护成本持续攀升；退市通知发出太晚（应提前 18-24 个月），客户措手不及；退市流程不规范，品牌受损 |
| **开发优先级** | **P2** |

**核心功能：**

1. **退市时机判断**：基于产品健康度数据，判断产品是否已达到退市触发条件（华为标准触发信号：财务持续恶化连续 2 季度 / 市场份额萎缩到阈值 / 替代技术/产品就绪 / 维护成本超过收入 X%）

2. **退市规划生成**：按华为标准退市时间线（T-24 月到 T+0），生成完整的退市计划：
   - **T-24 月**：识别退市信号，提交退市建议至 IPMT

   - **T+X 月**：合同履约结束，完全退市

3. **客户影响评估**：分析退市决策对不同客户群体的影响（高价值客户 / 合同期内客户 / 长尾客户），识别需要特殊关注的关键客户

4. **客户迁移方案**：制定客户从退市产品迁移到替代产品的方案（含迁移时间表、差异化支持政策、迁移优惠建议）

5. **版本生命周期管理**：针对多版本并行情况，生成版本收敛计划（从全面支持期 → 维护支持期 → 延长支持期 → 版本退市），引导客户向新版本迁移

6. **EoL 通知文件生成**：生成规范的退市通知文件，包含：退市产品范围（版本/型号）/ 关键时间节点 / 推荐替代方案 / 客户迁移支持政策

**典型使用场景：**
> 一个有 12 年历史的成熟产品，维护成本越来越高，市场份额持续下滑。产品经理将产品的经营数据输入退市决策助手，得到明确的退市建议（触发条件：维护成本已占收入 43%，连续 3 季度亏损）和完整的 18 个月退市规划。包括：何时停止接受新订单、如何引导现有 40 家客户迁移到新产品、给不同客户的差异化迁移支持方案，以及符合华为标准格式的 EoL 通知模板。

### IPD智枢 的核心护城河

1. **薄云华为系 IPD 知识壁垒（最核心）**
   IPD 是一套复杂的企业研发管理体系，涵盖 Charter/DCP/PDT/MM/OR/PLM 等完整框架，且有大量华为特有的决策逻辑（Go/Kill/Hold 三态决策/财务门机制/五级成熟度/四维健康度模型）。薄云多年 IPD 咨询的一手知识库，是任何通用 AI 无法短期复制的核心资产。

2. **成熟度诊断驱动销售漏斗**
   IPD成熟度自诊断是绝佳的销售钩子——它帮客户"自我发现"痛点，建立对问题的清晰认知，从而产生付费意愿。没有其他工具有这种"先诊断后用药"的完整产品设计。

3. **全流程嵌入深度**
   IPD智枢 不是一个独立工具，而是嵌入企业 IPD 流程的每一个关键节点（立项→需求→开发→评审→健康度→路标→退市）。用得越多，知识库越丰富，产出质量越高，替换成本越高。

4. **与薄云咨询服务的协同效应**
   IPD智枢 是薄云咨询服务的 AI 延伸——咨询客户用 IPD智枢 持续执行；新客户通过 IPD智枢 发现问题，自然引导至咨询服务。两者相互促进，形成"咨询获客→工具留存→咨询升级"的正向飞轮。

5. **企业数据积累形成护城河**
   随着使用时间增长，系统积累了企业的历史项目数据、Charter 模版优化记录、成熟度改进轨迹、产品健康度历史。这是第一个使用 IPD智枢 3 年的客户和刚开始使用的客户之间的核心差距——越用越聪明，越用越难替换。

## 附录：快速决策参考

### 客户适合买哪个套餐？

**判断问题：**

1. 你们 IPD 导入多久了？
   - < 18 个月或还在建立基础机制 → **入门版**
   - 1-2 年，基础跑通，想提升执行效率 → **标准版**
   - 2 年以上，成熟运作，想做产品组合战略智能化 → **专业版**

2. 你们现在最痛的是什么？
   - Charter 写不好、DCP 评审没质量、不知道自己 IPD 什么水平 → **入门版** 直接解决
   - 需求管理乱、项目风险漏、产品健康没数据 → **标准版**
   - 产品组合资源分配不科学、不知道什么时候退市 → **专业版**

3. 产品线数量？
   - 1-2 条 → 入门版足够
   - 3-5 条 → 标准版
   - 5 条以上 → 专业版

4. 成熟度自诊断结果？（最直接的判断依据）
   - L1-L2 → 入门版
   - L2-L3 → 标准版
   - L3-L4 → 专业版
