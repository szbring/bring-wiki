# 流程诊断方法论

## 一句话定义
从流程成熟度、组织匹配度、IT支撑度三个维度系统诊断，识别流程断点和改进方向，为变革方案提供精准输入。

---

## 一、三维度诊断框架

```
┌─────────────────────────────────────────────────┐
│                流程诊断三维模型                      │
├───────────┬───────────┬───────────────────────────┤
│ 流程成熟度  │ 组织匹配度  │ IT支撑度                   │
│ ·流程完整度 │ ·组织对齐度 │ ·系统覆盖度                │
│ ·流程规范度 │ ·角色匹配度 │ ·数据贯通度                │
│ ·流程执行度 │ ·绩效对齐度 │ ·自动化程度                │
│ ·流程效率度 │ ·能力匹配度 │ ·用户体验度                │
└───────────┴───────────┴───────────────────────────┘
```

---

## 二、维度一：流程成熟度诊断

### 2.1 流程完整度

<!-- chunk: consulting-流程诊断方法论.part1 -->

| 检查项 | 评估标准 | 评分(1-5) |
|---|---|---|
| 流程架构 | 是否有完整的L1-L4流程架构 | |
| 流程文件 | 每个L3流程是否有流程图+SOP | |
| 角色定义 | 每个流程步骤是否有明确的RACI | |
| 输入输出 | 流程间的接口是否有明确传递物 | |
| 业务规则 | 关键控制点是否有明确的规则 | |

### 2.2 流程规范度

| 检查项 | 评估标准 | 评分(1-5) |
|---|---|---|
| 流程版本管理 | 是否有版本控制和变更管理 | |
| 流程文档标准 | 是否有统一的文档模板 | |
| 流程Owner | 每个流程是否有明确的Owner | |
| 流程审批 | 流程变更是否有正式审批 | |
| 流程培训 | 是否有流程培训机制 | |

### 2.3 流程执行度

| 检查项 | 评估标准 | 评分(1-5) |
|---|---|---|
| 执行遵从率 | 实际执行与标准流程的偏差率 | |
| IT固化率 | 有多少流程步骤在IT系统中强制执行 | |
| 偏差处理 | 偏离标准流程时是否有处理机制 | |
| 审计发现 | 内控审计发现的流程执行问题数 | |

### 2.4 流程效率度

| 检查项 | 评估标准 | 评分(1-5) |
|---|---|---|
| 流程周期时间 | 端到端流程的周期时间 | |
| 一次通过率 | 一次完成无需返工的比例 | |
| 审批层级 | 关键流程的审批层级 | |
| 等待时间 | 流程步骤间的等待时间 | |

## 三、维度二：组织匹配度诊断

### 3.1 组织对齐度

| 检查项 | 评估标准 | 评分(1-5) |
|---|---|---|
| 组织vs流程 | 组织架构是否与端到端流程对齐 | |
| 跨部门团队 | 是否有PDT/项目组等跨部门团队 | |
| 端到端责任 | 是否有人对端到端流程结果负责 | |
| 部门墙 | 部门间信息传递的阻碍程度 | |

### 3.2 角色匹配度

| 检查项 | 评估标准 | 评分(1-5) |
|---|---|---|
| 流程角色→岗位 | 流程定义的角色是否映射到具体岗位 | |
| 人员能力 | 在岗人员是否具备流程角色的能力要求 | |
| 人员数量 | 流程角色的人员配置是否充足 | |

### 3.3 绩效对齐度

| 检查项 | 评估标准 | 评分(1-5) |
|---|---|---|
| 流程KPI | 是否有流程绩效指标 | |
| 考核挂钩 | 流程KPI是否纳入绩效考核 | |
| 端到端考核 | 是否有端到端流程的考核指标 | |

## 四、维度三：IT支撑度诊断

### 4.1 系统覆盖度

| 检查项 | 评估标准 | 评分(1-5) |
|---|---|---|
| 流程线上化率 | 有多少流程步骤在IT系统中运行 | |
| 系统完整性 | 是否覆盖了端到端流程 | |
| 系统集成 | 各系统之间是否打通 | |

### 4.2 数据贯通度

| 检查项 | 评估标准 | 评分(1-5) |
|---|---|---|
| 主数据管理 | 是否有统一的主数据管理 | |
| 数据一致性 | 跨系统数据是否一致 | |
| 数据实时性 | 数据是否实时或准实时 | |

### 4.3 自动化程度

| 检查项 | 评估标准 | 评分(1-5) |
|---|---|---|
| 自动化率 | 人工操作占流程步骤的比例 | |
| 智能化 | 是否有AI/ML辅助的流程步骤 | |
| 异常处理 | 系统是否能自动检测和处理异常 | |

## 五、诊断输出

### 5.1 三维雷达图

```
        流程成熟度
          5
       /    \
      /      \
 IT    4      3
 支撑度 |      | 组织匹配度
      \      /
       \    /
          1
```

### 5.2 断点清单

| 断点 | 类型 | 影响范围 | 严重程度 | 改进建议 |
|---|---|---|---|---|
| | 流程/组织/IT | | □高□中□低 | |

### 5.3 改进路线图

```
短期(1-3月)：关键断点修复    ──→ 流程成熟度+1级
中期(3-6月)：组织+流程对齐   ──→ 整体+1级
长期(6-12月)：IT支撑+自动化  ──→ 整体+2级
```
