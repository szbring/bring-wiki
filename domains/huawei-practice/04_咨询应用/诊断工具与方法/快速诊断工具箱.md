# 快速诊断工具箱

## 一句话定义
提供1天/3天/1周三种诊断周期，适配不同深度需求，快速识别核心问题和改进方向，为后续深度咨询提供精准输入。

---

## 一、三种诊断周期

| 周期 | 适用场景 | 诊断深度 | 产出 | 投入 |
|---|---|---|---|---|
| 1天快扫 | 初次接触、判断是否有咨询价值 | 定性判断 | 1页纸诊断结论 | 1顾问1天 |
| 3天快诊 | 客户有明确意向，需要出方案建议 | 半定量 | 诊断简报+改进路线 | 1-2顾问3天 |
| 1周深诊 | 签约前的正式诊断 | 定量 | 完整诊断报告+方案框架 | 2-3顾问5天 |

## 二、1天快扫工具箱

### 2.1 时间分配

| 时间 | 活动 | 产出 |
|---|---|---|
| 09:00-10:00 | CEO/高管访谈（核心挑战、期望） | 关键问题假设 |
| 10:00-12:00 | 中层访谈（2-3个部门负责人） | 验证/修正假设 |
| 14:00-15:00 | 核心流程观察（选1个主流程） | 流程断点识别 |
| 15:00-16:00 | 数据速览（经营数据+组织数据） | 量化佐证 |
| 16:00-17:00 | 形成结论 | 1页纸结论 |

### 2.2 高管访谈5问

1. "您觉得公司目前最大的3个管理挑战是什么？"
2. "如果有一个顾问能帮您解决1个问题，您希望是什么？"
3. "您的竞争对手在哪些方面做得比您好？"
4. "您觉得公司的战略和执行脱节吗？"
5. "3年后您希望公司变成什么样？"

### 2.3 中层访谈5问

<!-- chunk: consulting-快速诊断工具箱.part2 -->

1. "您部门最头疼的问题是什么？"
2. "您和其他部门协作中最大的摩擦是什么？"
3. "公司流程对您部门工作有帮助还是阻碍？"
4. "您觉得公司最大的浪费在哪里？"
5. "如果让您改1件事，您改什么？"

### 2.4 一页纸结论模板

```
┌─────────────────────────────────────────┐
│  快扫诊断结论                              │
│                                           │
│  核心挑战：[1句话]                          │
│  根因假设：[1-2句话]                        │
│  成熟度估算：L[?]/5                         │
│  可改进方向：                               │
│   1. [方向1]                              │
│   2. [方向2]                              │
│   3. [方向3]                              │
│  咨询价值判断：□高 □中 □低                  │
│  建议下一步：[3天快诊/1周深诊/直接出方案]     │
└─────────────────────────────────────────┘
```

## 三、3天快诊工具箱

### 3.1 时间分配

<!-- chunk: consulting-快速诊断工具箱.part3 -->

| 天 | 上午 | 下午 |
|---|---|---|
| Day1 | 高管访谈(3人)+战略文档审阅 | 中层访谈(4-6人)+组织架构分析 |
| Day2 | 流程诊断(选2-3个核心流程) | 数据分析(经营+流程+组织) |
| Day3 | 问题汇总+根因分析 | 方向建议+路线图+汇报 |

### 3.2 流程速诊工具

| 流程 | 诊断维度 | 快速判断方法 |
|---|---|---|
| IPD | 端到端 | 问：从需求到上市，几个部门经手？平均多长时间？ |
| LTC | 端到端 | 问：从线索到回款，平均多长时间？预测偏差多少？ |
| DSE | 战略落地 | 问：战略规划完成后，有多少比例落到个人PBC？ |
| HR | 人才管理 | 问：关键岗位的人才密度是多少？干部有几个梯队？ |

### 3.3 诊断简报模板

| 章节 | 内容 | 页数 |
|---|---|---|
| 执行摘要 | 核心发现+改进方向 | 1页 |
| 现状分析 | 分维度的现状描述 | 3-4页 |
| 问题清单 | 按优先级排列的问题列表 | 1-2页 |
| 改进路线 | 3-12月改进路线图 | 1页 |

## 四、1周深诊工具箱

### 4.1 时间分配

| 天 | 主要活动 | 产出 |
|---|---|---|
| Day1 | 高管访谈+战略/经营文档审阅 | 背景理解+问题假设 |
| Day2 | 中层访谈(8-12人)+核心流程walkthrough | 问题验证+流程断点 |
| Day3 | 成熟度评估(2-3个核心流程)+数据分析 | 量化评估 |
| Day4 | 标杆对比+根因分析 | 差距识别+根因确认 |
| Day5 | 报告撰写+方案框架+汇报 | 完整诊断报告 |

### 4.2 完整诊断报告模板

| 章节 | 内容 | 页数 |
|---|---|---|
| 执行摘要 | 核心发现+关键建议 | 1页 |
| 客户背景 | 企业概况+行业环境 | 1-2页 |
| 诊断方法 | 使用的诊断工具和方法 | 1页 |
| 现状分析 | 分维度现状描述+成熟度评估 | 5-8页 |
| 问题识别 | 问题清单+根因分析+影响评估 | 3-5页 |
| 标杆对比 | 与华为/行业标杆的差距 | 2-3页 |
| 改进建议 | 优先级+路线图+预期效果 | 2-3页 |
| 附录 | 访谈记录、数据表 | 若干 |

---

## 五、诊断质量保障

| 保障项 | 标准 |
|---|---|
| 访谈覆盖 | 高管3+人，中层8+人，覆盖所有核心部门 |
| 数据佐证 | 关键结论必须有数据支撑 |
| 验证闭环 | 初步结论与客户验证，避免误判 |
| 标杆校准 | 用华为/行业标杆校准评估标准 |
| 客户确认 | 诊断结论必须经客户确认 |
