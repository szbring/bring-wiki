# 薄云咨询知识库质量标准 v1.0

> 创建于 2026-03-24 · 适用范围：华为管理体系知识库（BringVault）

---

## 一、结构完整性标准（必须满足）

每个一级流程目录必须包含以下6个子目录，且每个子目录至少有1个 `.md` 文件：

| 子目录 | 内容要求 |
|---|---|
| `01_知识框架` | 核心理念、华为标准体系、行业最佳实践 |
| `02_诊断工具` | 成熟度评估模型、访谈问题库、GAP分析框架 |
| `03_解决方案` | 标准服务包（含周期/人天/报价/适用场景）|
| `04_交付指南` | 分阶段实施路径、常见阻力与应对、验收标准 |
| `05_销售材料` | 客户痛点话术、价值主张、异议处理 |
| `06_资产库` | 工具模板索引、脱敏案例、参考资料 |

**当前状态（2026-03-24 第二轮质检）：**
- ✅ 17 / 17 个流程达到结构完整性标准
- 总文件数：134个

---

## 二、内容质量标准（分级）

### L1 基础级（当前状态 ✅）
- 每个子目录有 ≥1 个 `.md` 文件
- 内容覆盖该子目录的核心要素
- 有薄云视角的咨询应用说明

### L2 标准级（下阶段目标）
- 每个子目录有 ≥2 个 `.md` 文件
- `06_资产库` 有可直接使用的工具模板（非仅索引）
- 有至少1个脱敏客户案例
- 报价数据基于真实项目校准

### L3 精品级（长期目标）
- 每个子目录有 ≥3 个 `.md` 文件
- 知识框架与华为最新版本保持同步
- 有多个行业的案例（科技/制造/金融等）
- 诊断工具有量化评分和对比基准

---

## 三、内容准确性标准

| 维度 | 标准 |
|---|---|
| 华为方法论 | 与华为BPA最新版本一致，有明确版本号 |
| 报价参考 | 每年更新一次，反映市场实际水平 |
| 案例 | 脱敏处理，不含客户识别信息 |
| 工具模板 | 可直接使用，有填写说明 |

---

## 四、维护更新标准

| 触发条件 | 更新内容 | 责任人 |
|---|---|---|
| 每个项目完成后 | `06_资产库/案例/` 补充脱敏案例 | 项目负责顾问 |
| 发现新的客户痛点 | `02_诊断工具/访谈问题库` 追加 | 任何顾问 |
| 定价调整 | `03_解决方案/标准服务包` 更新报价 | 运营负责人 |
| 每季度 | 全库完整性质检（用独立 Agent 执行）| 小智 |
| 华为方法论更新 | `01_知识框架` 同步更新 | 方法论负责人 |

---

## 五、质检流程标准（三 Agent 工作流）

建立于 2026-03-24，参照今天的实践固化：

```
Step 1：质检 Agent
→ 检查17个流程的6个子目录完整性
→ 输出 JSON 缺失清单

Step 2：补齐 Agent
→ 根据缺失清单填充内容
→ 输出「✅ 已完成：[文件路径]」

Step 3：复核 Agent
→ 验证补齐结果
→ 输出最终完成度报告（目标：100%）
```

**质检频率**：每季度至少一次，重大更新后立即执行
