# 客户满意度管理体系

> 华为[[ITR]]流程中的客户满意度管理——从被动响应到主动管理，以CSQC为核心，建立客户回访和满意度指标体系。

---

<!-- chunk: 客户满意度管理体系.一-客户满意度在itr中的位置 -->
## 一、客户满意度在ITR中的位置

ITR的终极目标不只是"解决技术问题"，而是**通过卓越服务赢得客户信任**，让客户愿意给你更多机会。

华为的服务理念：
- 不仅满足客户的契约要求，更要超出客户的期望
- 服务是下一单产品销售的开始
- 售后服务部门要从成本中心转变为利润增长点

<!-- chunk: 客户满意度管理体系.二-csqc体系 -->
## 二、CSQC体系

### 2.1 CSQC定义

CSQC（Customer Service Quality Control）是华为客户服务质量管控体系，确保服务过程和结果满足客户期望。

### 2.2 CSQC四个维度

| 维度 | 含义 | 衡量指标 |
|------|------|---------|
| **C - 契约履行** | 是否满足合同约定的SLA | SLA达标率 |
| **S - 服务过程** | 服务过程是否专业高效 | 过程规范率、客户投诉率 |
| **Q - 服务质量** | 解决方案是否有效彻底 | 一次解决率、问题复发率 |
| **C - 客户感知** | 客户主观感受如何 | NPS、满意度评分 |

### 2.3 CSQC评分模型

| 维度 | 权重 | 评分(1-5) | 加权得分 |
|------|------|----------|---------|
| 契约履行 | 30% | | |
| 服务过程 | 20% | | |
| 服务质量 | 30% | | |
| 客户感知 | 20% | | |
| **综合** | **100%** | | **/5** |

## 三、客户回访机制

### 3.1 回访类型

| 回访类型 | 触发条件 | 回访时间 | 回访人 |
|---------|---------|---------|--------|
| 工单回访 | 每个工单关闭后 | 关闭后24-48小时 | 客服专员 |
| V/B级问题回访 | 处理V/B级问题后 | 解决后24小时 | 服务经理 |
| 投诉回访 | 客户投诉处理后 | 处理后48小时 | 服务总监 |
| 定期满意度调研 | 例行 | 季度/年度 | 客户成功团队 |

### 3.2 回访内容

| 回访问题 | 评分维度 | 目的 |
|---------|---------|------|
| 问题是否已彻底解决？ | 服务质量 | 确认解决有效性 |
| 响应速度是否满意？ | 契约履行 | 评估SLA达标感知 |
| 工程师专业度如何？ | 服务过程 | 评估服务能力 |
| 整体服务是否满意？ | 客户感知 | 总体满意度 |
| 是否愿意推荐？ | NPS | 客户忠诚度 |
| 是否有其他需求？ | 机会识别 | ITR→[[LTC]]转化 |

### 3.3 回访结果处理

| 回访结果 | 处理方式 |
|---------|---------|
| 满意（4-5分） | 记录归档，纳入KPI |
| 一般（3分） | 记录归档，分析改进点 |
| 不满意（1-2分） | 启动不满意处理流程 |

**不满意处理流程**：
1. 24小时内联系客户了解原因
2. 制定改善措施并执行
3. 再次回访确认满意度
4. 归档并纳入改进计划

## 四、满意度指标体系

### 4.1 核心指标

| 指标 | 计算方式 | 目标值 |
|------|---------|--------|
| **CSAT**（客户满意度） | 满意客户数/回访客户数 | ≥90% |
| **NPS**（净推荐值） | 推荐者%-贬损者% | ≥50 |
| **FCR**（一次解决率） | 一次解决工单数/总工单数 | ≥70% |
| **SLA达标率** | SLA内解决工单数/总工单数 | ≥95% |
| **问题复发率** | 复发工单数/总工单数 | ≤5% |
| **投诉率** | 投诉工单数/总工单数 | ≤2% |

### 4.2 过程指标

| 指标 | 计算方式 | 目标值 |
|------|---------|--------|
| 平均响应时间 | 所有工单响应时间平均值 | ≤SLA要求的80% |
| 平均解决时间 | 所有工单解决时间平均值 | ≤SLA要求的80% |
| 升级率 | 升级工单数/总工单数 | ≤15% |
| 客户回访覆盖率 | 已回访工单数/关闭工单数 | ≥90% |

### 4.3 业务转化指标

| 指标 | 计算方式 | 目的 |
|------|---------|------|
| ITR→[[LTC]]转化率 | 服务中发现新订单数/总工单数 | 评估服务驱动销售 |
| 续约率 | 维保续约客户数/到期客户数 | 评估服务对续约的影响 |
| 客户留存率 | 续约客户数/总客户数 | 评估服务对留存的影响 |

## 五、满意度改进闭环

### 5.1 PDCA循环

```
Plan（计划）：设定满意度目标和改进计划
  ↓
Do（执行）：实施改进措施
  ↓
Check（检查）：通过回访和指标监控效果
  ↓
Act（行动）：标准化有效措施，纠正偏差
```

### 5.2 满意度改进的常见方向

| 低分维度 | 常见原因 | 改进方向 |
|---------|---------|---------|
| 契约履行低 | SLA不达标 | 增加资源/优化流程/调整SLA |
| 服务过程低 | 沟通不畅/不专业 | 培训/规范SOP/改善沟通 |
| 服务质量低 | 解决不彻底 | 知识库建设/二线支持加强 |
| 客户感知低 | 态度差/不主动 | 服务意识培训/主动沟通 |

### 5.3 质量回溯

对重大问题和投诉进行质量回溯：

| 回溯内容 | 输出 |
|---------|------|
| 问题根因 | 根因分析报告 |
| 流程漏洞 | 流程改进建议 |
| 能力短板 | 培训和认证计划 |
| 产品缺陷 | 输入[[IPD]]改进 |

## 六、薄云咨询的应用建议

### 诊断切入点
- 评估是否有系统性的客户回访机制
- 检查CSAT/NPS是否定期测量
- 诊断满意度低分的根因是否被追踪和改进

### 辅导重点
- 建立CSQC四维度评估体系
- 设计客户回访SOP和问卷
- 建立满意度改进闭环机制
- 推动ITR→[[LTC]]的业务转化
