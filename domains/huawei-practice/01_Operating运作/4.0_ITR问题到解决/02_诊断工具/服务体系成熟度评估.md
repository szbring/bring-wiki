# 服务体系成熟度评估

> 基于华为ITR流程实践，构建五级服务体系成熟度模型，支撑[[ITR]]体系诊断和改进规划。

---

<!-- chunk: 服务体系成熟度评估.一-成熟度模型总览 -->
## 一、成熟度模型总览

| 级别 | 名称 | 核心特征 | 典型表现 |
|------|------|---------|---------|
| **L1** | 被动响应 | 无标准流程，问题来了才处理 | 靠个人能力，无知识库 |
| **L2** | 流程规范 | 有工单流程和SLA，但执行不力 | 有流程但一线二线不分明 |
| **L3** | 分级服务 | 三级服务体系建立，SLA可衡量 | 分层处理，知识库可用 |
| **L4** | 主动服务 | 主动维护+预测性维护 | 数据驱动改进，客户满意可衡量 |
| **L5** | 智能服务 | AI辅助诊断和预防，服务产品化 | 自动化+智能化，服务变利润中心 |

<!-- chunk: 服务体系成熟度评估.二-六维度评估 -->
## 二、六维度评估

### 2.1 流程维度

| 评估项 | L1 | L2 | L3 | L4 | L5 |
|--------|-----|-----|-----|-----|-----|
| 服务流程 | 无标准 | 有工单流程 | 三阶段流程完整 | 流程持续优化 | AI辅助流程 |
| SLA管理 | 无 | 有但不严格 | SLA可衡量达标 | SLA动态调整 | 智能SLA预测 |
| 升级机制 | 靠个人关系 | 有升级规则 | 三级升级畅通 | 升级率持续降低 | AI自动升级 |

### 2.2 组织维度

| 评估项 | L1 | L2 | L3 | L4 | L5 |
|--------|-----|-----|-----|-----|-----|
| 服务团队 | 无专职团队 | 有客服但无分层 | 一线/二线/三线 | 专家体系完善 | AI+人工混合 |
| 服务能力 | 依赖个人 | 基本可解决 | 分层能力匹配 | 持续培训提升 | AI辅助诊断 |
| 服务意识 | 被动响应 | 满足基本需求 | 客户满意导向 | 主动超越期望 | 个性化服务 |

### 2.3 工具维度

| 评估项 | L1 | L2 | L3 | L4 | L5 |
|--------|-----|-----|-----|-----|-----|
| 工单系统 | 无/Excel | 基础工单 | 完整工单系统 | CRM+工单集成 | AI+IoT+工单 |
| 知识库 | 无 | 零散文档 | 结构化知识库 | 知识库持续更新 | AI自动匹配 |
| 客户自助 | 无 | FAQ | 在线知识库 | 智能客服 | 全自助平台 |

### 2.4 数据维度

| 评估项 | L1 | L2 | L3 | L4 | L5 |
|--------|-----|-----|-----|-----|-----|
| 服务数据 | 无记录 | 基础记录 | 完整数据采集 | 数据驱动改进 | 预测性分析 |
| 满意度测量 | 不测量 | 偶尔问卷 | 定期CSAT/NPS | 满意度改进闭环 | 实时NPS监控 |
| 问题分析 | 不分析 | 简单统计 | 根因分析 | 趋势分析和预防 | AI根因诊断 |

### 2.5 业务转化维度

| 评估项 | L1 | L2 | L3 | L4 | L5 |
|--------|-----|-----|-----|-----|-----|
| ITR→LTC | 无 | 偶尔发现机会 | 有转化机制 | 转化率可衡量 | 系统化服务营销 |
| 服务产品化 | 无 | 标准维保 | 服务包设计 | 增值服务 | 服务即产品 |
| 客户留存 | 不关注 | 基本维系 | 主动留存 | 留存率可衡量 | 流失预警 |

### 2.6 持续改进维度

| 评估项 | L1 | L2 | L3 | L4 | L5 |
|--------|-----|-----|-----|-----|-----|
| 问题回溯 | 不做 | 偶尔复盘 | V/B级必回溯 | 全量回溯 | 自动化回溯 |
| 改进闭环 | 无 | 提建议不追踪 | PDCA闭环 | 效果可量化 | AI驱动持续优化 |
| 产品反馈 | 不反馈 | 偶尔反馈 | 有反馈通道 | 反馈驱动产品改进 | 实时反馈闭环 |

## 三、评估打分

### 3.1 打分规则

每个评估项按1-5分打分，对应L1-L5级别。

### 3.2 综合得分计算

| 维度 | 得分 | 权重 | 加权得分 |
|------|------|------|---------|
| 流程 | /5 | 25% | |
| 组织 | /5 | 15% | |
| 工具 | /5 | 15% | |
| 数据 | /5 | 20% | |
| 业务转化 | /5 | 15% | |
| 持续改进 | /5 | 10% | |
| **综合** | | **100%** | **/5** |

### 3.3 成熟度判定

| 综合得分 | 成熟度等级 | 下一步目标 |
|---------|-----------|-----------|
| 1.0-1.5 | L1 | 建立基础工单和SLA |
| 1.5-2.5 | L2 | 分层服务和知识库 |
| 2.5-3.5 | L3 | 数据驱动和满意度管理 |
| 3.5-4.5 | L4 | 主动服务和业务转化 |
| 4.5-5.0 | L5 | 智能化和服务产品化 |

## 四、快速诊断问题清单

1. **「客户有问题时，怎么联系你们？响应时间多久？」**
   - 无统一入口/响应慢 → L1-L2

2. **「一线工程师解决不了的问题，怎么升级？升级到谁？」**
   - 靠个人关系升级 → L1-L2
   - 有升级路径但执行不畅 → L2-L3
   - 三级升级畅通 → L3+

3. **「客户满意度怎么衡量？多高？有没有改进闭环？」**
   - 不衡量 → L1
   - 偶尔问卷 → L2
   - 定期NPS+改进闭环 → L3+

4. **「服务过程中发现的销售机会，有没有机制转化？」**
   - 不关注 → L1-L2
   - 偶尔发现 → L2-L3
   - 有ITR→[[LTC]]转化机制 → L3+

5. **「同样的问题反复出现吗？有没有知识库和根因分析？」**
   - 反复出现，无知识库 → L1-L2
   - 有知识库但不完善 → L3
   - 根因分析+预防 → L4+
