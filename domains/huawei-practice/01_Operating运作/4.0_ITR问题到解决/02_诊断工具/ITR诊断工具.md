# ITR诊断工具

<!-- chunk: itr诊断工具.成熟度评估 -->
## 成熟度评估

| 级别 | 特征 |
|---|---|
| L1 | 客户问题靠销售个人协调，无统一渠道，结果随机 |
| L2 | 有客服团队，但响应慢，同类问题反复发生 |
| L3 | 有SLA承诺，问题分类和分配规范，知识库初建 |
| L4 | 数据驱动服务改进，客户满意度可量化追踪 |
| L5 | 主动服务（预测问题在发生前解决），AI辅助 |

<!-- chunk: itr诊断工具.快速诊断问题 -->
## 快速诊断问题

1. **「客户遇到问题，通过什么渠道反馈？有没有统一入口？」**
   - 各自找销售/微信/电话 → L1，渠道混乱
   - 有统一电话/邮件 → L2
   - 有统一工单系统 → L3+

2. **「P0级别的紧急问题，你们的承诺响应时间是多少？实际做到了吗？」**
   - 没有SLA承诺 → L1-L2
   - 有承诺但经常违约 → L2
   - 有承诺且基本达标 → L3+

3. **「同一个客户同一类问题，有没有反复出现过？」**
   - 经常有 → 知识库和根因分析缺失
   - 偶尔有 → L2-L3
   - 很少有 → L3+，知识库运转良好

4. **「客服团队和研发/产品团队的协作机制是什么？」**
   - 客服说不清研发什么时候修复 → 跨部门协作缺失
   - 有流程但不顺畅 → L2-L3
   - 有SLA+透明进度 → L3+

## 核心KPI基线采集

| 指标 | 采集方式 |
|---|---|
| 首次响应时间 | 从工单提交到首次响应的时间 |
| 平均解决时长 | 从工单提交到关闭的时间 |
| 首次解决率 | 第一次接触就解决的比例 |
| 客户满意度（CSAT） | 问题关闭后的评分 |
| 重复问题率 | 30天内同一客户同类问题再次出现 |
