# 销售漏斗方法论

> 销售漏斗是[[LTC]]流程的核心管理工具——从线索到回款，通过量化各阶段转化率，变不确定为确定，实现销售过程的可视、可控、可预测。

---

<!-- chunk: 销售漏斗方法论.一-销售漏斗的本质 -->
## 一、销售漏斗的本质

> "团队销售中最难处理的地方在于过程的不确定性。因为正在运作的项目总是处于各种可能发生的变化之中，而销售漏斗的基本思想就是**变不确定为确定**。只有确定才能量化，只有量化才能管理。"——华为销售法

销售漏斗不是简单的阶段划分，而是一套**从预测到管理**的完整方法论。

---

<!-- chunk: 销售漏斗方法论.二-漏斗阶段定义 -->
## 二、漏斗阶段定义

### 2.1 标准五阶段模型

```
         ┌──────────┐
         │  线索    │  宽口：广开喇叭口
         │  Lead    │  市场活动、客户转介绍、自开发
         ├──────────┤
         │  机会点  │  收窄：有预算有需求
         │ Opp      │  立项确认、方案设计
         ├──────────┤
         │  方案    │  进一步收窄
         │  Sol     │  标前引导、投标答标
         ├──────────┤
         │  合同    │  接近出口
         │  Contract│  谈判签约、合同评审
         ├──────────┤
         │  回款    │  出口：价值实现
         │  Cash    │  交付验收、回款核销
         └──────────┘
```

### 2.2 各阶段的进入标准

| 阶段 | 进入标准 | 主责角色 | 关键动作 |
|------|---------|---------|---------|
| **线索** | 客户有潜在购买意愿 | AR（客户经理） | 收集、验证、培育 |
| **机会点** | 有明确预算+确定启动时间+匹配方案 | AR牵头，SR介入 | 立项决策、客户需求分析 |
| **方案** | 机会点立项通过 | SR牵头，AR/FR协同 | 标前引导、方案设计、投标 |
| **合同** | 中标且通过签约评审 | AR牵头商务谈判 | 谈判、合同评审、签约 |
| **回款** | 合同签订生效 | FR牵头交付 | 交付验收、回款跟进 |

<!-- chunk: 销售漏斗方法论.三-转化率管理 -->
## 三、转化率管理

### 3.1 关键转化率指标

| 转化率 | 计算方式 | 行业参考值 | 华为标杆值 |
|--------|---------|-----------|-----------|
| 线索→机会点 | 机会点数/线索数 | 20-30% | 30-40% |
| 机会点→方案 | 立项通过数/机会点数 | 60-70% | 70-80% |
| 方案→合同 | 中标数/投标数 | 25-35% | 35-45% |
| 合同→回款 | 回款完成数/合同数 | 80-90% | 90-95% |
| **端到端** | 回款数/线索数 | 3-5% | 8-12% |

### 3.2 漏斗健康度判断

| 症状 | 诊断 | 处方 |
|------|------|------|
| 线索多但机会点少 | 线索质量差或培育不力 | 优化ICP，加强线索培育 |
| 机会点多但方案少 | 立项通过率低，可能项目质量差 | 加强立项前的筛选和培育 |
| 方案多但合同少 | 赢单率低 | 加强标前引导和方案竞争力 |
| 合同多但回款少 | 交付问题或回款不力 | 提升交付质量和回款管理 |
| 整体漏斗很窄 | 喇叭口不够大 | 加强[[MTL]]，扩大线索来源 |

### 3.3 漏斗流速管理

不仅要看"量"和"转化率"，还要看**流速**（每个阶段的停留时间）：

| 阶段 | 正常停留时间 | 预警阈值 |
|------|------------|---------|
| 线索 | 1-3个月 | >6个月 |
| 机会点 | 1-2个月 | >4个月 |
| 方案 | 1-3个月 | >6个月 |
| 合同 | 0.5-1个月 | >2个月 |
| 回款 | 1-3个月 | >6个月 |

停留时间过长的项目需要特别关注：可能是客户犹豫不决、竞争对手阻击、或我方跟进不力。

## 四、销售预测体系

### 4.1 三级预测机制

| 预测层级 | 时间范围 | 用途 | 数据来源 |
|---------|---------|------|---------|
| **长期预测** | 1-3年 | 战略规划、投资决策 | 市场空间、客户业务规划、产品路标 |
| **中期预测** | 季度/年度 | 资源规划、产能准备 | 合同/订单趋势、年度销售目标 |
| **短期预测** | 月度/3个月 | 运营管理、风险预警 | 计/发/收/回实时数据 |

### 4.2 加权预测法

根据漏斗阶段赋予不同的赢单概率：

| 阶段 | 赢单概率 | 示例 |
|------|---------|------|
| 线索 | 10% | 10个线索×100万×10% = 100万预测 |
| 机会点 | 30% | 5个机会点×200万×30% = 300万预测 |
| 方案/投标 | 50% | 3个方案×300万×50% = 450万预测 |
| 谈判 | 75% | 2个谈判×400万×75% = 600万预测 |
| 已签约 | 95% | 1个签约×500万×95% = 475万预测 |
| **合计** | | **1925万预测收入** |

### 4.3 预测的准确性管理

- **月度校准**：实际值与预测值对比，调整概率权重
- **偏差分析**：哪个阶段预测偏差最大，重点改进
- **滚动预测**：每月更新，保持3个月滚动预测窗口

## 五、漏斗管理与铁三角

### 5.1 不同角色的漏斗关注点

| 角色 | 重点关注 | 管理动作 |
|------|---------|---------|
| **AR（客户经理）** | 线索数量、机会点转化 | 广开喇叭口，客户关系拓展 |
| **SR（解决方案经理）** | 方案竞争力、赢单率 | 标前引导，方案差异化 |
| **FR（交付经理）** | 交付可行性、回款率 | 交付风险评估，回款跟进 |
| **销售管理者** | 整体漏斗形状、预测准确性 | 资源调配，瓶颈突破 |

### 5.2 漏斗审查会议

| 会议 | 频率 | 参与人 | 议题 |
|------|------|--------|------|
| 线索审查 | 周度 | AR+市场 | 线索跟进状态、新线索来源 |
| 机会点审查 | 周度 | 铁三角 | 项目进展、竞争态势、风险 |
| 漏斗总览 | 月度 | 销售管理层 | 漏斗形状、转化率、预测 |

## 六、漏斗管理常见问题

### 6.1 问题一：喇叭口不够大

**症状**：线索不足，销售无项目可做
**根因**：[[MTL]]流程不健全，营销投入不足
**对策**：
- 加强[[MTL]]流程建设，扩大线索来源
- 建立全员营销机制
- 激活沉睡客户和休眠线索

### 6.2 问题二：线索质量差

**症状**：线索多但转化率极低
**根因**：ICP不清晰，市场销售标准不对齐
**对策**：
- 明确ICP和MQL/SQL标准
- 建立线索评分机制
- 市场销售定期对齐

### 6.3 问题三：赢单率低

**症状**：投标多但中标少
**根因**：标前引导不足，方案竞争力弱
**对策**：
- 强化标前引导——80%结果在发标前决定
- 提升SR方案能力
- 做好竞争分析和差异化

### 6.4 问题四：有单无利

**症状**：合同多但不赚钱
**根因**：低价竞标，合同质量差
**对策**：
- 建立"赢盈并重"的合同质量标准
- 加强ATA签约评审
- 从卖产品到卖价值（TCO/TVO）

## 七、CRM中的漏斗管理

### 7.1 CRM系统支撑

| 功能 | 说明 |
|------|------|
| 线索管理 | 线索录入、评分、分配、跟进 |
| 商机管理 | 商机阶段、金额、概率、预期成交日 |
| 销售过程 | 阶段推进、动作记录、文档管理 |
| 漏斗报表 | 各阶段数量、金额、转化率 |
| 预测报表 | 加权预测、月度趋势、偏差分析 |

### 7.2 数据质量要求

漏斗管理的有效性取决于数据质量：
- **及时性**：项目状态变更后24小时内更新
- **准确性**：金额、概率、时间如实填写
- **完整性**：每个商机都有阶段、金额、预计时间
- **一致性**：全员对阶段定义理解一致

## 八、薄云咨询的应用建议

### 诊断应用
- 绘制客户当前的销售漏斗，计算各阶段转化率
- 与行业标杆对比，识别瓶颈环节
- 评估预测准确性，分析偏差来源

### 辅导应用
- 帮助客户定义漏斗阶段和进入标准
- 建立漏斗审查会议机制
- 设计加权预测模型
- 在CRM中配置漏斗管理和报表
