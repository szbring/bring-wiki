# LTC流程架构 — 三个阶段与九个二级流程

> LTC（Lead to Cash，从线索到回款）是华为三大主业务流程之一，承载从销售线索发现到现金回款的端到端价值创造全过程。

---

<!-- chunk: ltc流程架构.一-ltc在华为流程体系中的定位 -->
## 一、LTC在华为流程体系中的定位

```
MTL（市场→线索）  →  LTC（线索→回款）  →  ITR（问题→解决）
     营销                销售                服务
```

LTC的核心使命：**把市场产生的线索，通过规范化的销售流程，转化为高质量的合同和回款，实现客户满意和公司盈利的双赢。**

华为LTC流程变革历时10年（2007-2017），EMT总结认为："LTC变革发明了交易通，把整个交易系统打通了。"

## 三、阶段一：管理线索（ML）

### 3.1 核心目标

**广开喇叭口**——更早介入客户的采购流程，让更多线索进入可视、可管理的范围，做到更早的客户沟通、更好的技术匹配。

### 3.2 线索的定义与分类

**线索**：客户对特定产品或服务的潜在购买意愿，是项目早期在客户内部的信息表现形式。

| 线索类型 | 特征 | 跟进策略 |
|---------|------|---------|
| **机会线索** | 有明确预算和启动时间 | 优先跟进，快速转机会点 |
| **重要线索** | 有需求但预算不确定 | 重点培育，引导预算 |
| **一般线索** | 有潜在需求但时间遥远 | 定期触达，维持关系 |
| **无价值线索** | 不匹配ICP或无真实需求 | 关闭或转交 |

### 3.3 线索管理流程

```
收集线索 → 验证线索 → 生成线索 → 申请线索决策 → 培育线索 → 跟踪线索 → 转机会点/关闭
```

**收集线索**：12种来源

| 来源 | 说明 | 特点 |
|------|------|------|
| 市场活动 | 行业峰会/研讨会/自办活动 | 高质量，成本高 |
| 客户转介绍 | 老客户推荐 | 最高质量，最低成本 |
| 官网/数字化 | SEO/SEM/内容下载 | 大量，需筛选 |
| 销售自开发 | 陌生拜访/行业扫描 | 依赖销售个人能力 |
| 合作伙伴 | 渠道伙伴推荐 | 覆盖面广 |
| 政策/招标 | 政府采购/招投标信息 | 公开透明 |
| 客户服务 | [[ITR]]问题中发现新需求 | 老客户拓展 |
| 行业研究 | 研究报告/行业动态 | 预判需求 |

**验证线索**：用"错位验证法"——由线索提交人以外的其他人员与客户对接人多角度确认，避免单点沟通导致误判。

**培育线索**：通过解决方案销售逻辑模型，制定培育策略，将潜在购买意愿转化为确定的需求。

### 3.4 线索→机会点的判断标准

| 判断维度 | 线索 | 机会点 |
|---------|------|--------|
| 购买意愿 | 潜在的 | 确定的 |
| 预算 | 无或不确定 | 有明确预算 |
| 启动时间 | 不确定 | 有确定时间 |
| 决策链 | 未知或不完整 | 已识别关键决策人 |

## 四、阶段二：管理机会点（MO）

### 4.1 核心目标

将培育好的线索，通过销售沟通、引导、投标等动作完成合同签订，同时管理合同质量，确保项目高质量交付。**变不确定为确定**。

### 4.2 机会点管理流程

```
验证机会点 → 管理决策链 → 标前引导 → 制定提交标书 → 谈判签约
```

### 4.3 验证机会点（立项决策ATI）

这是销售流程中**最关键的控制节点**：

| 评估维度 | 负责角色 | 评估内容 |
|---------|---------|---------|
| 客户关系 | AR（客户经理） | 客户关系深度、决策链覆盖 |
| 解决方案 | SR（解决方案经理） | 需求匹配度、方案竞争力 |
| 交付可行性 | FR（交付经理） | 交付风险、资源可用性 |
| 商务 | AR | 竞争策略、盈利空间 |

**立项决策的好处**：
1. **筛选项目**——将成功率高的项目挑选出来，聚焦投入
2. **模拟考试**——对销售团队的一次全面检视，发现短板补强

### 4.4 标前引导

> **80%的结果在发标前就已经决定了。**

标前引导的核心动作：
- **以解决方案为核心对客户进行全方位引导**
- 让客户将我们的优势写进标书
- 在标书中设置门槛，对竞争对手形成屏蔽
- 扩大项目范围（从1期到2期3期），做大盘子

### 4.5 谈判与签约

- 中标后签订合同前，还有一次评审决策点（ATAC）
- 评估合同盈利水平、工期保障、成本风险
- 如果综合评估存在风险，可以选择放弃
- 从ATB（投标准备）开始计提至少5%的风险准备金

### 4.6 项目分级分类

从**价值区域、价值客户、价值产品**三个维度确定机会点级别：

| 级别 | 高层支持 | 研发支持 | 项目预算 | 奖金系数 |
|------|---------|---------|---------|---------|
| S级 | CEO站台 | 定制开发 | 充足 | 高 |
| A级 | VP支持 | 方案支持 | 适中 | 中高 |
| B级 | 总监支持 | 标准方案 | 有限 | 中 |
| C级 | 无额外支持 | 标准产品 | 无 | 低 |

## 五、阶段三：管理合同执行（MCE）

### 5.1 核心目标

**契约化交付，保障有效回款**。很多企业回款困难不是因为客户不想付，而是因为合同履行不到位，总留尾巴导致回款缓慢。

### 5.2 合同执行流程

```
合同交接 → 合同履行 → 管理变更 → 关闭评价合同
```

**合同交接**：通过合同交底会实现项目指挥权从销售到交付的转移。

**合同履行**：交付经理FR牵头，客户经理AR负责回款和客户关系维护，解决方案经理SR负责方案澄清。

**管理变更**：
- 客户需求变化时启动变更评审（ATAC）
- 重点测算变更增加的成本是否在风险准备金范围内
- 考验客户线和交付线的谈判能力

**关闭评价合同**：
- 完成交付所有环节的复查
- 评价合同签订质量和履行质量
- 评价结果作为改进流程的输入

### 5.3 合同质量评价维度

| 维度 | 评价内容 |
|------|---------|
| 盈利水平 | 毛利率、净利润率 |
| 交付质量 | 是否按时交付、是否一次性验收通过 |
| 变更次数 | 客户要求整改的次数 |
| 回款情况 | 回款及时性、应收账款天数 |
| 客户满意度 | 客户对交付结果的满意程度 |

## 六、九个二级流程

华为LTC的9个二级流程：

| 序号 | 二级流程 | 所属阶段 | 核心内容 |
|------|---------|---------|---------|
| 1 | 管理线索 | ML | 收集、验证、培育、分发线索 |
| 2 | 管理机会点 | MO | 验证、立项、标前引导、投标 |
| 3 | 管理合同执行 | MCE | 交付、变更管理、回款 |
| 4 | 管理销售决策 | 全流程 | AT系列决策点设置和授权 |
| 5 | 管理线索到机会点转化 | ML→MO | 线索培育成熟转机会点的标准 |
| 6 | 管理机会点到合同转化 | MO→MCE | 投标到签约的流程管理 |
| 7 | 管理客户关系 | 全流程 | [[MCR]]客户关系管理体系 |
| 8 | 管理销售资源 | 全流程 | 铁三角资源调配和成本核算 |
| 9 | 管理销售质量 | 全流程 | 流程遵从度、审计、改进 |

## 七、铁三角组织运作

### 7.1 铁三角定义

CC3（Customer Centric Three）——以客户为中心的项目团队：

| 角色 | 全称 | 职责 | 牵头阶段 |
|------|------|------|---------|
| **AR** | Account Responsibility | 客户满意度负责，客户关系管理 | 线索→机会点 |
| **SR** | Solution Responsibility | 解决方案负责，方案设计和竞争 | 标前引导→投标 |
| **FR** | Fulfill Responsibility | 交付负责，合同履行和回款 | 合同执行→关闭 |

### 7.2 铁三角在LTC中的协同

```
线索阶段：AR牵头，SR/FR不同程度介入
  ↓
机会点阶段：AR→SR过渡，方案引导SR牵头
  ↓
投标阶段：SR牵头，AR/FR协同
  ↓
合同执行阶段：FR牵头，AR/FR延续到合同关闭
```

### 7.3 铁三角高效运作的四个关键

1. **共同KPI**——端到端流程中的协作牵引
2. **能力培养和管理**——归属、招聘、培养、认证体系
3. **授权与决策**——项目发起人授权，解决内部冲突
4. **人力成本核算和使用效率**——资源标价、上架、调配

## 八、LTC的决策机制

### 8.1 AT系列决策点

| 决策点 | 名称 | 位置 | 决策内容 |
|--------|------|------|---------|
| ATI | 立项决策 | 机会点验证 | 做不做这个项目 |
| ATB | 投标准备 | 标书制定 | 投不投标，投什么标 |
| ATP | 评审决策 | 标书提交前 | 标书质量是否达标 |
| ATA | 签约决策 | 中标后 | 签不签合同 |
| ATAC | 变更决策 | 合同执行中 | 接不接受变更 |

### 8.2 授权原则

- 决策前移，充分授权，使一线更快响应客户
- 不同级别项目对应不同授权层级
- 决策效率与风险控制平衡

## 九、薄云咨询的应用建议

### 诊断切入点
- 检查销售漏斗各阶段转化率——哪个环节是瓶颈
- 评估是否有立项决策机制——还是什么项目都做
- 诊断铁三角是否名副其实——还是销售一个人单打独斗
- 检查合同质量——是否"赢"了单但"亏"了钱

### 辅导重点
- 建立线索→机会点的转化标准和SLA
- 设计ATI立项决策机制和评审要素
- 推动铁三角组织从"名存"到"实至"
- 建立合同质量评价和复盘机制

### 常见误区
- 什么项目都做——缺乏立项筛选，资源分散
- 重签约轻交付——合同签完就不管了，交付问题频发
- 销售单打独斗——没有铁三角协同，组织能力无法发挥
- 标前不引导——等客户发标再投标，80%已经输了
