# LTC关键角色与铁三角

> 华为铁三角作战模式：AR/SR/FR如何在LTC中协同赢单

---

<!-- chunk: ltc关键角色与铁三角.一-铁三角的起源与背景 -->
## 一、铁三角的起源与背景

### 1.1 铁三角的诞生

2007年前后，华为在非洲、拉美等海外市场持续碰壁：产品方案有竞争力、价格也不贵，但就是拿不到大项目。问题出在哪里？

深入分析后，华为发现根本原因在于**作战模式的内耗**：销售只关心签合同、技术只关心方案演示、交付只管项目实施，三方各自为战，在关键时刻无法形成合力。客户感受是：跟华为打交道要见三批不同的人，问问题还互相踢皮球。

2008年，任正非提出"铁三角"理念，要求在关键客户的作战体系中，形成以**客户经理（AR）、解决方案经理（SR）、交付经理（FR）**为核心的三角形"铁板一块"的作战单元，三个角色"如同铁板一样连在一起"，共进退。

### 1.2 铁三角与LTC流程的关系

铁三角不是一种组织形式，而是一种**协同作战机制**，它在LTC（线索到回款）流程的各个阶段都发挥核心作用：

```
LTC流程阶段    铁三角主导角色    协同要点
─────────────────────────────────────────────
线索发现       AR主导            SR提供技术洞察，FR提供交付案例
机会评估       AR主导+SR参与     评估技术可行性与交付复杂度
方案开发       SR主导            AR负责客户关系协调，FR确认交付承诺
投标报价       三方联合           统一对外口径，价格/方案/交付一体化
合同谈判       AR主导            SR技术支撑，FR交付条款支撑
项目交付       FR主导            AR维护客户高层关系，SR支持技术变更
回款跟进       AR主导            FR确保交付达到验收条件
```

## 二、铁三角三大角色详解

### 2.1 AR（Account Representative）— 客户经理

**核心定位**：客户关系的主要负责人，铁三角的"灵魂"角色

#### 核心职责
- **客户关系管理**：建立并维护客户全层级关系网络（决策层、管理层、执行层）
- **商机识别与推进**：发现客户潜在需求，推动机会进入销售漏斗
- **商务谈判**：主导价格谈判和合同条款协商
- **回款管理**：跟进账款回收，处理商务纠纷
- **竞争情报**：收集竞争对手信息，制定竞争策略

#### 核心能力要求
| 能力维度 | 具体要求 |
|---------|---------|
| 客户洞察 | 能够深入理解客户的业务目标、政治格局和个人动机 |
| 关系建设 | 善于建立信任，维护长期客户关系 |
| 商务谈判 | 能够在保护公司利益的前提下促成交易 |
| 全局协调 | 能够协调内部资源，为客户提供最优解决方案 |

#### AR的成功要素
优秀的AR通常有一个"客户关系地图"——清楚地知道客户内部谁是决策者、谁是影响者、谁是障碍者，并为每个关键人物制定个性化的关系发展策略。

在华为的实践中，AR对重点客户会制定**关系发展计划（Relationship Development Plan）**，按照"了解-接受-认可-支持"四个层次推进客户关系，而不是简单地"维护关系"。

---

### 2.2 SR（Solution Representative）— 解决方案经理

**核心定位**：技术与业务价值的桥梁，铁三角的"大脑"角色

#### 核心职责
- **需求理解**：深入理解客户的业务需求和技术挑战
- **方案设计**：设计匹配客户需求的整体解决方案
- **价值呈现**：清晰阐述解决方案的业务价值（ROI/TCO分析）
- **技术标准影响**：参与客户技术标准和招标规格的制定（影响标准制定）
- **竞争差异化**：分析竞争对手方案，突出自身方案优势

#### 核心能力要求
| 能力维度 | 具体要求 |
|---------|---------|
| 技术深度 | 深入理解自身产品/方案的技术能力和边界 |
| 业务理解 | 能够将技术能力与客户业务场景关联，转化为业务价值语言 |
| 方案设计 | 能够设计系统性的整体解决方案，而非单点产品推销 |
| 演讲呈现 | 能够面向客户技术决策者进行有说服力的技术方案演讲 |

#### SR的成功要素
顶尖SR最重要的能力是**"先理解，再建议"**——在提出方案前，深入调研客户的业务痛点、技术现状和预算约束，而不是直接推产品。

华为内部有一个判断SR水平的标准：SR能不能在没有产品手册的情况下，用客户的语言（业务语言，而非技术语言）讲清楚"为什么我们的方案对你最合适"。

---

### 2.3 FR（Fulfillment Representative）— 交付经理

**核心定位**：交付承诺的履行者，铁三角的"手脚"角色

#### 核心职责
- **交付规划**：在售前阶段参与评估交付可行性，提供交付方案建议
- **资源协调**：统筹项目所需的各类内外部资源（人员、物料、服务商）
- **项目管理**：主导项目实施，确保按时、按质、按价格完成交付
- **风险管理**：识别和处理交付过程中的风险与问题
- **验收管理**：推动项目验收，为后续回款创造条件
- **客户成功**：确保客户真正从解决方案中获得价值

#### 核心能力要求
| 能力维度 | 具体要求 |
|---------|---------|
| 项目管理 | 熟练掌握项目管理方法，能够有效管控进度、质量和成本 |
| 技术能力 | 对方案有足够深的技术理解，能够指导实施团队 |
| 问题解决 | 在复杂环境下快速识别和解决问题 |
| 客户沟通 | 能够管理客户期望，处理交付过程中的摩擦 |

#### FR的成功要素
优秀FR有一个核心能力：**"承诺管理"**——在售前阶段就参与进来，对SR和AR做出的商业承诺进行可行性把关，避免"虚假承诺"留下的交付雷区。

很多企业的问题是：销售为了拿单，做出超出公司能力的承诺；到了交付阶段，FR发现根本无法实现，导致客户投诉、项目亏损。铁三角机制的重要价值之一，就是让FR在承诺做出之前就介入，提前对齐可行性。

## 三、铁三角的协同机制

### 3.1 "一个声音对外"原则

铁三角最核心的运作原则：**在客户面前永远是一个声音**。

这意味着：
- 任何对客户的承诺，必须在三角内部事先对齐
- 遇到客户质疑，三方不互相甩锅，共同承担
- 客户提出需求变更，AR不能擅自答应，必须与SR/FR协商后统一回复

### 3.2 三角内部分工与决策机制

| 事项类型 | 主导角色 | 其他角色 |
|---------|---------|---------|
| 商务条款变更 | AR主导 | SR/FR提供可行性意见 |
| 技术方案调整 | SR主导 | AR评估对商务的影响，FR评估对交付的影响 |
| 交付资源追加 | FR主导 | AR与客户商定费用，SR评估技术影响 |
| 紧急问题响应 | 三方联合 | 快速拉通决策，统一对客户口径 |

### 3.3 周期性同步机制

铁三角需要建立定期同步机制，确保信息对称：

**日常同步（每日/每周）**
- 关键客户动态更新
- 待决策事项对齐
- 客户投诉/需求快速响应

**阶段性评估（月度）**
- 机会推进状态回顾
- 赢单率分析
- 竞争策略调整

**客户关系盘点（季度）**
- 客户关系健康度评估
- 下一季度战略规划
- 资源需求预测

## 四、铁三角在LTC关键阶段的应用

### 4.1 机会识别与评估阶段

**核心挑战**：如何判断一个机会是否值得投入？

铁三角联合完成**机会评估**（Opportunity Assessment），从三个维度打分：

| 评估维度 | 主导角色 | 评估要点 |
|---------|---------|---------|
| 商务可行性 | AR | 客户预算、采购决策流程、合同风险 |
| 技术可行性 | SR | 需求匹配度、方案可行性、竞争差异化 |
| 交付可行性 | FR | 资源可用性、交付复杂度、风险可控性 |

三方共同决定：是否立项推进（GO/NO-GO）。

**华为实践**：在推行铁三角机制后，华为的"无效投标"比例大幅下降——因为FR的早期参与阻止了大量"拿不下来"或"接了亏损"的机会。

### 4.2 方案开发与标书准备阶段

**SR主导**，AR和FR配合，形成"铁板一块"的标书：

- SR负责技术方案和价值主张
- AR负责商务条款和竞争策略
- FR负责项目实施计划和资源承诺

关键：标书中的每一个承诺，都经过三方内部对齐，避免后期"翻车"。

### 4.3 合同谈判阶段

**AR主导**谈判，但SR和FR必须在现场或随时待命：

- 客户对技术细节提出疑问 → SR即时响应
- 客户要求调整交付条款 → FR即时评估可行性
- 客户要求降价 → AR与SR/FR快速商讨底线

**避免的错误**：AR独自谈判，答应了SR和FR无法兑现的条款。

### 4.4 交付与回款阶段

**FR主导**交付，AR和SR保持参与：

- AR维护客户高层关系，在出现重大问题时能够直接上报和协调
- SR在客户提出技术变更需求时，快速评估影响并给出建议
- AR和FR联合推进验收签字和回款

## 五、铁三角的常见失效模式

### 失效模式1：三角变成"接力棒"

**表现**：销售拿单后直接"甩给"交付，FR不了解销售承诺，客户对交付结果不满意。
**根本原因**：FR没有在售前阶段参与，或参与但没有话语权。
**解决方案**：强制要求FR在合同签署前完成交付可行性确认（Delivery Commitment Review）。

### 失效模式2：三角内部互相甩锅

**表现**：项目出问题后，AR说"技术方案不行"，SR说"销售承诺太多"，FR说"资源不够"。
**根本原因**：没有建立真正的共同责任机制，KPI各自独立。
**解决方案**：将客户满意度和项目整体利润作为三角的共同考核指标。

### 失效模式3：强SR弱化AR/FR

**表现**：技术实力强的SR主导一切，AR沦为"打单工"，FR变成"执行工"。
**根本原因**：能力不均衡 + 考核机制不合理。
**解决方案**：根据项目类型合理分配话语权；确保AR和FR的培养投入。

### 失效模式4：铁三角只在大项目中用

**表现**：铁三角只在战略大客户中执行，中小客户仍然是"单打独斗"。
**后果**：中小客户体验差，大量"小而美"的机会被浪费。
**解决方案**：建立轻量级铁三角（一人兼多角）适配不同规模的项目。

## 六、非华为企业的铁三角落地路径

### 阶段一：试点导入（1-3个月）

- 在2-3个重点客户中试点铁三角机制
- 明确AR/SR/FR角色分工，制定协同SOP
- 建立周例会机制，养成协同习惯

### 阶段二：机制固化（3-6个月）

- 将铁三角评估引入机会立项流程
- 调整KPI，增加共同绩效指标（客户满意度、项目利润）
- 培训全体销售/技术/交付人员理解铁三角理念

### 阶段三：文化融合（6-12个月）

- 铁三角协同成为日常工作方式
- 建立最佳实践案例库，定期分享
- 将铁三角合作评价纳入绩效体系

## 七、薄云咨询的应用价值

**铁三角诊断价值**：
- 快速判断客户的销售/交付体系是否有明显的"孤岛"问题
- 识别"有销售、无技术"或"有技术、无销售"的能力失衡问题

**铁三角辅导价值**：
- 帮助客户从"职能协作"升级为"目标协同"
- 设计铁三角KPI体系和协同流程
- 通过实战陪练加速能力建立
