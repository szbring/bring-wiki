# 铁三角（AR/SR/FR）

## 定义
华为 LTC 的核心项目级运作组织，由客户经理（AR, Account Responsible）、解决方案经理（SR, Solution Responsible）、交付经理（FR, Fulfilling Responsible）三人共同对项目负责，端到端覆盖 LTC 全流程。

## 输入 / 输出
- **输入**：客户机会 + 销售线索 + 公司战略与产品组合
- **输出**：项目级销售方案、合同、交付计划、回款；客户关系深化
- **职责分工**：
  - **AR（客户经理）**：客户关系、客户需求理解、商务谈判、回款负责
  - **SR（解决方案经理）**：技术方案、产品配置、与研发协同
  - **FR（交付经理）**：交付计划、资源协调、风险预案、客户验收

## 角色
- **三角内部**：AR/SR/FR 共同决策，无谁主谁次（强调"共担"）
- **协同方**：财经、法务、供应链、技术专家、生态伙伴
- **客户侧**：对应客户决策链的经办/技术/采购等角色

## 上下游
- **上游**：销售线索分配 → AR 牵头组建铁三角
- **下游**：合同执行后 FR 主导交付，AR 维系客户关系，SR 反馈产品改进
- **关口**：在 [[id:huawei.ltc.concept.ati]] / [[id:huawei.ltc.concept.atb]] / [[id:huawei.ltc.concept.atc]] / [[id:huawei.ltc.concept.atac]] 各阶段共同承担评审

## 典型误区
- **三人必须全职**：SMB 没那么多资源——关键是"三类职责到位"，可以兼任，但 KPI 必须分别考核
- **AR 一言堂**：AR 决定一切，SR/FR 听命——失去铁三角共担的本质
- **只在大项目用铁三角**：小项目不组——结果交付出问题没人管
- **铁三角不与客户决策链对齐**：自己内部分工 OK，但没匹配客户经办/技术/采购——沟通错位
