# 销售漏斗管理服务包

> 帮助企业建立销售漏斗管理体系，实现从"凭感觉销售"到"数据驱动销售"的转型。

---

<!-- chunk: 销售漏斗管理服务包.一-服务定位 -->
## 一、服务定位

**适用场景**：
- 销售过程不可视，管理者不知道项目进展到什么程度
- 销售预测不准，偏差率>30%
- 不知道线索-机会点-合同各阶段转化率
- 销售资源分配凭感觉，没有数据支撑

---

<!-- chunk: 销售漏斗管理服务包.二-服务产品线 -->
## 二、服务产品线

| 服务包 | 周期 | 人天 | 报价 | 适用场景 |
|--------|------|------|------|---------|
| 漏斗诊断 | 2周 | 3-5天 | 8-12万 | 首次诊断，识别瓶颈 |
| 漏斗体系建立 | 4-6周 | 10-15天 | 20-30万 | 从0到1建立漏斗管理 |
| 漏斗优化 | 4周 | 8-10天 | 15-20万 | 已有漏斗但效果不佳 |

<!-- chunk: 销售漏斗管理服务包.三-sp1-漏斗诊断 -->
## 三、SP1 — 漏斗诊断

### 3.1 交付物

- 销售漏斗现状图（各阶段数量、金额、转化率）
- 瓶颈分析报告
- 与行业标杆的差距对比
- 改进建议优先级

### 3.2 诊断步骤

1. 从CRM导出最近6个月的销售数据
2. 按线索→机会点→方案→合同→回款各阶段统计
3. 计算各阶段转化率和平均停留时间
4. 分析预测准确性（预测vs实际）
5. 识别瓶颈环节和根因
6. 提出改进建议

<!-- chunk: 销售漏斗管理服务包.四-sp2-漏斗体系建立 -->
## 四、SP2 — 漏斗体系建立

### 4.1 交付物

| 交付物 | 内容 |
|--------|------|
| 漏斗阶段定义 | 各阶段名称、进入标准、主责角色 |
| 转化率目标 | 各阶段目标转化率（分阶段设定） |
| 漏斗审查机制 | 周/月审查会议设计和议程 |
| 预测模型 | 加权预测法，赢单概率权重设定 |
| CRM配置建议 | 漏斗管理所需的CRM配置方案 |

### 4.2 实施步骤

```
Step 1：定义漏斗阶段（1周）
  - 确定阶段划分和进入标准
  - 与销售团队对齐理解

Step 2：设定目标转化率（1周）
  - 基于行业标杆和客户现状
  - 设定可实现的分阶段目标

Step 3：建立审查机制（2周）
  - 设计线索审查/机会点审查/漏斗总览会议
  - 确定参会人和议程

Step 4：CRM配置（2周）
  - 配置商机阶段和必填字段
  - 配置漏斗报表和预测报表

Step 5：培训和试运行（2周）
  - 培训销售团队使用新漏斗标准
  - 试运行4周后校准
```

## 五、SP3 — 漏斗优化

### 5.1 典型优化场景

| 场景 | 优化方向 |
|------|---------|
| 喇叭口不够大 | 优化MTL流程，增加线索来源 |
| 线索转化率低 | 优化ICP和线索评分标准 |
| 赢单率低 | 加强标前引导和方案竞争力 |
| 预测不准 | 校准赢单概率权重，提升数据质量 |
| 流速慢 | 识别并消除流程中的等待和返工 |

### 5.2 优化效果追踪

| 指标 | 优化前 | 3个月后 | 6个月后 | 目标 |
|------|--------|---------|---------|------|
| 线索→机会点转化率 | | | | |
| 赢单率 | | | | |
| 预测偏差率 | | | | |
| 平均销售周期 | | | | |

## 六、验收标准

- [ ] 漏斗阶段定义清晰，全员理解一致
- [ ] CRM中商机阶段和必填字段配置完成
- [ ] 漏斗报表可自动生成
- [ ] 漏斗审查会议稳定运行（周度/月度）
- [ ] 预测偏差率<30%
