# 销售漏斗诊断

> 通过量化分析销售漏斗各阶段转化率、流速和预测准确性，识别[[LTC]]瓶颈，为LTC流程优化提供数据支撑。

---

<!-- chunk: 销售漏斗诊断.一-诊断框架 -->
## 一、诊断框架

### 1.1 五维诊断模型

| 维度 | 诊断内容 | 数据来源 |
|------|---------|---------|
| **漏斗形状** | 各阶段数量和金额分布 | CRM系统 |
| **转化率** | 各阶段间转化率，与标杆对比 | CRM数据计算 |
| **流速** | 各阶段平均停留时间 | CRM时间戳 |
| **预测准确性** | 预测值vs实际值偏差 | 月度对比 |
| **质量指标** | 赢单率、合同利润率、回款及时性 | 合同和财务数据 |

### 1.2 诊断输出

- 漏斗瓶颈位置
- 转化率差距（vs行业标杆）
- 流速异常项目清单
- 预测偏差分析
- 改进建议优先级

## 二、漏斗形状诊断

### 2.1 健康漏斗的特征

```
健康漏斗                    不健康漏斗（喇叭口小）
  ██                          █
  ███                         ██
  █████                       ███
  ███████                     █████
  ██████████                  ███████
  线索→回款                    线索→回款
```

### 2.2 漏斗形状分类

| 形状 | 特征 | 诊断 | 处方 |
|------|------|------|------|
| **倒三角** | 线索多回款少 | 转化率全线偏低 | 从最大瓶颈环节开始改进 |
| **纺锤形** | 中间宽两头窄 | 喇叭口不够+出口堵塞 | 加强MTL+提升回款 |
| **正三角** | 线索少但转化高 | 喇叭口太小 | 扩大线索来源 |
| **漏斗堵塞** | 某阶段突然变窄 | 该阶段是瓶颈 | 重点突破该环节 |

## 三、转化率诊断

### 3.1 各环节转化率评估

| 转化环节 | 实际值 | 行业平均 | 华为标杆 | 差距 | 原因分析 |
|---------|--------|---------|---------|------|---------|
| 线索→机会点 | | 25% | 35% | | |
| 机会点→方案 | | 65% | 75% | | |
| 方案→合同 | | 30% | 40% | | |
| 合同→回款 | | 85% | 95% | | |
| **端到端** | | **4%** | **10%** | | |

### 3.2 转化率根因分析

| 低转化率环节 | 可能原因 | 验证方法 |
|------------|---------|---------|
| 线索→机会点低 | ICP不准/线索质量差/培育不力 | 分析线索来源分布和MQL→SQL转化 |
| 机会点→方案低 | 立项通过率低/项目质量差 | 分析立项拒绝原因 |
| 方案→合同低 | 赢单率低/竞争不过 | 分析输单原因和竞争对手 |
| 合同→回款低 | 交付问题/客户拖延/合同条款不利 | 分析逾期原因分布 |

## 四、流速诊断

### 4.1 各阶段停留时间

| 阶段 | 平均停留时间 | 标杆时间 | 超时项目数 | 超时原因 |
|------|------------|---------|-----------|---------|
| 线索 | | 2个月 | | |
| 机会点 | | 1.5个月 | | |
| 方案 | | 2个月 | | |
| 合同 | | 0.5个月 | | |
| 回款 | | 2个月 | | |

### 4.2 流速异常识别

**僵尸项目**：某阶段停留超过标杆2倍的项目
- 处理方式：逐个复盘，决定推进/挂起/关闭

**流速突然变慢**：某月整体流速明显下降
- 可能原因：季节性因素/团队士气/竞争加剧

### 5.2 常见偏差模式

| 模式 | 特征 | 原因 | 改进 |
|------|------|------|------|
| 系统性偏高 | 预测总大于实际 | 销售乐观偏差 | 校准概率权重 |
| 系统性偏低 | 预测总小于实际 | 销售保守或压单 | 激励及时赢单 |
| 大幅波动 | 偏差方向不稳定 | 数据质量差 | 规范CRM录入 |
| 最后一月突变 | 最后一月大幅调整 | 期末突击 | 加强过程管理 |

## 八、快速诊断问题清单

1. **「销售漏斗各阶段数量分布如何？哪个阶段项目最多/最少？」**
   - 某阶段特别窄 → 该环节是瓶颈

2. **「从线索到成单，平均要多长时间？哪个阶段最耗时？」**
   - 总周期超6个月 → 流程效率低

3. **「赢单率是多少？输单的主要原因是什么？」**
   - 赢单率<25% → 标前引导和方案竞争力不足

4. **「销售预测的准确性如何？偏差通常在什么方向？」**
   - 偏差>30% → 预测模型需校准

5. **「S/A级客户的赢单率和普通客户有什么区别？」**
   - 差距大 → 客户分级策略需要调整
