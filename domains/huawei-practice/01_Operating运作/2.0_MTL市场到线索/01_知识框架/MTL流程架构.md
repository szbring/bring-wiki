# MTL流程架构 — 六大模块详解

> MTL（Market to Lead，从市场到线索）是华为面向客户的四大价值创造流程之一，作为[[IPD]]与LTC之间的桥梁，解决研发与销售的衔接问题。

---

<!-- chunk: mtl流程架构.一-mtl在华为流程体系中的定位 -->
## 一、MTL在华为流程体系中的定位

```
IPD（把产品做出来）  →  MTL（培育市场、生成线索）  →  LTC（成交回款）
     研发                     营销                        销售
```

MTL的核心使命用20个字概括：**培育市场、牵引研发、生成线索、打造品牌、促进增长**。

MTL与IPD、LTC的本质区别：
- **IPD**承载研发流，输出产品
- **LTC**承载人财物，输出合同和回款
- **MTL**不承载资金流和物流，输出的是**方法和方法论**——教会企业如何细分市场、选择客户、设计营销手段、管理线索

## 二、六大模块总览

```
┌─────────────────────────────────────────────────────────────────┐
│                        MTL 流程全景                              │
│                                                                 │
│  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐  ┌──────┐  │
│  │  MI  │→│  MM  │→│  JI  │→│  SCE │→│  DG  │→│  MQM │  │
│  │市场  │  │市场  │  │联合  │  │销售  │  │激发  │  │营销  │  │
│  │洞察  │  │管理  │  │创新  │  │赋能  │  │需求  │  │质量  │  │
│  └──────┘  └──────┘  └──────┘  └──────┘  └──────┘  └──────┘  │
│                                                                 │
│  输入：市场环境、客户需求          输出：高质量销售线索             │
└─────────────────────────────────────────────────────────────────┘
```

| 模块 | 英文全称 | 核心职责 | 关键输出 |
|------|---------|---------|---------|
| MI | Market Insight | 分析市场环境和客户需求，支撑市场管理决策 | 市场洞察报告、竞争情报 |
| MM | Market Management | 细分市场选择、投资组合、市场计划制定 | 细分市场战略、H1/H2/H3组合 |
| JI | Joint Innovation | 与客户联合创新，挖掘真实场景需求 | 创新方案、商机线索 |
| SCE | Seller Competence Enablement | 基于角色的销售赋能，统一客户形象 | 赋能资料、培训体系 |
| DG | Demand Generation | 规划并执行营销活动，激发客户购买意愿 | 销售线索、品牌影响力 |
| MQM | Marketing Quality Management | 全流程质量监控，持续推动营销变革 | 质量评估体系、改进措施 |

<!-- chunk: mtl流程架构.三-模块一-市场洞察-mi -->
## 三、模块一：市场洞察（MI）

### 3.1 核心目的

通过持续开展市场洞察工作，分析市场环境和客户需求，为市场管理决策提供输入。MI不是市场部自己的任务，而是汇集一线市场、销售、交付、财经等多方面信息和力量的协同工作。

### 3.2 两大分析维度

**市场维度**：

| 分析要素 | 说明 | 典型工具 |
|---------|------|---------|
| 市场细分 | 识别不同客户群体的独特需求和市场潜力 | MACS原则 |
| 市场趋势 | 提前布局，顺应行业发展潮流 | 炒作曲线（Hype Cycle） |
| 市场规模与增长 | 量化市场机会，支撑投资决策 | TAM/SAM/SOM |
| 竞争分析 | 了解竞争对手的优势与劣势 | 雷达图、18要素分析 |
| 颠覆性力量 | 识别可能颠覆行业的力量 | PESTEL |

**客户维度**：

| 分析要素 | 说明 |
|---------|------|
| 客户细分 | 将客户群体划分为具有相似需求的细分市场 |
| 客户行为分析 | 理解购买决策过程和消费习惯 |
| 客户价值分析 | 识别高价值客户，合理分配资源 |
| 客户声音（VOC） | 系统收集和分析客户反馈 |
| 客户体验测量 | 量化客户在各触点的体验 |
| 社交媒体分析 | 捕捉客户需求和市场动态 |

### 3.3 MI的节奏管理

| 洞察类型 | 频率 | 输出 |
|---------|------|------|
| 战略级市场洞察 | 年度 | 年度市场洞察报告，输入SP/BP |
| 细分市场更新 | 季度 | 季度市场动态简报 |
| 竞争情报 | 月度 | 月度竞情简报 |
| 客户反馈整合 | 实时 | 客户反馈数据库 |

## 四、模块二：市场管理（MM）

### 4.1 核心职责

MM是MTL的"大脑"，负责基于洞察做出市场选择和资源分配决策：

1. **细分市场选择与优先级排序** — 基于市场吸引力和自身能力评估，完成投资组合
2. **产品和解决方案规划** — 识别客户需求和痛点，规划开发有竞争力的方案
3. **市场计划制定与绩效管理** — 制定目标细分市场的市场计划并管理绩效

### 4.2 H1/H2/H3三层业务组合

华为在MM中创造性地将细分市场分为三类，对应不同的策略重点：

| 类别 | 市场特征 | 核心策略 | MTL重点模块 |
|------|---------|---------|------------|
| **H1 成熟市场** | 核心业务，市场份额稳定 | 绩效管理，收割利润 | SCE（销售赋能）促进销售 |
| **H2 成长市场** | 快速增长，竞争加剧 | 扩大影响力，抢占份额 | DG（激发需求）扩大品牌 |
| **H3 探索市场** | 新兴机会，需要培育 | 投入探索，联合创新 | MI+JI（洞察+创新）产出方案 |

> **关键原则**：华为要求70%的业务收入来自明星区域（H2），如果所有业务都在金牛区域（H1），说明公司虽大但不强，缺乏未来规划。

### 4.3 MM与IPD的衔接

MM的细分市场战略直接输入[[IPD]]的产品规划：
- MI/MM识别客户需求 → 输入[[IPD]]的需求管理
- MM的产品组合决策 → 输入[[IPD]]的产品路标规划
- MM的市场计划 → 输入[[IPD]]的GTM（Go-to-Market）策略

## 五、模块三：联合创新（JI）

### 5.1 核心理念

JI强调与客户建立深度合作关系，**不是闭门造车式的研发创新，而是基于客户真实使用场景的联合创新**。

### 5.2 JI的工作方式

```
识别关键客户 → 深入客户场景 → 共同定义需求 → 联合开发验证 → 创新转化为商机
```

- **挖掘真实使用场景**：研发团队深入客户现场，理解业务流程和运营模式
- **确保需求真实理解**：与客户技术人员和业务专家共同探讨方案
- **持续创新**：不是一次性交付，而是持续迭代优化
- **转化为商机**：创新成果不仅是技术成果，更要转化为商业机会

### 5.3 JI的价值

| 价值维度 | 说明 |
|---------|------|
| 深度理解需求 | 比客户自己更理解其痛点 |
| 构建客户粘性 | 联合创新建立深度合作关系 |
| 提前布局市场 | 引领行业趋势，占据先发优势 |
| 生成高质量线索 | 创新过程中自然产生销售线索 |

## 六、模块四：销售赋能（SCE）

### 6.1 四个核心问题

SCE围绕"4W"框架设计赋能体系：

| 问题 | 回答 | 示例 |
|------|------|------|
| **给谁赋能（Who）** | 基于角色的赋能 | 客户经理、售前、解决方案经理、市场经理 |
| **谁来赋能（By Whom）** | 不同内容来源 | 销售赋能专员、细分市场经理、产品管理 |
| **赋能什么（What）** | 按销售历程组织 | 主打胶片、白皮书、案例、视频 |
| **如何赋能（How）** | 多样化传递渠道 | 培训网站、移动应用、组织会议、考试 |

### 6.2 营销资料的范式转变

传统营销资料以产品为导向，强调"我行""我能"；MTL要求营销资料转变为**说客户的语言、说客户能听懂的语言、说能为客户带来价值的语言**。

| 维度 | 传统方式 | MTL方式 |
|------|---------|---------|
| 视角 | 以产品为中心 | [[以客户为中心_Ch1_为客户服务是唯一理由|以客户为中心]] |
| 语言 | 技术语言 | 客户业务语言 |
| 逻辑 | 我有什么功能 | 你有什么痛点、我如何解决 |
| 目标 | 展示技术实力 | 帮助客户成功 |

### 6.3 赋能资料的分类

按客户购买历程组织赋能资料：

| 购买阶段 | 赋能资料类型 | 目的 |
|---------|------------|------|
| 认知阶段 | 行业白皮书、趋势报告 | 建立思想领导力 |
| 兴趣阶段 | 解决方案介绍、成功案例 | 激发兴趣和信任 |
| 评估阶段 | 技术方案、POC案例 | 支撑决策评估 |
| 决策阶段 | 商务方案、ROI分析 | 推动签约决策 |

## 七、模块五：激发需求（DG）

### 7.1 核心任务

通过规划并执行系列营销活动，激发客户购买意愿，为产品、服务或解决方案**生成销售线索**。

### 7.2 三类营销活动

| 活动层级 | 对象 | 内容特点 | 目标 |
|---------|------|---------|------|
| **公司级** | 全行业 | 标准化内容，思想领导力 | 提升品牌形象和行业影响力 |
| **区域级** | 区域客户 | 地域特色内容 | 提高区域市场知名度和份额 |
| **客户群级** | 单个客户群 | 客户化解决方案内容 | 牵引客户创造线索和机会 |

### 7.3 五个子场景

1. **战略对话与匹配** — 提升思维领导力，与客户高层对齐战略方向
2. **扩大产业空间** — 线索生成与市场培育，做大盘子
3. **营销引导客户规划** — 线索生成与孵化，影响客户采购规划
4. **支撑销售项目拓展** — 同时提升品牌，配合销售打单
5. **支撑普遍客户关系** — 收集客户痛点，为产品优化提供依据

### 7.4 DG的关键原则

- 营销活动要**统一规划**，预算集中管控
- 考核标准不是做了多少活动，而是**活动获得了什么效果**
- 营销模式以**客户购买心理为主线**，将所有活动串联起来
- 结合MTL和[[LTC]]**扩大喇叭口**：MTL把一个喇叭口变成若干个，LTC深入客户价值链继续扩大

## 八、模块六：营销质量管理（MQM）

### 8.1 核心职责

MQM是MTL的"守护者"，覆盖战略规划、预算及资源配置、流程优化、运营管理等多个方面，确保整个营销体系持续改进。

### 8.2 质量管理的四个层面

| 层面 | 内容 |
|------|------|
| **战略规划** | 确保营销战略与企业整体战略紧密结合 |
| **预算及资源** | 合理分配营销资源，提高利用效率 |
| **流程优化** | 简化和优化营销流程，减少不必要环节和成本 |
| **人员技能** | 通过培训和实践提高营销人员专业素养 |

### 8.3 质量评估维度

从MI、MM、JI、SCE、DG各关键模块入手，建立评估体系：

- **MI质量**：洞察报告的准确性和决策支撑度
- **MM质量**：细分市场选择的精准度和投资回报
- **SCE质量**：赋能资料的使用率和销售满意度
- **DG质量**：营销活动的线索转化率和ROI
- **端到端质量**：从洞察到线索的整体转化效率

## 九、MTL带来的三大变革

### 9.1 作战阵型变革

**业务前移**：营销业务从办公室编辑材料，前移到一线直接接触客户和市场获取信息。

**建立三个中心**：依托MTL在区域/代表处构建：
1. **市场策略中心** — 执行MI和MM
2. **营销活动中心** — 落地DG
3. **销售赋能中心** — 支撑SCE

### 9.2 作战方式变革

建立有战斗力的**解决方案机动部队**：既沉淀典型场景的标准化方案，又能快速支援一线完成客户化方案设计。

### 9.3 核心能力变革

| 能力 | 内涵 |
|------|------|
| **看得更远** | 比客户看得更远的洞察能力，从第三方视角思考 |
| **做得更好** | 面向客户解决实际痛点，面向产品反哺驱动开发 |
| **呈现价值** | 设计不可拒绝的价值主张，体系化营销策划与执行 |

## 十、MTL与其他流程的关系

```
          ┌──────┐
          │ DSTE │ 战略管理
          └──┬───┘
             ↓ 输入
┌──────┐  ┌──┴───┐  ┌──────┐
│ IPD  │←─│ MTL  │─→│ LTC  │
│研发  │  │营销  │  │销售  │
└──────┘  └──────┘  └──────┘
 ↑需求输入   ↓线索输出    ↓
 产品构想   高质量线索   商机转化
```

- **MTL → [[IPD]]**：MI/MM输出客户需求，牵引产品规划
- **MTL → [[LTC]]**：DG/SCE输出高质量线索，支撑销售转化
- **DSTE → MTL**：战略解码为MTL提供方向和目标
- **MTL → [[ITR]]**：客户反馈输入洞察，形成闭环

<!-- chunk: mtl流程架构.十一-薄云咨询的应用建议 -->
## 十一、薄云咨询的应用建议

### 诊断切入点
- 评估客户是否有系统性的市场洞察机制（而非靠销售人员个人感觉）
- 检查市场策略是否有数据支撑，还是"拍脑袋"决策
- 诊断营销与销售是否割裂——市场产生线索销售不跟，或销售抱怨线索质量差

### 辅导重点
- 帮助客户建立MI到MM的闭环，从"被动收集线索"转为"主动挖掘线索"
- 培训营销团队使用4W框架设计SCE赋能体系
- 设计DG营销活动的规划-执行-评估闭环

### 常见误区
- 把MTL等同于"做市场活动"——忽视了MI/MM的基础作用
- 忽略H3探索市场投入——只看眼前成熟市场，缺乏未来增长点
- 赋能资料仍然以产品为中心——没有转变为客户语言
