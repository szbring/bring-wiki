# MTL成熟度评估详细版

> 基于华为[[MTL]]实践，构建五级成熟度模型，从L1到L5逐级提升，支撑企业营销体系诊断和变革规划。

---

<!-- chunk: mtl成熟度评估详细版.一-成熟度模型总览 -->
## 一、成熟度模型总览

| 级别 | 名称 | 核心特征 | 典型表现 |
|------|------|---------|---------|
| **L1** | 线索自生自灭 | 没有市场部，线索靠销售自找或老客户介绍 | 销售就是"跑关系"，没有营销概念 |
| **L2** | 营销活动驱动 | 有市场部做活动/内容，但线索质量差、转化低 | 市场只管做活动，销售嫌线索不靠谱 |
| **L3** | 流程对齐 | 市场与销售对齐，MQL/SQL清晰，线索有跟进机制 | 市场产生线索，销售有SLA跟进 |
| **L4** | 数据驱动 | 能量化各渠道ROI，预测线索量，优化投放 | 营销决策基于数据，而非经验 |
| **L5** | 智能赋能 | AI辅助个性化营销，精准触达目标客户 | 全流程自动化+智能化 |

<!-- chunk: mtl成熟度评估详细版.二-l1-l2-评估维度 -->
## 二、L1 → L2 评估维度

### 2.1 组织维度

| 评估项 | L1表现 | L2表现 | 评分(1-5) |
|--------|--------|--------|----------|
| 市场部门设置 | 无专职市场团队 | 有市场部但职能单一（活动/PR） |  |
| 市场人员能力 | 无 | 有执行能力但缺乏策略能力 |  |
| 市场与销售关系 | 无协作机制 | 有协作但市场被边缘化 |  |

### 2.2 流程维度

| 评估项 | L1表现 | L2表现 | 评分(1-5) |
|--------|--------|--------|----------|
| 线索来源 | 老客户介绍/销售自开发 | 有官网/活动等渠道但量少 |  |
| 线索管理 | 无系统，Excel管理 | 有CRM但线索来源单一 |  |
| ICP定义 | 没有或模糊 | 有但未数字化验证 |  |

---

<!-- chunk: mtl成熟度评估详细版.三-l2-l3-评估维度 -->
## 三、L2 → L3 评估维度

### 3.1 核心差距：市场与销售割裂

L2最常见的痛点是"市场销售两张皮"：
- 市场部追求线索数量，销售部抱怨线索质量
- 市场活动做了不少，但真正转化为商机的寥寥无几
- 没有统一的线索质量标准

### 3.2 评估项

| 评估项 | L2表现 | L3表现 | 评分(1-5) |
|--------|--------|--------|----------|
| MQL/SQL定义 | 无统一定义 | MQL/SQL标准清晰，双方认可 |  |
| 线索评分 | 无 | 有评分模型（属性分+行为分） |  |
| 线索SLA | 无 | 市场销售有SLA协议（48h跟进） |  |
| 线索反馈机制 | 销售不反馈线索质量 | 定期反馈线索质量和转化情况 |  |
| 市场销售对齐会 | 无 | 月度对齐，共同优化ICP和渠道 |  |

<!-- chunk: mtl成熟度评估详细版.四-l3-l4-评估维度 -->
## 四、L3 → L4 评估维度

### 4.1 核心差距：经验驱动→数据驱动

L3有流程但缺乏数据支撑，决策仍靠经验：
- 不知道哪个渠道线索质量最高
- 不知道哪类内容转化效果最好
- 无法预测下季度线索量

### 4.2 评估项

| 评估项 | L3表现 | L4表现 | 评分(1-5) |
|--------|--------|--------|----------|
| 渠道ROI分析 | 只看线索数量 | 能追踪线索→商机→成交全链路ROI |  |
| 内容效果追踪 | 看阅读量/下载量 | 看内容贡献的线索转化率 |  |
| 营销预测 | 无预测能力 | 基于历史数据预测线索量 |  |
| A/B测试 | 不做 | 系统性测试不同营销策略效果 |  |
| 营销归因模型 | 无 | 多触点归因（首次/末次/线性） |  |

---

<!-- chunk: mtl成熟度评估详细版.五-l4-l5-评估维度 -->
## 五、L4 → L5 评估维度

### 5.1 核心差距：人工驱动→智能驱动

L5是MTL的理想态，实现营销的智能化和自动化：
- AI预测最可能转化的线索
- 个性化内容推荐
- 自动化线索培育流程
- 智能化营销活动策划

### 5.2 评估项

| 评估项 | L4表现 | L5表现 | 评分(1-5) |
|--------|--------|--------|----------|
| 线索评分 | 规则评分 | AI预测评分（机器学习） |  |
| 内容个性化 | 统一内容 | 基于客户画像的个性化内容 |  |
| 线索培育 | 人工 nurturing | 自动化 nurturing 流程 |  |
| 营销活动优化 | 人工分析后优化 | 实时AI优化投放策略 |  |
| 预测性分析 | 描述性分析 | 预测性+处方性分析 |  |

## 六、六模块专项评估

### 6.1 MI（市场洞察）成熟度

| 级别 | 特征 |
|------|------|
| L1 | 无市场洞察，靠销售反馈和个人判断 |
| L2 | 偶尔做市场调研，但不系统 |
| L3 | 有年度洞察报告，覆盖五看模型 |
| L4 | 季度更新洞察，数据支撑决策 |
| L5 | 实时洞察，AI辅助竞争情报分析 |

### 6.2 MM（市场管理）成熟度

| 级别 | 特征 |
|------|------|
| L1 | 无细分市场概念，什么单都接 |
| L2 | 有细分但靠经验，无SPAN矩阵 |
| L3 | 用SPAN矩阵做细分市场选择 |
| L4 | H1/H2/H3组合管理，动态调整 |
| L5 | AI辅助细分市场发现和组合优化 |

### 6.3 SCE（销售赋能）成熟度

| 级别 | 特征 |
|------|------|
| L1 | 无赋能体系，销售靠自悟 |
| L2 | 有产品培训但无体系 |
| L3 | 按4W框架设计赋能体系 |
| L4 | 赋能效果可量化，持续优化 |
| L5 | 个性化赋能路径，AI推荐学习内容 |

### 6.4 DG（激发需求）成熟度

| 级别 | 特征 |
|------|------|
| L1 | 无营销活动，纯靠销售 |
| L2 | 有零散活动但无统一规划 |
| L3 | 活动按三类五场景统一规划 |
| L4 | 活动ROI可量化，预算优化配置 |
| L5 | AI辅助活动策划和效果预测 |

## 七、诊断访谈问题清单

### 7.1 CEO/业务负责人

1. 公司现在的线索主要从哪里来？占比最大的是什么渠道？
2. 营销投入占营收比例是多少？您认为这个投入产出合理吗？
3. 公司有没有清晰的目标客户画像（ICP）？
4. 品牌建设在公司战略中的位置是什么？
5. 您认为市场团队对业绩的贡献有多大？

### 7.2 营销负责人

1. 你们有哪些营销渠道？各渠道的线索量和质量如何？
2. MQL/SQL的定义是什么？销售是否认可？
3. 市场产生的线索，销售跟进率是多少？平均转化周期多长？
4. 你们有市场洞察的例行机制吗？频率如何？
5. 营销活动的ROI如何衡量？

### 7.3 销售负责人

1. 销售团队目前的线索来源分布？
2. 市场产生的线索质量如何？赢单率多少？
3. 你们和市场团队怎么协作？有什么痛点？
4. 销售团队的赋能体系是什么？效果如何？
5. 你们有销售漏斗管理吗？各阶段转化率多少？
