# MTL诊断工具

<!-- chunk: mtl诊断工具.成熟度评估 -->
## 成熟度评估

| 级别 | 特征 |
|---|---|
| L1 | 没有市场部，线索靠销售自找或老客户介绍 |
| L2 | 有市场部，做活动/内容，但线索质量差、转化低 |
| L3 | 市场与销售对齐，MQL/SQL清晰，线索有跟进机制 |
| L4 | 数据驱动，能量化各渠道ROI，预测线索量 |
| L5 | AI辅助，个性化营销，精准触达目标客户 |

---

<!-- chunk: mtl诊断工具.快速诊断问题 -->
## 快速诊断问题

1. **「公司现在的线索主要从哪里来？占比最大的是什么渠道？」**
   - 全靠老客户介绍/销售自开发 → L1-L2
   - 有官网/活动但量少 → L2
   - 多渠道且有ROI数据 → L3+

2. **「市场团队和销售团队怎么协作？市场产生的线索销售跟进率多少？」**
   - 互不搭理/销售嫌线索质量差 → 市场与销售割裂
   - 有流程但跟进不及时 → L2-L3
   - 有SLA和反馈机制 → L3+

3. **「公司有没有清晰的目标客户画像（ICP）？」**
   - 没有或模糊 → L1-L2
   - 有但未数字化 → L2-L3
   - 有且指导市场投放决策 → L3+

## 薄云咨询自身的MTL实践

**薄云最有效的[[MTL]]方式**：
- 思想领导力内容（华为方法论深度文章/白皮书）
- 行业活动演讲（建立专家影响力）
- 客户案例传播（口碑扩散）
- 知识星球/社群运营（目标客户社区）

> 咨询公司本质上靠「可信度」获取线索，而非广告投放。
