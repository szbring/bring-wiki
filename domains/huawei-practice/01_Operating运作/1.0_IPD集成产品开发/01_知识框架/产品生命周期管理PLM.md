# 产品生命周期管理（PLM）

> **定位**：PLM是[[IPD]]体系在DCP4之后的延伸，负责产品从上市到退市的全生命周期商业经营管理。

---

<!-- chunk: plm-overview.一-plm定义与战略意义 -->
## 一、PLM定义与战略意义

### 1.1 华为PLM定位

**产品生命周期管理（Product Lifecycle Management，PLM）** 是指从产品立项批准（DCP0）到最终退市（EoL）全过程的系统性管理体系。

华为将PLM定义为：
- **不只是技术管理**：PLM不仅管理产品的技术演进，更管理产品的**商业价值实现**
- **IPD的后半程**：IPD主流程（DCP0-DCP3）解决"如何做对产品"，PLM（DCP4之后）解决"如何持续经营好产品"
- **贯穿全局的视角**：需要市场、研发、销售、财务、供应链协同，而非研发单打独斗

### 1.2 PLM在IPD体系中的位置

```
IPD主流程（立项→开发→发布）
    DCP0 → DCP1 → DCP2 → DCP3 → DCP4 → 生命周期管理（PLM）
                                          ↑
                                    PLM正式介入
                                    （年度PLCR评审机制启动）
```

### 1.3 核心目标

| 目标维度 | 具体内容 |
|---------|---------|
| **商业价值最大化** | 在每个生命周期阶段做出最优投资决策 |
| **资源高效配置** | 成熟期产品与新兴产品的资源动态平衡 |
| **有序退出** | 避免产品自然死亡，做到体面退市 |
| **客户持续满意** | 全生命周期的客户体验与服务保障 |

> **华为核心理念**：产品不是永生的，关键是在对的时机做对的事——该投就投，该收就收，该退就退。

<!-- chunk: plm-overview.二-华为产品生命周期阶段划分 -->
## 二、华为产品生命周期阶段划分

### 2.1 五阶段模型

```
市场规模
    ▲
    │        ╭────────╮
    │      ╭─╯  成熟期  ╰─╮
    │    ╭─╯              ╰─╮
    │  ╭─╯  成长期    衰退期  ╰─╮
    │╭─╯导入期                退市期╰─
    └──────────────────────────────→ 时间
```

### 2.2 各阶段特征与管理重点

#### 🚀 导入期（Launch）

| 维度 | 内容 |
|------|------|
| **市场特征** | 市场认知度低，客户教育成本高，竞争对手少 |
| **管理重点** | 上市推广、样板客户建设、渠道搭建 |
| **研发重点** | 快速响应市场反馈，修复上市缺陷 |
| **财务特征** | 投入大于产出，处于亏损或盈亏平衡期 |
| **关键动作** | 发布产品路标、获取早期客户、建立参考案例 |

#### 📈 成长期（Growth）

| 维度 | 内容 |
|------|------|
| **市场特征** | 快速扩张，竞争者大量涌入，价格战开始 |
| **管理重点** | 规模化复制、版本迭代加速、市场份额争夺 |
| **研发重点** | 功能增强、平台化、成本优化 |
| **财务特征** | 收入快速增长，利润率逐步改善 |
| **关键动作** | 扩大渠道、差异化竞争、提升交付能力 |

#### 💼 成熟期（Maturity）

| 维度 | 内容 |
|------|------|
| **市场特征** | 市场饱和，增速放缓，竞争格局相对稳定 |
| **管理重点** | 市场份额维护、成本精细化、客户深耕 |
| **研发重点** | 稳定性提升、成本削减、准备下一代产品 |
| **财务特征** | 利润率最高，现金流最佳，是公司现金牛 |
| **关键动作** | 优化产品组合、提升客户NPS、规划替代产品 |

#### 📉 衰退期（Decline）

| 维度 | 内容 |
|------|------|
| **市场特征** | 市场萎缩，替代技术/产品出现，客户流失 |
| **管理重点** | 控制投资、引导客户迁移、维护核心大客户 |
| **研发重点** | 最小化维护，仅做安全补丁 |
| **财务特征** | 收入下滑，维护成本上升，贡献毛利持续压缩 |
| **关键动作** | 启动替代产品、发布EoL路标、锁定迁移计划 |

#### ⛔ 退市期（End of Life, EoL）

| 维度 | 内容 |
|------|------|
| **市场特征** | 停止接受新订单，仅服务存量客户 |
| **管理重点** | 有序退出、合同履约、客户平滑迁移 |
| **研发重点** | 停止新功能开发，仅提供安全维护 |
| **财务特征** | 最小化退市成本，确保合规退出 |
| **关键动作** | 执行EoL通知、维护期截止、合同收尾 |

## 三、生命周期关键决策机制（DCP4）

### 3.1 年度产品生命周期评审（PLCR）

**PLCR（Product Lifecycle Review）** 是DCP4机制的核心载体：

- **频率**：每年至少一次，重大市场变化时随时触发
- **参与方**：IPMT（集成产品管理团队）、IRB（投资评审委员会）、产品经理
- **评审输入**：
  - 产品年度经营报告（收入/利润/市场份额）
  - 竞争态势分析
  - 下一代产品准备情况
  - 客户满意度数据

### 3.2 三种决策类型

| 决策 | 触发条件 | 后续行动 |
|------|---------|---------|
| **继续（Go）** | 产品仍有商业价值，指标健康 | 确认下一年投资计划 |
| **调整（Adjust）** | 部分指标偏差，需策略转向 | 制定改善计划，设定观察期 |
| **加速退市（Accelerate EoL）** | 市场丧失、亏损恶化、替代产品就绪 | 启动EoL流程 |

### 3.3 IRB/IPMT的角色与职责

| 角色 | 职责 |
|------|------|
| **IRB（投资评审委员会）** | 最终投资决策权；审批重大退市决策；资源跨产品线调配 |
| **IPMT（集成产品管理团队）** | 日常产品经营管理；PLCR主持与执行；提交退市建议给IRB |
| **产品经理** | 产品经营数据准备；制定生命周期策略；执行IPMT决策 |

### 3.4 退市规划（EoL Planning）

华为标准退市流程要求**提前18-24个月启动规划**：

```
T-24月：识别退市信号，提交退市建议
T-18月：IPMT/IRB决策，发布EoL公告
T-12月：停止接受新客户订单
T-6月：停止定制开发
T-0月：停止所有技术支持
T+X月：合同履约结束，完全退市
```

**EoL通知要素**：
- 退市产品范围（版本/型号）
- 关键时间节点
- 推荐替代方案
- 客户迁移支持政策

## 四、华为产品组合管理

### 4.1 产品线投资组合视图

华为通过**产品组合管理**统筹管理整个产品线的投资分配，避免"全面撒网、重点落空"：

- **战略产品**：优先投资，保持技术领先
- **主力产品**：维持投入，维护市场份额
- **现金奶牛产品**：控制投入，最大化现金提取
- **候选退市产品**：最小化投入，准备退出

### 4.2 BCG矩阵在产品组合中的应用

```
市场增长率
    ▲
高  │  问题（Question Mark）  │  明星（Star）
    │  高增长/低份额          │  高增长/高份额
    │  需要重点投资或放弃      │  持续投资，保持领先
    │─────────────────────────│─────────────────────
低  │  瘦狗（Dog）            │  现金牛（Cash Cow）
    │  低增长/低份额          │  低增长/高份额
    │  考虑退市               │  提取现金，控制投入
    └──────────────────────────→ 市场份额（相对）
         低                       高
```

| 类型 | 华为管理策略 |
|------|------------|
| **明星** | 大力投资，巩固领先地位；准备向现金牛过渡 |
| **现金牛** | 控制研发投入；最大化利润提取；资金反哺明星和问题产品 |
| **问题** | 评估市场潜力；选择性重点投入或快速退出 |
| **瘦狗** | 设定退市时间表；最小化维护成本 |

### 4.3 S曲线管理（新老产品接替节奏）

华为强调"在第一条S曲线顶峰之前，启动第二条S曲线"：

```
收益
  ▲          第一代产品 ╮
  │      ╭──────────────╯    第二代产品
  │    ╭─╯         ╭──────────────────
  │  ╭─╯         ╭─╯
  │╭─╯         ╭─╯
  └──────────────────────────────→ 时间
                ↑
           接替窗口：
           在此时启动新产品，
           避免收益断崖
```

**接替节奏原则**：
- 新产品在老产品成熟期中期开始立项
- 新老产品重叠期提供迁移过渡窗口
- 避免新产品未就绪老产品就退市（收益断层）

### 4.4 跨产品线资源优化

- 从**现金牛产品**抽取资金，投入**明星/问题产品**研发
- 从**衰退产品**释放人员，充实**成长产品**团队
- 共享平台/CBB能力，降低跨产品重复投入

## 五、产品版本管理策略

### 5.1 版本层级体系

华为产品版本遵循三层体系：

| 版本类型 | 定义 | 发布节奏 |
|---------|------|---------|
| **主版本（Major Version）** | 重大功能升级或架构变更 | 12-18个月/版 |
| **商用版本（Commercial Version）** | 功能增强、性能优化 | 3-6个月/版 |
| **补丁版本（Patch/Hotfix）** | 缺陷修复、安全补丁 | 随需发布 |

### 5.2 版本生命周期（VLM）

每个版本独立管理其生命周期：

```
版本发布 → 全面支持期 → 维护支持期 → 延长支持期 → 版本退市
              (1-2年)      (1-2年)       (可选)       (EoV)
```

- **全面支持期**：新功能、缺陷修复、安全补丁全面提供
- **维护支持期**：仅提供安全补丁和严重缺陷修复
- **延长支持期**：针对重要客户的付费延伸服务
- **版本退市（EoV）**：停止所有支持

### 5.3 异步开发模式（多版本并行）

```
主版本V3（开发中）    ─────────────────────→
商用版本V2.2（维护）  ──────────────╮
商用版本V2.1（退市）  ────────╮
商用版本V2.0（退市）  ────╮
```

**核心挑战**：
- 资源分配：新版本开发 vs 旧版本维护的平衡
- 版本收敛：避免版本碎片化造成维护成本爆炸
- 客户引导：推动客户向新版本迁移

## 六、薄云咨询视角：帮助客户建立PLM体系

### 6.1 客户常见问题诊断

| 典型问题 | 表现症状 | 影响 |
|---------|---------|------|
| **没有PLM意识** | 产品"自然死亡"，无退市规划 | 维护成本失控，客户投诉 |
| **决策机制缺失** | 无PLCR，靠领导"感觉"决策 | 产品投资错位，资源浪费 |
| **版本失控** | 版本数量爆炸，维护压力巨大 | 研发资源70%在维护，30%在创新 |
| **组合管理缺失** | 所有产品平均分配资源 | 重点产品得不到足够投入 |
| **退市无序** | 临时宣布退市，客户措手不及 | 客户流失，品牌受损 |

### 6.2 薄云PLM建设路径

```
阶段一：诊断（1-2周）
  → 产品生命周期成熟度评估
  → 现有产品组合健康度分析

阶段二：设计（2-4周）
  → 建立生命周期阶段定义与标准
  → 设计PLCR评审机制
  → 制定版本管理规范

阶段三：落地（1-3个月）
  → 试点产品线PLM实施
  → 建立产品组合管理看板
  → 培训IPMT/产品经理

阶段四：持续优化
  → 年度PLCR机制运行
  → 数字化PLM系统建设
```

### 6.3 核心价值主张

> **薄云咨询帮助客户实现**：
> - **战略对齐**：产品投资决策与公司战略一致
> - **资源聚焦**：将有限资源集中在高价值产品
> - **有序经营**：每个产品都有清晰的经营路标
> - **体面退场**：退市有序，客户满意，品牌无损
