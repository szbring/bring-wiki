# 产品生命周期成熟度评估

<!-- chunk: plm-maturity-assessment.一-评估目的与适用场景 -->
## 一、评估目的与适用场景

**评估目的**：帮助企业快速识别产品生命周期管理体系的能力短板，明确改进优先级。

**适用场景**：
- 企业正在引入或优化[[IPD]]，需要评估PLM成熟度基线
- 产品线扩张后，需要建立统一的生命周期管理标准
- 已有PLM体系，但产品退市混乱或投资决策缺乏依据

**评估输入**：产品负责人、研发总监、市场/销售VP、财务BP访谈及相关制度文档

---

<!-- chunk: plm-maturity-assessment.二-五级成熟度模型 -->
## 二、五级成熟度模型

| 级别 | 标签 | 核心特征 |
|------|------|---------|
| L1 | 无意识 | 没有明确的产品生命周期概念，产品"自然死亡"，无退市规划 |
| L2 | 初步意识 | 有基本的上市/退市概念，但缺乏系统流程，主要靠个人经验 |
| L3 | 流程化 | 建立了PLM基本流程，有阶段评审机制，但执行不一致 |
| L4 | 量化管理 | 各阶段有明确KPI，数据驱动决策，产品组合管理趋于成熟 |
| L5 | 持续优化 | PLM与战略规划深度融合，自动化监控，产品组合动态优化 |

<!-- chunk: plm-maturity-assessment.三-评估维度-六维度 -->
## 三、评估维度（六维度）

### 维度1：生命周期阶段划分清晰度（权重15%）
评估企业是否对产品生命周期各阶段有清晰定义，并能识别产品当前所处阶段。

### 维度2：DCP决策机制完善度（权重20%）
评估产品关键节点（立项、发布、年度评审、退市）的决策机制是否规范。

### 维度3：产品组合管理能力（权重20%）
评估企业管理多产品投资组合的能力，包括资源分配和新老产品接替节奏。

### 维度4：绩效指标体系完整性（权重20%）
评估各生命周期阶段的KPI设置、数据采集和目标达成管理。

### 维度5：退市流程规范度（权重15%）
评估产品退市决策、EoL通知、客户迁移、服务终止的规范程度。

### 维度6：数字化支撑能力（权重10%）
评估PLM系统、产品数据管理工具的使用情况。

<!-- chunk: plm-maturity-assessment.四-评估问卷-30题 -->
## 四、评估问卷（30题）

### 维度1：生命周期阶段划分（5题）

1. 企业是否有正式文件定义产品生命周期的各阶段（导入/成长/成熟/衰退/退市）？
   - [ ] 无定义（0分）
   - [ ] 有非正式共识（1分）
   - [ ] 有正式文件，部分产品遵循（2分）
   - [ ] 有正式文件，所有产品遵循（3分）

2. 产品团队能否准确判断其产品当前处于哪个生命周期阶段？
   - [ ] 不清楚（0分）
   - [ ] 有大致判断，缺乏依据（1分）
   - [ ] 有明确标准和数据支撑（2分）
   - [ ] 实时可见，有自动化仪表盘（3分）

3. 各阶段的进入/退出标准是否明确？
   - [ ] 无标准（0分）
   - [ ] 有部分标准（1分）
   - [ ] 有完整标准（2分）
   - [ ] 标准与绩效系统联动（3分）

4. 产品路标（Roadmap）是否按生命周期阶段规划？
   - [ ] 无产品路标（0分）
   - [ ] 有路标，未区分阶段（1分）
   - [ ] 路标按阶段规划（2分）
   - [ ] 路标与市场洞察、竞品动态联动（3分）

5. 公司是否有跨产品线的生命周期视图？
   - [ ] 无（0分）
   - [ ] 有，但不定期更新（1分）
   - [ ] 有，定期更新（2分）
   - [ ] 实时更新，支持管理层决策（3分）

### 维度2：DCP决策机制（5题）

6. 是否有正式的产品立项评审（类DCP0）机制？
   - [ ] 无，老板拍板（0分）
   - [ ] 有，但不规范（1分）
   - [ ] 有标准流程，跨部门参与（2分）
   - [ ] 有量化的投资标准（IRR/NPV等）（3分）

7. 产品发布前是否有正式的上市就绪评审（类DCP3）？
   - [ ] 无（0分）
   - [ ] 有，但主要看功能（1分）
   - [ ] 涵盖功能、质量、市场就绪（2分）
   - [ ] 含客户验证、支撑体系完整评估（3分）

8. 是否有年度或定期产品生命周期评审（PLCR）？
   - [ ] 无（0分）
   - [ ] 不定期，随机触发（1分）
   - [ ] 年度例行，有议程模板（2分）
   - [ ] 例行+异常触发，决策记录完整（3分）

9. 生命周期评审的决策权是否清晰（谁有权决定继续/加速退市）？
   - [ ] 不清晰（0分）
   - [ ] 由老板个人决定（1分）
   - [ ] 有授权矩阵（2分）
   - [ ] IRB/IPMT集体决策，记录完整（3分）

10. DCP决策是否有明确的数据/分析作为输入？
    - [ ] 无（0分）
    - [ ] 有但不系统（1分）
    - [ ] 有标准分析模板（2分）
    - [ ] 自动化数据采集+分析（3分）

### 维度3：产品组合管理（5题）

11. 是否有整体的产品线投资组合视图（含在研/在售/退市产品）？
    - [ ] 无（0分）
    - [ ] 有，但不完整（1分）
    - [ ] 完整，定期更新（2分）
    - [ ] 实时可见，含财务贡献（3分）

12. 资源（研发人力/市场费用）是否按产品生命周期阶段差异化分配？
    - [ ] 无差异，平均分配（0分）
    - [ ] 有意识区别，但无明确标准（1分）
    - [ ] 有明确的分配规则（2分）
    - [ ] 动态调整，与阶段绩效联动（3分）

13. 是否有新老产品的接替节奏规划（S曲线管理）？
    - [ ] 无（0分）
    - [ ] 有，但较粗糙（1分）
    - [ ] 有正式规划流程（2分）
    - [ ] 与市场份额和技术趋势联动（3分）

14. 产品组合是否有"止血"机制（亏损产品加速退出）？
    - [ ] 无，持续投入（0分）
    - [ ] 有意识，但执行困难（1分）
    - [ ] 有机制，定期执行（2分）
    - [ ] 自动触发止血评审（3分）

15. 共用平台/CBB的复用率是否被追踪和优化？
    - [ ] 无追踪（0分）
    - [ ] 有，但不系统（1分）
    - [ ] 有KPI并定期回顾（2分）
    - [ ] 与产品开发效率目标挂钩（3分）

### 维度4：绩效指标体系（5题）

16. 各产品是否有按生命周期阶段设定的财务目标（收入/毛利/IRR）？
    - [ ] 无（0分）
    - [ ] 有，但只看收入（1分）
    - [ ] 含收入/毛利/现金流（2分）
    - [ ] 全面财务指标，含全生命周期ROI（3分）

17. 市场绩效指标（市场份额/客户满意度/NPS）是否被定期追踪？
    - [ ] 无（0分）
    - [ ] 有，但数据不准确（1分）
    - [ ] 有标准数据源，定期报告（2分）
    - [ ] 实时追踪，预警机制（3分）

18. 研发效率指标（上市周期/缺陷率/版本交付准时率）是否纳入产品绩效？
    - [ ] 无（0分）
    - [ ] 有，但孤立（1分）
    - [ ] 与商业目标关联（2分）
    - [ ] 全面集成，驱动改进（3分）

19. 产品经理/产品总监是否对产品全生命周期商业结果负责？
    - [ ] 只负责功能交付（0分）
    - [ ] 部分负责（1分）
    - [ ] 明确责任，纳入绩效考核（2分）
    - [ ] 与长期激励挂钩（3分）

20. 是否有产品健康度综合评分（技术/市场/财务三维）？
    - [ ] 无（0分）
    - [ ] 非正式评估（1分）
    - [ ] 有标准模型，定期评分（2分）
    - [ ] 自动计算，可视化仪表盘（3分）

### 维度5：退市流程（5题）

21. 退市决策是否有明确的触发标准？
    - [ ] 无，凭感觉（0分）
    - [ ] 有，但不系统（1分）
    - [ ] 有量化标准（如连续X季度亏损）（2分）
    - [ ] 自动触发评审（3分）

22. 是否提前12-24个月发布产品退市通知（EoL Notice）？
    - [ ] 无提前通知（0分）
    - [ ] 有，但时间不够（1分）
    - [ ] 提前12-18个月（2分）
    - [ ] 提前18-24个月，有客户沟通计划（3分）

23. 是否有正式的客户迁移支持方案？
    - [ ] 无（0分）
    - [ ] 有，但不系统（1分）
    - [ ] 有标准迁移方案和支持流程（2分）
    - [ ] 含替代产品推荐+专项服务（3分）

24. 退市后的备件/维护支持期限是否明确承诺给客户？
    - [ ] 无承诺（0分）
    - [ ] 有，但口头承诺（1分）
    - [ ] 有正式书面承诺（2分）
    - [ ] 纳入合同条款，可追溯（3分）

25. 退市过程的执行进度是否被监控？
    - [ ] 无监控（0分）
    - [ ] 有，非正式（1分）
    - [ ] 有正式里程碑追踪（2分）
    - [ ] 自动化监控+异常预警（3分）

### 维度6：数字化支撑（5题）

26. 是否有专用的PLM/产品数据管理系统？
    - [ ] 无，用Excel（0分）
    - [ ] 有，但功能有限（1分）
    - [ ] 有成熟PLM系统（2分）
    - [ ] 与ERP/CRM深度集成（3分）

27. 产品生命周期数据（阶段/KPI/评审记录）是否集中管理？
    - [ ] 分散（0分）
    - [ ] 部分集中（1分）
    - [ ] 统一数据源（2分）
    - [ ] 实时数据湖（3分）

28. 产品路标是否有数字化工具支撑（可视化/多人协作）？
    - [ ] 无（0分）
    - [ ] 有，功能基础（1分）
    - [ ] 有，跨部门协作（2分）
    - [ ] 与战略规划系统联动（3分）

29. DCP评审材料是否有标准化模板且在系统中管理？
    - [ ] 无（0分）
    - [ ] 有模板，分散存储（1分）
    - [ ] 有模板，集中存储（2分）
    - [ ] 系统引导，自动归档（3分）

30. 是否有产品生命周期绩效的自动化报告？
    - [ ] 无（0分）
    - [ ] 手工报告（1分）
    - [ ] 半自动化（2分）
    - [ ] 全自动，实时可见（3分）

<!-- chunk: plm-maturity-assessment.五-评分计算与结果解读 -->
## 五、评分计算与结果解读

### 评分计算

各维度得分 = 该维度各题得分之和 / (题数 × 3) × 100

加权总分 = Σ（各维度得分 × 权重）

### 成熟度对应表

| 加权总分 | 成熟度级别 | 核心改进方向 |
|---------|---------|-----------|
| 0-20分 | L1 无意识 | 建立PLM基本概念，明确生命周期阶段 |
| 21-40分 | L2 初步意识 | 规范DCP评审机制，建立基础指标体系 |
| 41-60分 | L3 流程化 | 强化产品组合管理，量化绩效体系 |
| 61-80分 | L4 量化管理 | 数字化升级，退市流程精细化 |
| 81-100分 | L5 持续优化 | 与战略规划深度融合，AI辅助决策 |

## 六、薄云咨询服务切入点

根据评估结果，薄云建议的改进优先级：

**得分<40（L1-L2）**：先建基础
- 建立产品生命周期阶段定义和DCP评审机制
- 重点：让产品退市有章可循，避免"僵尸产品"

**得分40-70（L3）**：强化量化
- 建立分阶段KPI体系和产品健康度评估
- 重点：用数据驱动产品投资决策

**得分>70（L4-L5）**：数字化深化
- PLM系统升级、AI辅助产品组合优化
- 重点：实时预警和动态资源配置
