# IPD成熟度评估模型

<!-- chunk: ipd-maturity-assessment.五级成熟度定义 -->
## 五级成熟度定义

| 级别 | 名称 | 特征描述 |
|---|---|---|
| L1 | 初始级 | 研发无流程，依赖个人英雄主义，结果不可预期 |
| L2 | 受管理级 | 有基本项目管理，但研发与市场脱节，立项靠经验 |
| L3 | 已定义级 | 有IPD流程框架，Charter+DCP初步建立，PDT开始运作 |
| L4 | 量化管理级 | 流程数据化，研发ROI可量化，决策有数据支撑 |
| L5 | 持续优化级 | 流程持续改进，与战略深度整合，行业最佳实践 |

> **大多数中国企业处于L1-L2，华为目标是L4-L5**
> **薄云咨询帮助客户从L1/L2提升到L3，是最高频的服务场景**

## 评估维度（六维雷达图）

### 1. 立项决策机制
- [ ] 是否有正式的立项评审流程？
- [ ] 评审标准是否清晰（市场/技术/财务三维）？
- [ ] 评审参与者是否覆盖跨职能角色？
- [ ] 是否有明确的Go/Kill决策机制？

**评分标准**：4项全有=L3，3项=L2.5，2项=L2，1项以下=L1

### 2. 需求管理
- [ ] 客户需求是否有系统化收集机制？
- [ ] 需求是否经过优先级排序？
- [ ] 需求变更是否有正式审批流程？
- [ ] 需求是否可追溯到产品功能？

### 3. 跨部门协作
- [ ] 研发与市场是否有固定的协作机制？
- [ ] 是否有跨功能项目团队（[[IPD]]）？
- [ ] PDT Leader是否有实质性的资源调度权？
- [ ] 跨部门冲突是否有清晰的解决机制？

### 4. 项目过程管理
- [ ] 是否有里程碑和阶段评审？
- [ ] 项目进展是否可视化？
- [ ] 风险是否被系统性识别和管理？
- [ ] 项目结束后是否有复盘总结？

### 5. 研发ROI管理
- [ ] 每个项目是否有商业回报预测？
- [ ] 是否追踪实际ROI vs 立项预测？
- [ ] 研发投入是否与产品组合战略挂钩？
- [ ] 是否有产品生命周期管理机制？

### 6. 工具与系统支撑
- [ ] 是否有PLM/研发管理系统？
- [ ] 研发数据是否可追溯？
- [ ] 知识产权管理是否系统化？
- [ ] 研发工具链是否统一规范？

## 快速诊断问卷（访谈版）

### 5分钟快速判断（用于售前）

1. **「你们每年有多少个新产品立项？立项靠什么标准决定？」**
   - 靠感觉/老板拍板 → L1
   - 有讨论但无标准 → L2
   - 有评审流程和标准 → L3+

2. **「研发团队和市场/销售之间，最常见的冲突是什么？」**
   - 「研发做的市场不需要」→ 需求管理缺失，L1-L2
   - 「市场总是改需求」→ 需求变更管理缺失，L2
   - 「基本都能对齐」→ L3+

3. **「一个研发项目，从立项到上市，谁对商业成功负责？」**
   - 「研发负责做出来，销售负责卖」→ 无PDT，L1-L2
   - 「产品经理/PDT Leader」→ L3+

4. **「上一个失败的研发项目，主要原因是什么？」**
   - 「方向错了」→ MM流程缺失
   - 「做出来卖不掉」→ 研发与市场脱节
   - 「超时超预算」→ 项目管理缺失
