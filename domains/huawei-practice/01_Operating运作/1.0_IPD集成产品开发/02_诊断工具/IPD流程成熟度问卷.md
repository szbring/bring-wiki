# IPD流程成熟度问卷（详细版）

<!-- chunk: ipd-maturity-questionnaire.使用说明 -->
## 使用说明
本问卷用于系统化评估企业[[IPD]]成熟度，支持顾问在诊断阶段使用。建议通过访谈方式逐项确认，而非让客户自行填写。

**适用对象**：研发负责人、产品经理、项目经理、市场/销售代表、质量负责人
**访谈时长**：每位受访者60-90分钟
**评分规则**：1-5分制（1=完全不具备，5=行业最佳实践水平）

<!-- chunk: ipd-maturity-questionnaire.一-立项决策机制-权重20 -->
## 一、立项决策机制（权重20%）

### 1.1 立项流程规范性
| # | 评估项 | 1分 | 3分 | 5分 | 得分 |
|---|---|---|---|---|---|
| 1.1.1 | 是否有正式的立项评审流程？ | 无任何评审，老板拍板 | 有评审会议但无标准流程 | 有标准化流程，评审有明确标准 | |
| 1.1.2 | 评审标准是否包含市场/技术/财务三个维度？ | 仅看技术可行性 | 有市场和财务分析但不系统 | 三维论证完整且有数据支撑 | |
| 1.1.3 | 评审参与者是否覆盖跨职能角色？ | 仅研发参与 | 研发+市场参与 | 研发+市场+供应链+财务+质量 | |
| 1.1.4 | 是否有明确的Go/Kill/Hold决策机制？ | 没有，所有项目都做 | 有但很少Kill项目 | 有且严格执行，Kill率≥30% | |
| 1.1.5 | 立项评审是否有文档化输出？ | 无文档 | 有简单记录 | 有完整的Charter和评审纪要 | |

### 1.2 项目选择质量
| # | 评估项 | 1分 | 3分 | 5分 | 得分 |
|---|---|---|---|---|---|
| 1.2.1 | 立项前的市场分析深度？ | 无分析，凭感觉 | 有定性分析 | 有定量市场数据和客户调研 | |
| 1.2.2 | 竞争分析是否系统化？ | 不做竞争分析 | 有简单竞品对比 | 有系统化竞争分析框架 | |
| 1.2.3 | 财务回报预测是否量化？ | 不做财务预测 | 有粗略估算 | 有详细的ROI/IRR测算 | |
| 1.2.4 | 项目Portfolio是否平衡？ | 想做就做，无优先级 | 有简单排序 | 有组合分析和资源平衡 | |

## 二、需求管理（权重20%）

### 2.1 需求收集与分析
| # | 评估项 | 1分 | 3分 | 5分 | 得分 |
|---|---|---|---|---|---|
| 2.1.1 | 需求收集渠道是否系统化？ | 无固定渠道，靠个人 | 有部分渠道但不稳定 | 有多渠道系统化收集机制 | |
| 2.1.2 | 需求是否有分类标准？ | 无分类 | 有简单分类 | 有完整的OR分类体系（$/$$/$$$/Bug/…） | |
| 2.1.3 | 需求优先级排序方法？ | 无排序，谁声音大优先 | 有简单排序（高/中/低） | 有量化优先级模型（如RICE/Kano） | |
| 2.1.4 | 需求追溯性如何？ | 需求与功能无法对应 | 部分需求可追溯 | 全链路可追溯：需求→特性→代码→测试 | |

### 2.2 需求变更管理
| # | 评估项 | 1分 | 3分 | 5分 | 得分 |
|---|---|---|---|---|---|
| 2.2.1 | 需求变更是否有正式审批流程？ | 无，随意变更 | 有审批但不严格 | 有CCB变更控制委员会，严格审批 | |
| 2.2.2 | 需求变更影响评估？ | 不评估影响 | 有粗略评估 | 有系统化影响分析（进度/成本/质量） | |
| 2.2.3 | 需求基线管理？ | 无基线概念 | 有基线但不严格执行 | 有严格的基线管理机制 | |

## 三、跨部门协作（权重20%）

### 3.1 PDT/跨功能团队
| # | 评估项 | 1分 | 3分 | 5分 | 得分 |
|---|---|---|---|---|---|
| 3.1.1 | 是否有跨功能项目团队？ | 无，各做各的 | 有临时协作机制 | 有正式PDT，成员代表各职能 | |
| 3.1.2 | PDT Leader是否有实权？ | 无实权，只是协调员 | 有部分资源调度权 | 有充分的资源调度权和绩效打分权 | |
| 3.1.3 | PDT成员投入度？ | 兼职，随时被拉回部门 | 核心成员50%以上时间投入 | 核心成员70%以上时间投入 | |
| 3.1.4 | 跨部门冲突解决机制？ | 靠领导协调 | 有会议讨论机制 | PDT内部解决，不升级 | |

### 3.2 功能部门支撑
| # | 评估项 | 1分 | 3分 | 5分 | 得分 |
|---|---|---|---|---|---|
| 3.2.1 | 功能部门是否愿意输出人才？ | 不愿意，留能力强的 | 有条件配合 | 有机制保障，输出合格人才 | |
| 3.2.2 | 是否有双轨考核机制？ | 无，只考核部门绩效 | 有项目绩效但权重低 | 双轨考核，项目绩效权重≥50% | |

## 四、项目过程管理（权重15%）

| # | 评估项 | 1分 | 3分 | 5分 | 得分 |
|---|---|---|---|---|---|
| 4.1 | 是否有阶段评审（DCP/TR）？ | 无阶段评审 | 有里程碑但评审不严格 | 有DCP+TR，评审严格执行 | |
| 4.2 | 项目进展是否可视化？ | 无可视化 | 有简单进度报告 | 有项目管理工具，进度实时可见 | |
| 4.3 | 风险管理？ | 无风险管理 | 有风险清单但不跟踪 | 有系统化风险识别、评估、应对和跟踪 | |
| 4.4 | 项目复盘？ | 不做复盘 | 做但不系统 | 有结构化复盘模板，经验入库 | |
| 4.5 | 研发周期可预测性？ | 完全不可预测 | 基本能预测±50% | 可预测±20%以内 | |

## 五、研发ROI管理（权重15%）

| # | 评估项 | 1分 | 3分 | 5分 | 得分 |
|---|---|---|---|---|---|
| 5.1 | 研发投入是否有商业回报预测？ | 无预测 | 有粗略估算 | 每个项目都有详细的财务模型 | |
| 5.2 | 是否追踪实际ROI vs 立项预测？ | 不追踪 | 偶尔追踪 | 系统化追踪，偏差分析 | |
| 5.3 | 研发预算管理？ | 无预算管理 | 有年度预算但无项目级管控 | 有项目级+组合级预算管理 | |
| 5.4 | 产品生命周期管理？ | 上市后不再管理 | 有年度销售分析 | 有完整的生命周期管理和退市机制 | |

## 评分汇总与成熟度定级

### 各维度得分计算

| 维度 | 权重 | 平均分 | 加权得分 |
|---|---|---|---|
| 立项决策机制 | 20% | __ /5 | |
| 需求管理 | 20% | __ /5 | |
| 跨部门协作 | 20% | __ /5 | |
| 项目过程管理 | 15% | __ /5 | |
| 研发ROI管理 | 15% | __ /5 | |
| 工具与系统支撑 | 10% | __ /5 | |
| **总分** | **100%** | | **__ /5** |

### 成熟度定级

| 总分范围 | 成熟度级别 | 特征 | 典型表现 |
|---|---|---|---|
| 1.0-1.5 | L1 初始级 | 研发无流程，依赖个人 | 老板拍板立项，需求靠口头传递 |
| 1.5-2.5 | L2 受管理级 | 有基本管理，但与市场脱节 | 有项目管理但无商业评审 |
| 2.5-3.5 | L3 已定义级 | 有IPD框架，初步运作 | Charter+DCP建立，PDT开始运作 |
| 3.5-4.5 | L4 量化管理级 | 数据驱动，ROI可量化 | 决策有数据支撑，流程可度量 |
| 4.5-5.0 | L5 持续优化级 | 行业最佳实践 | 持续改进，与创新深度融合 |
