# IPD分阶段导入方案

<!-- chunk: ipd-phased-induction.一句话定义 -->
## 一句话定义
[[IPD]]不能一次性全面铺开，必须分阶段、分层次导入，先建立"最小可用框架"，再逐步深化扩展，确保每个阶段都有可验证的交付成果。

---

<!-- chunk: ipd-phased-induction.核心原则 -->
## 核心原则

1. **先僵化，再固化，后优化**——任正非三步曲
2. **先建立框架，再填充细节**——不要试图一步到位
3. **先试点验证，再全面推广**——用真实项目验证流程可行性
4. **先高价值环节，再低价值环节**——聚焦ROI最高的改进点
5. **先流程后系统**——流程跑通了再上系统，不要用系统补流程的窟窿

---

<!-- chunk: ipd-phased-induction.四阶段导入路线图 -->
## 四阶段导入路线图

```
阶段1            阶段2            阶段3            阶段4
基础框架        试点深化         全面推广         持续优化
(2-3个月)      (3-4个月)       (3-4个月)       (持续)
   │              │              │              │
   ▼              ▼              ▼              ▼
Charter+DCP    PDT运作        MM/OR流程       数字化集成
PDT组建        TR评审         绩效体系        敏捷演进
培训宣贯       流程优化        制度固化        行业对标
```

<!-- chunk: ipd-phased-induction.阶段1-基础框架建立-2-3个月 -->
## 阶段1：基础框架建立（2-3个月）

### 目标
建立IPD的"骨架"——Charter评审、DCP决策机制、PDT基本运作规则。

### 适用企业特征
- L1-L2成熟度
- 研发50人以上
- 有多条产品线或并行项目
- 高层认识到研发管理问题但无系统化方案

### 关键动作

#### 第1-2周：现状诊断与框架设计

| 动作 | 说明 | 交付物 |
|---|---|---|
| IPD成熟度评估 | 使用[[IPD流程成熟度问卷]] | 成熟度评级报告 |
| 研发效能诊断 | 使用[[研发效能诊断]]工具 | 研发效能报告 |
| 高层对齐会 | CEO/CTO必须参加，确认变革决心 | 会议纪要 |
| 框架定制 | 基于企业规模定制IPD框架 | IPD框架文件 |

#### 第3-4周：核心机制建立

| 动作 | 说明 | 交付物 |
|---|---|---|
| Charter模板定制 | 简化华为模板，保留核心要素 | Charter模板 |
| DCP节点设计 | 中小企业3个DCP（立项/开发完成/年度评审） | DCP评审标准 |
| IPMT组建 | 确定5-7人评审委员会 | IPMT章程 |
| PDT组建方案 | 确定PDT角色和运作规则 | PDT运作规范 |

#### 第5-8周：培训与准备

| 动作 | 说明 | 交付物 |
|---|---|---|
| 骨干培训 | 研发/产品/市场Leader，1天 | 培训课件 |
| 关键问答 | "这对我有什么好处？" | FAQ文档 |
| 试点项目选择 | 选择2-3个即将立项的新项目 | 试点清单 |
| 工具模板包 | Charter模板、DCP评审表 | 模板包 |

### 阶段1验收标准
- [ ] Charter模板定稿，IPMT审核通过
- [ ] DCP评审标准文档化
- [ ] IPMT和PDT组建完成
- [ ] 关键人员培训完成（出勤率≥80%）
- [ ] 试点项目选定

<!-- chunk: ipd-phased-induction.阶段2-试点深化-3-4个月 -->
## 阶段2：试点深化（3-4个月）

### 目标
在真实项目中跑通IPD流程，发现问题，调整优化，让团队从"被动执行"变为"主动使用"。

### 关键动作

#### 试点项目辅导

| 辅导内容 | 频次 | 说明 |
|---|---|---|
| Charter撰写辅导 | 1-2次/项目 | 顾问手把手辅导PDT Leader |
| DCP评审参与 | 每次必到 | 顾问作为评审观察员+教练 |
| 周跟踪会 | 1次/周 | 跟踪PDT运作问题 |
| 月度复盘 | 1次/月 | 总结经验，调整流程 |

#### 重点深化内容

| 内容 | 说明 | 时机 |
|---|---|---|
| TR技术评审 | 在试点项目中引入TR评审 | DCP0之后 |
| 需求管理 | 建立需求收集和追踪机制 | 项目启动时 |
| 项目计划管理 | WBS分解+里程碑跟踪 | 计划阶段 |
| 风险管理 | 风险识别和应对机制 | 贯穿全程 |

#### 常见问题处理

| 问题 | 处理方式 |
|---|---|
| PDT成员不配合 | 升级IPMT，强调IPMT权威 |
| Charter写不出来 | 顾问手把手辅导，用已有信息填框架 |
| DCP评审走形式 | 要求每个问题有明确答案，不接受模糊 |
| 团队抱怨增加工作量 | 数据对比：试点vs非试点项目的效率和质量 |

### 阶段2验收标准
- [ ] 至少1个试点项目完成DCP0
- [ ] Charter文档质量评分≥7分（顾问评审）
- [ ] DCP评审有实质性讨论（非走过场）
- [ ] TR评审在试点项目中执行≥1次
- [ ] 流程问题记录和改进完整

---

<!-- chunk: ipd-phased-induction.阶段3-全面推广-3-4个月 -->
## 阶段3：全面推广（3-4个月）

### 目标
从试点扩展到全部新项目，制度化落地，建立内部持续运作能力。

### 关键动作

#### 全面推广

| 动作 | 说明 |
|---|---|
| 新项目全覆盖 | 所有新立项项目强制走IPD流程 |
| 存量项目管理 | 按现有进度纳入管理，补Charter |
| 扩展至MM/OR | 导入市场管理和需求管理流程 |

#### 制度化

| 动作 | 说明 |
|---|---|
| 制度文件发布 | IPD流程纳入公司管理制度 |
| 绩效挂钩 | 研发考核增加流程合规指标 |
| 流程Owner | 确定IPD流程Owner（通常产品管理部） |
| 内部教练 | 培养1-2名内部IPD教练 |

#### 绩效体系建立

| 动作 | 说明 |
|---|---|
| 双轨考核 | 项目绩效+部门绩效 |
| PDT绩效指标 | 商业结果+按时交付+质量 |
| 绩效结果应用 | 薪酬/晋升/项目分配 |

### 阶段3验收标准
- [ ] 所有新项目有Charter
- [ ] IPD制度文件正式发布
- [ ] 内部教练可独立主持DCP评审
- [ ] 绩效体系与IPD挂钩
- [ ] 团队满意度≥6分（10分制）

---

<!-- chunk: ipd-phased-induction.阶段4-持续优化-持续 -->
## 阶段4：持续优化（持续）

### 目标
数据驱动持续改进，与数字化系统集成，向行业最佳实践演进。

### 关键动作

| 动作 | 说明 | 频次 |
|---|---|---|
| 管理复盘会 | IPMT+PDT联合复盘，总结经验 | 季度 |
| 流程效率分析 | 研发效能指标追踪和改进 | 月度 |
| DCP评审质量评估 | 评审有效性分析 | 半年度 |
| 与系统深度集成 | PLM/项目管理系统与IPD流程集成 | 按计划 |
| 敏捷演进 | 结合Scrum/看板等敏捷方法 | 持续 |
| 行业对标 | 与行业最佳实践对比 | 年度 |

<!-- chunk: ipd-phased-induction.不同规模企业的导入策略 -->
## 不同规模企业的导入策略

### 小型企业（研发<50人）

| 阶段 | 内容 | 周期 |
|---|---|---|
| 阶段1 | Charter+3个DCP+IPMT | 4-6周 |
| 阶段2 | 1个试点项目 | 6-8周 |
| 阶段3 | 全面推广+制度化 | 4周 |
| 阶段4 | 持续优化 | 持续 |

**简化要点**：IPMT=CEO+CTO+市场VP，PDT=3-5人核心团队，不需要PMT/IRT

### 中型企业（研发50-200人）

| 阶段 | 内容 | 周期 |
|---|---|---|
| 阶段1 | 完整框架 | 2-3个月 |
| 阶段2 | 2-3个试点项目+TR评审 | 3-4个月 |
| 阶段3 | 全面推广+需求管理+绩效 | 3-4个月 |
| 阶段4 | 持续优化 | 持续 |

### 大型企业（研发>200人）

| 阶段 | 内容 | 周期 |
|---|---|---|
| 阶段1 | 完整框架+MM/OR | 3-4个月 |
| 阶段2 | 多产品线试点+TR+IRT | 4-6个月 |
| 阶段3 | 全面推广+绩效+系统 | 4-6个月 |
| 阶段4 | 数字化+行业对标 | 持续 |

## 风险管理

| 风险 | 概率 | 影响 | 应对措施 |
|---|---|---|---|
| 高层不支持 | 中 | 致命 | 绑定CEO痛点，拿CEO背书 |
| 中层抵触 | 高 | 严重 | 快速胜利案例，让大家看到效果 |
| 流程走形式 | 高 | 严重 | 顾问参与关键评审，严格把关 |
| 资源不足 | 中 | 中等 | 分阶段推进，不贪全 |
| 变革疲劳 | 中 | 中等 | 控制节奏，每阶段有明确成果 |
| 外部环境变化 | 低 | 中等 | 保持灵活性，框架不变细节可调 |
