### 阶段二：试点运行（6-8周）

**目标**：在真实项目中跑通流程，发现问题，调整优化

**关键动作**：
1. 试点项目辅导
   - 辅导PDT Leader撰写Charter
   - 顾问参与DCP0（立项评审），现场辅导
   - 每周1次跟踪会，发现并解决流程执行问题

2. 常见问题处理
   - PDT成员不配合（→ 升级到高层，强调IPMT权威）
   - Charter写不出来（→ 手把手辅导，用已有信息填框架）
   - DCP评审走形式（→ 要求每个问题有明确答案，不接受模糊）

3. 流程迭代
   - 收集试点反馈，调整不合理的流程设计
   - 更新工具模板和操作规范

**里程碑交付**：试点项目Charter + DCP0评审完成 + 流程优化版本

---

### 阶段三：固化推广（4-6周）

**目标**：从试点扩展到全部新项目，制度化落地

**关键动作**：
1. 全面推广
   - 所有新立项项目强制走IPD流程
   - 存量项目按现有进度纳入管理

2. 制度化
   - IPD流程纳入公司管理制度
   - 绩效考核增加流程合规指标
   - 建立IPD流程Owner（通常是产品管理部门）

3. 能力建设
   - 内部IPD教练培养（1-2人）
   - 知识库沉淀（典型案例、常见问题）

**里程碑交付**：IPD制度文件 + 全量推广完成 + 内部教练认证

## 常见阻力与应对

| 阻力 | 表现 | 应对策略 |
|---|---|---|
| 高层不重视 | IPD推进无人问津 | 绑定CEO痛点（研发失败成本），拿到CEO背书 |
| 研发抵触 | 「增加负担」「影响效率」 | 数据说话：试点后对比流片成功率/研发周期 |
| 市场不参与 | PDT成员缺席 | IPMT强制要求，纳入跨部门绩效考核 |
| 流程走形式 | Charter/DCP敷衍了事 | 顾问参与关键评审，提高评审质量标准 |
| 变革疲劳 | 员工对变革感到厌倦 | 快速胜利（先做一个成功案例），让大家看到效果 |

## 验收标准

### Phase 1 验收
- [ ] Charter模板定稿，通过IPMT审核
- [ ] DCP节点和评审标准文件化
- [ ] PDT运作规则文件化
- [ ] 关键人员培训完成（出勤率≥80%）

### Phase 2 验收
- [ ] 至少1个试点项目完成DCP0-DCP1
- [ ] Charter文档质量评分≥7分（顾问评审）
- [ ] DCP评审会有实质性讨论（非走过场）
- [ ] 流程问题记录和处理完整

### Phase 3 验收
- [ ] 所有新项目（最近3个月）均有Charter
- [ ] IPD制度文件正式发布
- [ ] 内部IPD教练可独立主持DCP评审
- [ ] 团队满意度调查≥6分（10分制）
