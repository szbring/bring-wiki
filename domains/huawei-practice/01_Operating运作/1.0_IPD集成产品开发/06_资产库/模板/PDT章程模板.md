## 一、PDT使命与目标

### 使命
本PDT负责 [产品名称] 的端到端开发，对产品的商业成功负责，包括：
- 按时、按质、按预算交付产品
- 产品上市后达成商业目标（收入/市占/客户满意度）
- 确保产品满足目标客户的核心需求

### 商业目标

| 指标 | 目标值 | 衡量方式 |
|---|---|---|
| 上市时间（TTM） | | GA日期 |
| 上市12个月收入 | | 财务数据 |
| 目标市场份额 | | 市场调研 |
| 客户满意度 | | CSAT调查 |
| 研发预算 | | 实际vs计划 |

### 产品核心指标

| 指标 | 目标值 |
|---|---|
| 核心功能特性数 | |
| 性能指标 | |
| 质量目标（缺陷密度） | |
| 成本目标（BOM/研发） | |

<!-- chunk: pdt-charter-template.三-pdt-leader权力 -->
## 三、PDT Leader权力

PDT Leader在本PDT内拥有以下权力：

| 权力 | 说明 |
|---|---|
| 任务分配权 | 分配PDT成员的工作任务 |
| 绩效打分权 | 对PDT成员的项目绩效有≥50%打分权 |
| 资源调度权 | 在批准预算内调度项目资源 |
| 内部仲裁权 | PDT内部冲突的裁决权 |
| 会议召集权 | 召集PDT全体会议的权力 |
| 信息知情权 | PDT相关所有信息的知情权 |

<!-- chunk: pdt-charter-template.四-运作规则 -->
## 四、运作规则

### 会议制度

| 会议 | 频次 | 参与者 | 时长 | 目的 |
|---|---|---|---|---|
| PDT周例会 | 每周 | 核心成员 | 60分钟 | 进度同步、问题决策 |
| DCP评审会 | 按节点 | PDT+IPMT | 2-4小时 | 商业决策 |
| TR评审会 | 按节点 | PDT+IRT | 2-4小时 | 技术评审 |
| 风险评审会 | 双周 | 核心成员 | 30分钟 | 风险管理 |
| 阶段复盘会 | 每阶段结束 | 全体 | 90分钟 | 经验总结 |

### 沟通规则

| 规则 | 说明 |
|---|---|
| 汇报路径 | PDT成员→PDT Leader→IPMT，不绕过PDT Leader |
| 问题升级 | PDT内部无法解决的，由PDT Leader升级IPMT |
| 信息透明 | PDT内所有信息对全体成员开放 |
| 决策时效 | 一般问题24小时内决策，重大问题48小时 |

### 变更管理

| 变更类型 | 审批权限 | 变更流程 |
|---|---|---|
| 需求变更（影响范围<10%） | PDT Leader审批 | 提交变更申请→影响分析→审批→执行 |
| 需求变更（影响范围≥10%） | IPMT审批 | 提交变更申请→影响分析→PDT Leader审核→IPMT审批→执行 |
| 进度变更（延期<2周） | PDT Leader审批 | 同上 |
| 进度变更（延期≥2周） | IPMT审批 | 同上 |
| 预算变更（±10%以内） | PDT Leader+财务代表 | 同上 |
| 预算变更（±10%以上） | IPMT审批 | 同上 |

## 六、绩效与考核

|---|---|---|---|

|---|---|---|---|---|
| PDT Leader | 80% | 20% | IPMT | — |
| 核心成员 | 70% | 30% | PDT Leader | 部门主管 |
| 扩展成员 | 50% | 50% | PDT Leader | 部门主管 |

<!-- chunk: pdt-charter-template.七-团队承诺 -->
## 七、团队承诺

### PDT Leader承诺
- 对产品商业成功负全责
- 公平分配任务和评价绩效
- 及时决策，不拖延
- 保护团队成员，勇于承担责任

### PDT成员承诺
- 按约定投入度参与PDT工作
- 对PDT目标负责，而非仅对部门目标
- 主动沟通问题，不隐瞒
- 遵守IPD流程和PDT运作规则
