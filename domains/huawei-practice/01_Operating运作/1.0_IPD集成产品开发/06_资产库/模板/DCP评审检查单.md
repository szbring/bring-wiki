# DCP评审检查单

<!-- chunk: dcp-checklist.使用说明 -->
## 使用说明
本检查单供IPMT成员在DCP评审时使用，确保评审覆盖所有关键维度，防止遗漏重要问题。评审前应逐项预审，评审会上逐项确认。

---

<!-- chunk: dcp-checklist.dcp0-概念评审检查单 -->
## DCP0 — 概念评审检查单

### 市场维度

| # | 检查项 | 通过标准 | 是否达标 | 备注 |
|---|---|---|---|---|
| M1 | 目标客户是否清晰定义 | 有具体的客户画像和场景 | ☐ | |
| M2 | 市场规模是否有数据支撑 | 有TAM/SAM/SOM估算 | ☐ | |
| M3 | 客户痛点是否真实验证 | 有客户访谈或调研数据 | ☐ | |
| M4 | 竞争分析是否完整 | 至少3个竞品的优劣势对比 | ☐ | |
| M5 | 差异化优势是否明确 | 至少2个可持续的差异化点 | ☐ | |

### 技术维度

| # | 检查项 | 通过标准 | 是否达标 | 备注 |
|---|---|---|---|---|
| T1 | 技术可行性是否验证 | 关键技术路径有POC或论证 | ☐ | |
| T2 | 技术风险评估是否完整 | 已识别TOP3技术风险及应对 | ☐ | |
| T3 | 系统架构方案是否初步确定 | 有系统架构概要 | ☐ | |
| T4 | 技术依赖是否识别 | 第三方技术/组件依赖已确认 | ☐ | |

### 商业维度

| # | 检查项 | 通过标准 | 是否达标 | 备注 |
|---|---|---|---|---|
| B1 | 收入预测是否合理 | 有3年收入预测，方法可解释 | ☐ | |
| B2 | 成本估算是否完整 | 包含研发/物料/认证/运营成本 | ☐ | |
| B3 | 盈亏平衡点是否可接受 | 回本周期在可接受范围内 | ☐ | |
| B4 | 投资风险场景是否分析 | 有悲观/基准/乐观三种情景 | ☐ | |

### 组织维度

| # | 检查项 | 通过标准 | 是否达标 | 备注 |
|---|---|---|---|---|
| O1 | PDT Leader是否到位 | 已确定PDT Leader | ☐ | |
| O2 | 核心团队是否组建 | 至少研发+市场代表已确定 | ☐ | |
| O3 | 资源是否初步落实 | 关键岗位人员有初步承诺 | ☐ | |

### DCP0整体判定

| 评判 | 条件 |
|---|---|
| **Go** | 以上所有检查项达标，或仅有1-2项轻微问题 |
| **Hold** | 有3-5项不达标，但可在1-2周内补齐 |
| **Kill** | 市场/技术/商业任一维度有致命问题（如市场不存在、技术不可行、回报极差） |

## DCP1 — 计划评审检查单

### 方案完整性

| # | 检查项 | 通过标准 | 是否达标 | 备注 |
|---|---|---|---|---|
| S1 | 详细技术方案是否完成 | 各子系统设计完整，接口定义清晰 | ☐ | |
| S2 | 产品规格是否冻结 | 规格基线已确认，变更走正式流程 | ☐ | |
| S3 | 测试策略是否制定 | 测试计划、用例框架已确认 | ☐ | |
| S4 | TR2/TR3是否通过 | 技术评审结论为"通过" | ☐ | |

### 计划可行性

| # | 检查项 | 通过标准 | 是否达标 | 备注 |
|---|---|---|---|---|
| P1 | 里程碑计划是否完整 | 所有DCP/TR节点有明确日期 | ☐ | |
| P2 | 资源是否确认到位 | 各部门已书面确认人员和时间 | ☐ | |
| P3 | 关键依赖是否识别 | 外部依赖有确认和备选方案 | ☐ | |
| P4 | 风险管理计划是否完整 | TOP5风险有应对和监控机制 | ☐ | |

### 预算与财务

| # | 检查项 | 通过标准 | 是否达标 | 备注 |
|---|---|---|---|---|
| F1 | 研发预算是否审批 | 预算已获财务确认 | ☐ | |
| F2 | 财务预测是否更新 | 基于详细方案更新收入/成本预测 | ☐ | |
| F3 | 投资回报是否仍可行 | ROI不低于立项时的80% | ☐ | |

### DCP1整体判定

| 评判 | 条件 |
|---|---|
| **Go** | 方案完整、计划可行、资源到位 |
| **Hold** | 计划有缺口但可在2周内补齐 |
| **Kill** | 方案不可行或投资回报大幅恶化 |

## DCP2 — 开发完成评审检查单

| # | 检查项 | 通过标准 | 是否达标 | 备注 |
|---|---|---|---|---|
| D1 | 开发工作是否全部完成 | 所有计划功能已编码完成 | ☐ | |
| D2 | 单元测试是否通过 | 覆盖率≥80%，关键模块100% | ☐ | |
| D3 | 代码审查是否完成 | 关键模块100%经过评审 | ☐ | |
| D4 | TR4是否通过 | 技术评审结论为"通过" | ☐ | |
| D5 | 测试环境是否就绪 | 集成测试环境已搭建 | ☐ | |
| D6 | 上市准备工作进展 | 市场物料/渠道准备按计划进行 | ☐ | |
| D7 | 遗留问题清单 | 无P0问题，P1问题有解决方案 | ☐ | |

### DCP2整体判定

| 评判 | 条件 |
|---|---|
| **Go** | 开发完成、质量达标、测试就绪 |
| **Hold** | 有少量遗留问题，1-2周可解决 |
| **Kill** | 开发严重延期或质量问题严重 |

## DCP3 — 发布评审检查单

| # | 检查项 | 通过标准 | 是否达标 | 备注 |
|---|---|---|---|---|
| R1 | 集成测试/系统测试是否通过 | 测试用例通过率≥95% | ☐ | |
| R2 | P0/P1缺陷是否全部关闭 | 0个P0，0个P1 | ☐ | |
| R3 | P2缺陷关闭率 | ≥95%，遗留的有规避方案 | ☐ | |
| R4 | 客户验证是否通过 | Beta客户验证满意 | ☐ | |
| R5 | TR6是否通过 | 技术评审结论为"通过" | ☐ | |
| R6 | 量产/交付是否就绪 | 供应链/制造准备完成 | ☐ | |
| R7 | 上市物料是否就绪 | 产品文档/营销物料/培训完成 | ☐ | |
| R8 | 运营支持是否就绪 | 客服/技术支持已培训 | ☐ | |

### DCP3整体判定

| 评判 | 条件 |
|---|---|
| **Go** | 质量达标、发布就绪 |
| **Hold** | 有少量P2缺陷或上市准备未完成 |
| **Kill** | 质量严重不达标，强行上市会损害品牌 |

## DCP4 — 生命周期评审检查单

| # | 检查项 | 通过标准 | 是否达标 | 备注 |
|---|---|---|---|---|
| L1 | 产品销售是否达到预期 | 实际收入vs预测≥80% | ☐ | |
| L2 | 市场份额趋势 | 稳定或增长 | ☐ | |
| L3 | 产品盈利状况 | 毛利率在目标范围内 | ☐ | |
| L4 | 竞争态势变化 | 是否出现颠覆性竞品 | ☐ | |
| L5 | 客户满意度 | CSAT≥目标值 | ☐ | |
| L6 | 下一代产品规划 | 是否有替代产品在研 | ☐ | |

### DCP4整体判定

| 评判 | 条件 |
|---|---|
| **继续投资** | 商业表现良好，继续推广 |
| **优化调整** | 有问题但可优化，调整策略 |
| **计划退市** | 商业价值不再，启动退市流程 |
